General Information

This on Prem | protel I/O Support Tool - User Guide will guide you through the basic functions of the IO Support Tool. If your hotel is using an IO interface, the IO Support Tool will show you exactly what is being sent to the third party through the interface, and what information is being received from the third party. Furthermore, it shows you how many messages have been processed for your IO connection. I will now show you how to use the IO Support Tool, and how it can improve your daily business.


User Guide




How to start to the protel I/O Support Tool?

To start the on Prem | protel I/O Support Tool - User Guide, please open your protel OnPremise Front Office. At the top of the screen, click on “Add-Ons". 


In the Add-Ons you will notice a new Tab called “IO Support Tool”. Just click on “Run Application” and the Support Tool automatically opens and displays the Info Tab.


Info Tab

The structure of the different tabs in the IO Support Tool is always the same. A general information line is displayed at the top of each tab. It contains the PWS Version number, the PMS Version number, the MPE Hotel ID, the CloudHotel ID, and the SQL Database to which the IO Support Tool is connected.


Under the general information, you can select the filter options for the IO Messages. For all Filters option, use the Refresh Button to see the current status. The following filter options are possible.

  • “MessagesPending”

  • “MessagesInProcess”

  • “MessagesInProcessStatus”


With the “MessagesPending” filter option, all messages that are waiting to be processed are displayed in this queue. The number should be 0, or at least the messages should go to zero. If the messages are not processed for a longer period of time, protel IO is not working as expected. In such a case, please inform the protel Support Team.





The second filter option “MessagesInProcess” displays all messages that are processed for a specific time-period. In the “From” and “To” box, you can select the start and end dates, and with a click on “Refresh” the Diagramm will be updated. The diagram shows the total number of messages grouped by message type.





With the last filter option - “MessagesInProcessStatus”-, again all messages that are processed for a specific period are displayed. However, in this filter the messages are grouped by the status and not by the message type, like in “MessagesInProcess”.






IO Messages Tab

The next tab is the "IOMessages” tab. In this tab, you will find an overview of the configured IO messages and you will find information about which messages have been enabled for your hotel. Of course, if your hotel is part of a multi-property selection, you can select your specific Hotel or check the entire MPE.

In this table, you can find the MPE Hotel Number, the Hotel Code, the name of the Message Type and if for the specific Message a trigger is needed. Of course you can check as well if a specific message type has been activated, or not. Triggers are only needed for specific message types. This includes Messages which are, for example, sent out because of an action in the Front Office, such as changing the Profile, or Reservation. Please note that this table only shows the configuration and it is not possible to change it.

LineChart

The graph in the next tab - the “LineChart” tab - allows you to check the IO message flow. The graph shows the exact time a message entered the "MessagesPending" queue and the time this message was processed as a “Working Message”.

You can decide for which period you would like to check the graph. But the graph will only display a time period in which messages were processed.

IncomingMessages

In the "IncomingMessages" tab, you can check all messages that have been imported into the PMS by protel IO. Once again, select the period for which the incoming messages should be checked.

In addition, it is possible to select whether "All Message Types", or only a certain message type are to be displayed.

The result is displayed in a table with Correlation ID, Date, and MessageType. Successfully imported messages are highlighted in green. Messages that were imported successfully, but with a warning, are displayed in yellow. And failed messages are displayed in red.


To get further information about the messages, just click on the specific message. On the right side of the screen, both the incoming message and the response message are displayed.







In addition to the "From" and “To" date and message types search, it is possible to add the Correlation ID to the search. The Correlation ID can be entered in this field here. Once the "Extend Search" checkbox is ticked, the search can also be extended as required. The "Extend Search" can only be executed for internal or external reservation IDs, or last names.

OutgoingMessages

The "OutgoingMessage" tab is built up identically to the "IncomingMessage" tab. But here, all messages sent from protel IO to the third party are displayed. Again, you can select a time period for which the outgoing messages can be checked.

In addition, you can also decide to display "All Message Types" or only a certain message type.

The results not only display the Correlation ID, Date, and Message Type, but also the Status of the Message. Successfully exported messages are highlighted in green, messages that failed during the creation process are highlighted in red.






To get further information about the messages, just click on the specific message. On the right side of the screen, only the outgoing message appears. In addition to the "From" and “To" date and message types search, it is possible to add the Correlation ID to the search. The CorrelationID can be entered into this field. The "Extend Search" which is implemented in the "incomingMessage" tab is not implemented for the "OutgoingMessage" tab.
