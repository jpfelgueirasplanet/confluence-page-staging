Bob installation: one for pAir (if not existing) and one for the External Fiscalization Service

    Data Security

    Please have in mind that both vendor and IO interface token information is sensitive information that might be transported by Slack, team viewer session in case of a 1:1 session,
    or with two-way authentication methods by other media. 

  • Special Bob configuration 

    The pAir Installation must hold the name of the fiscal service in IO.Gateway.IO_InvoiceFiscalizeRQ.TargetModule attribute, e.g.

One way Installation

Screenshots

1

Fiscalisation needs to be activated and configured for the property (FiscalCodes, Invoice number ranges etc.)

Leads to the following entries in Config Editor

  1. Configuration/currentProperty/propData/Fiscalization/Config/activateFiscalization : true
  2. Configuration/currentProperty/propData/Fiscalization/Config/autoFiscalization : true
  3. Configuration/currentProperty/propData/Fiscalization/Config/preSelectedFiscalCodeID: 1
    (uses the internal ID of the fiscal code which should be taken by default for fiscalization)

2

Additional Config Settings

  • Configuration/currentProperty/propData/Fiscalization/Config/preSelectedFiscalizationMode: "BOTH"

    The following settings should be created automatically after the BOB Installation has been added as described above:

  • Configuration/currentProperty/propData/Interfaces/IO/Gateway/IO_InvoiceFiscalizeRQ/Disabled: false
  • Configuration/currentProperty/propData/Interfaces/IO/Gateway/IO_InvoiceFiscalizeRQ/TargetModule: "com.developer.module"

3

Communication could be tested by fiscalizing an invoice in FO

  • Verify in Kibana that outgoing IO_InvoiceFiscalizeRQ is answered by IO_InvoiceFiscalizeRS and no error occurs
