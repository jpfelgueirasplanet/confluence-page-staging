Installation Steps

 

General steps

Examples

1

If there is the old interface via IDS in use: Deactivate it before you switch to the new one because of incoming reservations

2

Hotelpartner uses a different Token for each IO installation, but the same Endpoint.

Request authentication details (only the Token is needed) from the Integration partner for the new installation and request them to send you the authentication details on private Slack message or encrypted file by email and inform you by phone about the password.

Please provide them the HotelID as well - internally for Protel this is the CloudID

One way Installation

2

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


3

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ)
    for the last 2 years in the past and 2 years in the future, starting with the 1 January and ending 31 of December

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


4

Provide Hotelpartner the Market codes and Source codes from the System Data



   

5

Configuring an IDS Instance for I/O: How to configure a protel I/O IDS 

  • Special IDS configuration

Important note: The IDS is needed only for the reservation Import for HotelPartner

  • ESB server connection
  • Add new instance
  • Comserver URL
  • URL for Import
  • SQL database
  • Configure the Email Notification
  • Token and Module Name in the Special Tab
  • Activation of PCI Proxy is not required
  • the configuration for the children age range -  supports only Description 8 (children) - double check if the customer will use it
  • Create DCS mapping (ask Hotelpartner and the customer which rate plan codes / room types / Source codes are needed to be mapped in Protel and then include them in the DCS matchings. Please note that the codes are case sensitive and have to be matched exactly as they are mentioned - double check with HotelPartner how they send the values, as per our mapping or if they use their own values)


    Note: for a connection test, one rate code and room type is enough to be mapped in the DCS mapping





DCS mapping




6

Start a full data export after all settings in the IDS are done, to make sure that Hotelpartner receives the availability data.


7

Data Validation Report - DQE

  • after checking if the bulk successfully finished, provide the integration partner the Hotel Status Report with the Net values. Hotelpartner will process the DQE and send you a report afterwards. 
  • The Integration partner will process the DQE and send you a report afterwards
  • The report is to be found in the tab Add-Ons of the front office

Test incoming Rate update

(OTA_HotelRatePlanNotifRQ)

  • the integration partner should ask the customer to create a Test rate code
  • after the Test rate code has been created, ask the integration partner to push some rates
  • Make sure that the rate which needs to be updated is set to yieldable. Ask in the channel for which day the rate has been pushed, check the date and send a screenshot from the Protel Daily Rates if the rate for the specific date is showing in the list. 

  • as soon as this has been tested successfully repeat the tests for incoming rate updates chosen by the customer (normally this should always be one and the same rate)

    Note
    Hotelpartner usually uses ony one rate for the reservation import and rate updates, but can use multiple rates as well.

6

Test incoming Restrictions 

(OTA_HotelAvailNotifRQ)

  • ask the Integration Partner in the Slack channel of the integration to push a restriction for a specific day on the Test rate code and then check in the Price availability for the given date if the restriction has been set. 
  • ask the Integration partner to clarify with the customer to have the Test rate code invalidated (through expired validity) after the verification of the Restriction message is done

    The following restrictions are supported by Hotelpartner:

      -  Close Out

      -  Close to Arrival
      -  Minimum Length of Stay by stay

    If the property is a Best Western customer:
    - ask Hotelpartner to test a restriction on a rate group level that has been agreed with the customer
    - ask Hotelpartner for which rate group level has the restriction been set, so you can check in the system if the restriction was applied
