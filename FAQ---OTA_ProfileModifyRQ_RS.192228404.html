select collection, codevalue, mpe_hotel, 'roompref' as preftype from proteluser.g2rfeat rf

Missing LastModifyDateTime Attribute

In case that an inbound OTA_ProfileModifyRQ/OTA_ProfileCreateRQ is missing the not mandatory attribute @LastModifyDateTime.

protel updates the entry in kunden.changed with the value set @TimeStamp attribute.
For the case that even @TimeStamp is missing in the RQ, then protel will update the entry with the current business date value.

  • No@LastModifyDateTime Attribute --> OTA_ProfileCreateRQ/OTA_ProfileModifyRQ | @Timestamp should be used
    TimeStamp="2021-04-16T14:08:04Z" | kunden.changed="2021-04-16 00:00:00.000"

  • No @LastModifyDateTime & no @Timestamp | kunden.changed="2021-07-16 00:00:00.000"

onPremise / pAir

How can Integration Partner delete data from a Profile?

Rule for deleting data from profile

pAir: No profile data is deleted

We use protective approach to secure the data in the PMS. Thus - no deletion is applied on existing profile data UNLESS an OTA_ProfileModifyRQ for the profile is sent

e.g.: Email address exists in the PMS profile record. Inbound reservation is sent where

  • <Email/> is sent. As result in the PMS new empty line for email is created
  • <Email/> element is not sent a all. As result in the PMS nothing changes for the email
  • new <Email DefaultInd="true" EmailType="1">test111386@protel.de</Email> is sent. As result in the PMS new email address is added to the guest profile - but the old one still remains

PWS:

Single Attribute/Element (primary Email for example)

  • If element is provided but empty => Result: remove data from profile
  • If element is not provided at all => Result: the data is not touched from the profile

List of Attributes/Elements (secondary Email for example)

  • If element is provided but empty => Result: the data is not touched from the profile
  • If element is not provided at all => Result: the data is not touched from the profile
