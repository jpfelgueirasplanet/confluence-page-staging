Special XSetup Configuration: 

On Global settings, add the following xsetups:

  • profiles | optin | 1

  • Optin | default_type2_method0 | 2

  • Optin | default_type2_method1 | 2

  • Optin | default_type2_method2 | 2

  • Optin | default_type2_method3 | 2 

Where "2"="Not Asked".

Those values are meant to serve as default values in the cases where the Guest optin/out preference has not been explicitly set. In general I/O messaging uses only section "Advertisement".

NOTE

 If there is no default setting via XSetUp, the PWS will be exporting "No" instead, which would lead to inaccurate data exchanged. 

 Do inform the hotel of the default setting. We recommend that this is not changed. 

 For pws 2.16 or newer, the xsetup can be added from the pws protel IO Configuration menu "Profiles"

XSetup: 

On Global settings, add the following xsetups:

  • Profiles | mailingdefault_0 | 0
  • Profiles | mailingdefault_1 | 0
  • Profiles | mailingdefault_3 | 0

Where mailingdefault_0 sets the default values in the individual profile UI in the absence of individual contact defaults to be "NO" and will send the value "NO" in the "ShareMarketInd" fields in the xml message. Mailingdefault_1 controls the default settings for company profiles. Mailingdefault_3 controls the default settings for travel agent profiles.

These fields correspond to the contact methods. This setting should be set to "NO" for all sites unless agreed in advance with the site (*Not Asked will be sent as well as "NO" to the integration partner)

  • Address (this is letter in the UI)
  • Phone (including text message)
  • Email


NOTE

 Do inform the hotel of the default setting. We recommend that this is not changed.

Special CKit Configuration

xn_auto_gen_optin

Please backup the tables kunden and optin in case you overwrite the wrong settings!

Explanation

This ckit works in conjunction with the previously outlined optin and profile x-setups. The function of this ckit is to ensure that the individual preferences specified for "Advertisement" are reflected correctly in both the profile UI, and in the corresponding xml messages sent from via protelIO. 

In addition to populating the defaults in the optin table for new profiles it also can be used generate optin in the optin table for any profiles that haven't previously created optin entries

If this ckit is not installed at a site there is a possibility that the profile UI will not match what has been configured in the x-setup and that the optin table will not have values for all profiles

The confluence page below explains the  testing and functionality of this ckit

Installation guide - Creation of All Optin Settings for All Profiles

The main difference between these two versions is that version "A" will ignore individual optin values for new profiles in favour of the mailing default, whereas version "B" will take the individual optin preferences into consideration when creating new profiles.

Please create a support ticket with Partner support to receive the latest version of these ckits in the advent that access to the confluence page is denied 

Installation 

This ckit should be placed in the standard xml folder within the system directory and does not require to be registered in system data.

Ramifications of no ckit installed

If the ckit was not present and the user was to have the profile x-setup "mailingdefault_0 = 1" in place for example, then all individual preferences are ignored.

In this scenario, all permissions would be set to "Yes" instead as the "mailingdefault" setting is currently overriding the individual preferences.

The resulting default profile would appear as this, without the individual permissions being expressed as required.

2

Special PWS Configuration:
  1. Open pws, menu Settings / Protel I/O Configuration / sub-menu "CRM" ( on pws v 2.16 and higher it is sub-menu "Profiles")
  2. Choose the proper hotel (if MPE) from the top right corner
  3. Under "Bulk Transfer Service" choose the option "All Profiles"
  4. Click on "Save" at the bottom right

This specifies which profiles will be sent during the bulk upload. Must be set before the bulk upload.

 

3

Special PWS Configuration:

  1. Open pws, menu Settings / Protel I/O Configuration / sub-menu "General Settings" ( on pws v 2.16 and higher it is sub-menu "Reservations")
  2. Choose the proper hotel (if MPE) from the top right corner
  3. Check the box for "Include attached profiles in HotelResNotif".
  4. Click on "Save" at the bottom right

This will insert the following xsetup pws | io_include_attached_profiles_in_hotel_res_notif | 1

If there is an attached profile to a reservation, this would appear under element <ResGuest> / <RelatedTraveller> in the OTA_HotelResNotifRQ

 

4

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


5

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ) for the last 10 years and ending in 31.12.2099

  • for profiles (OTA_ProfileCreateRQ) for the last 10 years and ending in 31.12.2099

  • for  guest revenue detalis (OTA_HotelStayInfoNotifRQ) only in cases integration partner and customer request  

    NOTE

     If it is new hotel or there has been data migration of guest profiles, then there will be new profiles in the hotel where it is possible to have the protel default date assigned on them as "1900-01-01" and last modified date "1900-01-01". Those profiles are not included in the normal bulk upload. As one-time case - create the bulk job for the period [1900-01-01 : 31.12.2099] 

BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one.

  • The report is to be found in the tab Add-Ons of the front office

  2. Choose the period of the bulk 
  3. Match availability counts per day of stay

Test incoming OptIn change

(OTA_ProfileModifyRQ)

Please be aware that protel IO supports only the optin values set in the "Advertisement" section of the profile "Contact opt-in"

Testing if optin and mailingdefaults values are set to "YES"

  • Create an individual profile in protel and verify that the profile advertisement permissions are all set to "YES" in line with default xsetups
  • Provide SmartHost with the CorrelationID and profile number
  • ask the Integration partner to amend the ShareMarketind permissions for email, phone and address/letter from the defaults in the profile you created to "NO"
  • ask the Integration partner to provide the CorrelationID for the OTA_ProfileModifyRQ and verify its presence in Kibana
  • check that the corresponding OTA_ProfileModifyRS contains no errors
  • check in the OTA_ProfileModifyRQ, ShareMarketInd = "NO" for the advertisement contact methods and ShareMarketAllOptInd = "NO"
  • verify the Advertisement contact methods in the profile are displayed as "NO"
  • repeat the process for company and travel agent profiles. 

Note

The integration partner only sends unsubscribe from email
