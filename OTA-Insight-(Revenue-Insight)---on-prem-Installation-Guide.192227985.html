SetFormula(IsNet,1);
{ptemp_hotelstatus.stationid} = STATIONID()

PWS Restart Ckit

There can be an issue occasionally where the "Fiscal date" in the "HTNG_HotelFolioNotifRQ" is cached by the PWS and not updated.

This can lead to OTA_Insight not being able to calculate actual revenue currently as the actual revenue can appear after the fiscal date

A restart of the pws will update the cache with the current system date and this will resolve the issue.

In order to reduce the chances of this occurring in light of a permanent fix, a windows scheduled task can be configured to call a batch file to restart the windows service daily 

Attached is a sample batch file that can be used in conjunction with the windows task scheduler to restart the services daily.

***Please note that the name of the protel webservice in the batch file will need to reflect the name configured at the property and that the windows scheduler would need to be set to restart the pws daily***

One way Installation


1

PWS Configuration

  • enable LiveMode and activate ToggleTrigger for required message types for 1-way connection

Message type

LiveMode

ToggleTrigger

OTA_HotelResNotifRQ

OTA_HotelInvBlockNotifRQ

OTA_HotelStatsNotifRQ

HTNG_HotelFolioNotifRQ

Please make sure that "HideRateAmountExtensions" is not ticked. 
OTA Insight is requiring those numbers for calculation. 

2

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation.
    Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 

Note: 
Please make sure that all your tests are cancelled properly from the client´s database once the testing is done.


3

Create a test charge

  • Create a new test reservation with arrival date = today and check in the reservation. 
  • Test to charge an item on the client´s invoice in order to trigger an HTNG_HotelFolioNotifRQ. 
    Verify in Kibana that outgoing HTNG_HotelFolioNotifRQ is answered by HTNG_HotelFolioNotifRS and no error occurs. 

Note: 
Please make sure that all your tests are cancelled properly from the client´s database once the testing is done.


4

Create a test allotment 

  • Create a test allocation/allotment (block) in order to test the OTA_HotelInvBlockNotifRQ. 
    Verify in Kibana that outgoing OTA_HotelInvBlockNotifRQ is answered by OTA_HotelInvBlockNotifRS and no error occurs. 

Note: 
Please make sure that all your tests are cancelled properly from the client´s database once the testing is done.


5

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ)
    for the last 5 years and ending with 31.12.2099.

  • for folio details (HTNG_HotelFolioNotifRQ) 
    for the last 5 years and ending with 31.12.2099.

  • for blocks (OTA_HotelInvBlockNotifRQ) → PWS release >= 2.13.x needed
    for the last 5 years and ending with 31.12.2099.


BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one.

Data Validation Report - DQE


  • after checking if the bulk is successfully finished, provide the integration partner with the report, and please note the schedule required for the DQE report 
  • During the installation: The installer or installation partner  is to send OTA Insight a bulk and DQE
  • 1 week after installation: The installer or installation partner is to send OTA Insight a new DQE.  If the data is acceptable, OTA Insight will activate the hotel.
  • 4 weeks after installation: The Installer or installation partner  is to send a new DQE to verify the data is still acceptable
  • The Integration partner will process the DQE and send you a report afterward
  • The report is to be found in the internal reporting
  1. Please have a look at Step 5 of the General Preparation in order to see how to set up the report.
  2. Run the report "k00553_io_allot" for the whole timespan you want to compare.

  3. Export the result as an excel file and forward it to OTA Insight for comparison.
