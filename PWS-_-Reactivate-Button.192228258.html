PWS | Reactivate Button

  • PWS > 3.6.X
  • PWS < 3.6.X



PWS > 3.6.X

The "Hardwaretoken" was implemented in the PWS. It is intended to ensure that no two PWS are connected via WebSocket to the same Database. The hardware token is initially calculated for new IO customers and set in the xsetup. With every PWS start, the hardware token is calculated again and compared with the one in the xsetup.  I/O only starts if it is the same.


X-setup

The "hardware token" is stored in this xsetup 

If the "stored Hardwaretoken" is different from the newly created "Hardwaretoken" the following Popup will appear as soon as the IO Dialog is started


If yes:

The service will be restarted and a new token will be set for this machine. Old Websocket connections will not work anymore.


If no:

Nothing will be done. IO connection will not work for this PWS.

PWS < 3.6.X

The "Hardwaretoken" was implemented in the PWS. It is intended to ensure that no two PWS are connected via WebSocket to the same Database. The hardware token is initially calculated for new IO customers and set in the xsetup. With every PWS start, the hardware token is calculated again and compared with the one in the xsetup.  I/O only starts if it is the same.


X-setup

The "hardware token" is stored in this xsetup 


If the "stored Hardwaretoken" is different from the newly created "Hardwaretoken" the following Button will appear in the PWS Dialog


The button should therefore never be pressed when importing a customer database into the test environment (unless the hotelID and other x-settings have been changed from the production settings to test) This button should be pressed when moving servers or making similar changes to the server.

If the button is clicked, the Xsetup is cleared.

Now the PWS needs to be restarted, because otherwise no new "Hardwaretoken" will be set and stored by the PWS.
