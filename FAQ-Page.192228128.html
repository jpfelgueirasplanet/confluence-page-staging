In case these changes will not be applied in time, run sync on Pair!

CRS currency error

Hotelpartner sends a Booking.com reservation in EUR to a hotel in the Czech republic that only uses CZK as currency. Neither Hotelpartner nor Protel PMS has a service to convert the currency, and the reservation is booked with wrong revenue in the PMS.

The hotel customer should create a rate code in the PMS that uses EUR and is only used by Hotelpartner.

Tax and breakfast articles should be created as well in EUR in the PMS.
