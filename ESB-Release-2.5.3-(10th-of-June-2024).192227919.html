ESB Release 2.5.3 (10th of June 2024)

Ticket ID

Description

Solution


IO-8052

New Proxy for customized integrations

Epic of tasks.

Create new "Pull Proxy" service. The service is meant to be used for special integrations such as GHA, Delphi, HotSOS. It will replace completely the current OXI PROXY.

By the default the new "Pull Proxy" is not activated for the GHA customer that are already live. The activation is done per property based on a value in one of the environment variables. If the new value is not provided there - then the old known OXI PROXY flow is applied for that property.

Note - the service will not be activated on live hotels, yet!


IO-7721

XSLT | Apply mappings on ESB-IN

Currently the Mappings based on the message type IO_MappingNotifRQ are applied only on the direction of msg RQs from PMS to vendor. Now mappings will be possible to be applied vice versa.


IO-7878

IO_MappingNotifRQ | outbound | Logic refactoring for mapping duplication

Extend the message content storing in ESB for mappings to use an extra attribute "Options" that tells in which direction the mappings should be applied - from PMS to vendor or vice versa. Works in combination with ticket IO-7721


IO-8047

BTS | Automatic cancel of job where TargetDestination is not responding

  1. If a vendor service is set to DeadEP, then it will not be possible to start a bulk for this service. In the SMP UI the following error message will appear:
    "...Recipient for Hotel XXXX has a DeadEP and can not receive any messages. Investigate, make sure it is fixed and contact them to choose a better timing for a new bulk export..."
  2. If a bulk job is running and in the meantime the recipient EP is set to DeadEP, then the bulk job will be set to "Cancelled" and can be re-started only after the vendor service is up and running.


IO-8093

Map2Any | Missing Mapping Filter

If a message delivery fails due to missing mapping filter, then create a generic ESB RS to the sender to close the workflow.


IO-8094

PWS Safety Net | Extend RS message SOAPHeader for io.protel.pms destination

If the destination is io.protel.onpremise, then extend the soap header of the message by added a list of all services that are active and subscribed for the message and the name of the service that sent the current message to the PWS. The purpose of the extension is to try and build a PWS safety net by telling the PWS which services sent a RS and which didn't.


IO-8156

OpenSearch | Set automatic cleanup

Set automatic cleanup of logs to preserve the proper state of allocated resources.


IO-8157

FPS | Misleading ErrorDetails content

Provide more detailed description in the case of "Message expired and dropped"


IO-8174

ESB-OUT | Add timestamp to generic ESB RS message

Extend the ESB generic RS by adding the actual msg timestamp in the root element. The purpose is to facilitate error research initiated by customers.


IO-8202

SynXis | Edit OTA_HotelResNotifRQ from PMS to SynXis for PrimaryInd references

The logic in pAir of handling primary indicators, does not match the one of SynXis. The filter edit will convert the pAir logic to the SynXis one by using TPA_Extensions/@PrimaryInd values from PMS and setting them to the ResGuest/@PrimaryIndicator ones for SynXis


IO-8206

ESB-FPS | Generic Error Response & Logging

In case of timeout for delivering messages of sensitive flows (e.g. sync flow for payments), the ESB does not do retry - instead - it wait 55 sec and then logs a generic RS to the vendor to inform them that there is a problem and they need to investigate further. The ticket is to extend the generic error message so that it gives more base for research on the vendor side.
