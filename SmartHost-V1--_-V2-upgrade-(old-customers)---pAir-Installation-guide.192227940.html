Installation Steps

 

General steps

Examples

1

Request authentication details from the Integration partner for the new installation and request them to send you the authentication details.

Please provide them the HotelID as well - internally for Protel this is the CloudID 

Data Security

Please have in mind that both vendor and IO interface token information is sensitive information that might be transported by Slack,
team viewer session in case of a 1:1 session, or with two-way authentication methods by other media. 

  • Inactivate or delete the old SmartHost installation for the specific HotelID (com.smart-host)

  • Create a new installation with the V2 Manifest Name (com.smart-host.crmv2)

  • Authentication method using the same Endpoint as default and a User and a Password for each hotel customer (if this is changed by the integration partner)
    https://api.smarthosts.co/protel

Two way Installation

Examples

1

  • Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner . Make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs.

  • Create a profile  and add an email address. Set the marketing allow information to "Allow"
    Ask the integration partner to change the Marketing Allow information to "NotAllow"
