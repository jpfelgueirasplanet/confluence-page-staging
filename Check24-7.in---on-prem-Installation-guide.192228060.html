8

BAR Rates in the  OTA_HotelAvailRS Message

BAR rates returned in OTA_HotelAvailRS are in line with those in the F11 BAR mode and WBE. The message will display the lowest rate per room type from the BAR group, irrespective of whether it is a virtual or base rate.The message will not display all rates assigned to the BAR rate group.If a rate is required that is not present because it is not the lowest current rate in the BAR Group, then this rate must be moved from the BAR group into a standard group

12

Errors regarding the HTNG_ChargePostingRS

  • The third-party would not be able to process the payment because it took more than 30s for the HTNG_ChargePostingRS to be sent out. In Kibana The HTNG_ChargePostingRS will include a success

 To mitigate the problem the following configuration settings are suggested as may  reduce the comserver load

  1. Make sure PWS is connected to its own taskserver
  2. The new taskserver should have nothing else running against it, no ckits, no IDS
  3. IDS should be pointed to its own taskserver and not the comserver
  4. Taskserver should be set to reload system data cache every 60 minutes
    TaskServer_Setup
  5. Set this on taskserver.ini as that is where the pws is pointing
    [system]
    debugevtimerexecutiontimer=1
    thresholdevtimer=900
  6.  Make sure passerby account is configured and available
  7. Add the xsteup on global level
    ComServer | Whitelist | GetComputerName 
  8. If there is a timeout or error, check what the taskserver displays on the screen.

1

  • enable LiveMode and activate ToggleTrigger for required message types for 1-way connection

Message type

LiveMode

ToggleTrigger

2

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


3

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ)
    starting on the installation date and for the next 10years

BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


4

Data Validation Report - DQE


  • after checking if the bulk successfully finished, provide the integration partner the report
  • The Integration partner will process the DQE and send you a report afterwards
  • The report is to be found in the tab Add-Ons of the front office

5

Configuring an IDS Instance for I/O: How to configure a protel I/O IDS 

These points from the general page should be covered

  • Add a new instance
  • Configure the Comserver URL
  • Add the URL for Export
  • Add the SQL database
  • Configure the Email Notification
  • Add the Token and Module Name in the Special Tab
  • Enable "fake" sync mode
  • Import
    • Import - Reservation - Common
      • Mode MUST be set to PULL and PUSH
    • Import Children
  • Create DCS mapping /mpehotel / RoomType / RateCode / 

 Note: If automated matchings are used in the IDS configuration, then the IDS will create the matching for the respective category itself. But this will disable the option to do any adjustments on the auto-matched list. Thus, it is recommended to do the matchings manually

 For the purpose of the installation test, it is enough to do the mapping for a few room/rate combinations. Then instruct the Customer how to manage the mappings on their own

6

  • Please activate in the IDS the "Children age ranges" (Config see sample screenshot)

3

Test an incoming Availability Request
(OTA_HotelAvailRQ)

  • Ask the partner to request for room availability
  • Check in Kibana that the incoming OTA_HotelAvailRQ is answered by OTA_HotelAvailRS and contains no errors.



4

Test an incoming Room Assignment
(IO_ResModifyNotifRQ)

  • Request the partner to assign a room to the test reservation created during OTA_ReadRQ and provide the room number
  • Check in the PMS if the Reservation has allocated a RoomID successfully
  • Check whether the room has the correct status "clean" or "checked"
  • Verify in Kibana that incoming IO_ResModifyNotifRQ is an swered by IO_ResModifyNotifRS and no error occurs
  • Please Note from PWS >=2.20.4 the automatic room assignment during check-in is possible



5

Test an incoming Check In 
(HTNG_HotelCheckInNotifRQ)

  • Ask the integration partner to check-in the test reservation
  • Check in the PMS if the Reservation has checked in successfully
  • Verify in Kibana that incoming HTNG_HotelCheckInNotifRQ is answered by HTNG_HotelCheckInNotifRS and no error occurs



6

Test an incoming invoice detail request
(HTNG_HotelFolioRQ)

  • Ask the integration partner to request the folio of the previous reservation.
  • Verify in Kibana that incoming HTNG_HotelFolioRQ is answered by HTNG_HotelFolioRS and contains no errors.
  • open the invoice in the pms and compare the amount of the invoice with the amount of the integration partner



7

Test an incoming Posting
(HTNG_ChargePostingRQ)

Testing Revenue Posting

  • Request the integration partner to post a positive amount on the folio using the previous test reservation
  • The partner should use the TAA  codes you provided during the general preparation step
  • Check that the SubTypeID="Revenue" in the HTNG_ChargePostingRQ that the integration partner is sending
  • open the folio in the PMS and check if the amount was posted correctly
  • The partner should void the posting by sending a negative amount

Testing Payment Posting

  • Request the integration partner to post a negative amount on the folio
  • The partner should use the payment methods codes you provided during the general preparation step
  • Check that the SubTypeID="Payment" in the HTNG_ChargePostingRQ that the integration partner is sending
  • Check if the amount was posted correctly
  • The partner should void the posting by sending a positive amount

Test an incoming check-out

(HTNG_HotelCheckOutNotifRQ)

  • Create a day use reservation (arrival date needs to be current PMS date) without revenue on the invoice and provide the reservation number to your integration partner
  • Request the integration partner to check-out the reservation
  • Check in the PMS if the Reservation has been check-out
  • Verify in Kibana that incoming HTNG_HotelCheckOutNotifRQ is answered by HTNG_HotelCheckOutNotifRS and contains no errors.



10

Test an incoming Reservation 

(OTA_HotelResNotifRQ)

  • Ask the partner to create a test reservation and provide the arrival and departure date of the reservation
  • verify in the PMS whether the reservation was created
  • Check in Kibana that the incoming OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and contains no errors
  • Cancel the reservation after the verification is done.
