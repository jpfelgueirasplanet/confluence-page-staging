Special configurations:

  • Set a default cash register no. for protelIO integrations if it is not already one.

This should be clarified in advance with the customer in case that there has to be a new, specific cash register to be set up. 

In order to register the incoming PostPayments with a specific cash register number in pAir, the below configuration is needed to define and map a default cash register to tag on the incoming post payments through the HTNG_ChargePostingRQ.

1.Define a default cash register in pAir

System data / Financial Configuration/ Cash Registers 


2.Add an xsetup that is used to define the default cash register for all IO-related postings in the configuration editor

  • Add CashRegister and default folder in Configuration-editor: root/currentProperty/propData (Notice: "default" - low case letters !!!)
  • Add protelIO value with Content type: int and set the default cash register number in Data text box

5

  • Configure a KSK template for invoices

ConfigurationEditor/Root/currentProperty/propData/Interfaces/IO/Gateway/HTNG_HotelCheckOutNotifRS/TemplateNameInvoice

  • Create HTNG_HotelCheckOutNotifRS/TemplateNameInvoice folder if they do not exist
  • The Content-type for HTNG_HotelCheckOutNotifRS is null
  • The content-type for TemplateNameInvoice is String
  • The Data is the Name of the Template. Important: case sensitive!

  • The Invoice Template needs to be configured in the System Language

Please always  use the templates out of "Meine Textvorlagen" only.

Two way Installation

Examples

1

Kiosk Workflow overview 

Steps for Kiosk with already booked Reservation workflow:

  1. OTA_ReadRQ to Protel (via ResID or DateRange)
  2. HTNG_HotelFolioRQ to Protel in order to check folio’s forcasted balance
  3. OTA_HotelAvailRQ to Protel in order to get the RoomAvailabilities
  4. IO_ResModifyNotifRQ to Protel in order to assign a Room if it is not already assigned
  5. HTNG_ChargePostingRQ to Protel in order to prepay the unpaid forecast revenue
  6. IO_ResModifyNotifRQ to Protel in order to adding related traveler
  7. HTNG_HotelCheckInNotifRQ to Protel


Steps for WalkIn workflow:

  1. OTA_HotelAvailRQ to Protel in order to get the RoomAvailabilities
  2. OTA_HotelResNotifRQ to Protel 
  3. HTNG_HotelFolioRQ to Protel in order to check folio’s forcasted balance
  4. HTNG_HotelCheckInNotifRQ to Protel
  5. HTNG_ChargePostingRQ to Protel in order to prepay the unpaid forecast revenue
  6. HTNG_HotelCheckInNotifRQ to Protel

Steps for CheckOut workflow:

  1. IO_SystemStatusRQ
  2. OTA_ReadRQ to Protel (via ResID or DateRange)
  3. HTNG_HotelFolioRQ to Protel
  4. HTNG_ChargePostingRQ to Protel (invoice balance =0)
  5. HTNG_HotelCheckOutNotifRQ to Protel
