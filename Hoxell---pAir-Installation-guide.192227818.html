Special configurations:

Set a default cash register no. for Hoxell charge postings

This should be clarified in advance with the customer in case that there has to be a new, specific cash register to be set up. 

In order to register the incoming Charge Postings with a specific cash register number in pAir, the below configuration is needed to define and map a default cash register to tag on the incoming post Revenues through the HTNG_ChargePostingRQ.

  1. Define a default cash register in pAir

System data / Financial Configuration/ Cash registers
Create one if there is not one arranged for the Integration postings

2. Add an xsetup that is used to define the default cash register for all IO-related postings in the configuration editor

  • Add CashRegister and default folder in Configuration/currentProperty/propData (Notice: "default" - low case letters !!!)
  • Add protelIO value with Content type: int and set the default cash register number in Data text box

 

One way Installation


1

Create a test reservation on the current business date & provide the reservation number to your integration partner

  • make a modification and finally cancellation of your test reservation.
    Before cancelling ask  the customer to check also the 2-way messages that is mentioned below
    in the  '2-way Installation' paragraph
  • Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


2

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ)
    for the next     5 years


BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


3

Data Validation Report - DQE

  • after checking if the bulk is successfully finished, provide the integration partner with the report
  • The Integration partner will process the DQE and send you a report afterward
  • The report is to be found in the tab Reporting of the front office in pAir.

 

Two way Installation

Examples

1

Test an incoming System Data Request→ (IO_SystemData)
  • Check if the IO_SystemDataRS has been sent back without any errors
  • The Integration partner should check if the TAA's used  in the HTNG_ChargePostingRQ are in the RS message
    (the Integration partner should check with the hotel which TAA should be used for that)


2

Test a modification on Guests data (First Name) → (OTA_ProfileModifyRQ)

Check-in the reservation from the 1-way test installation test or  use one that the customer  has chosen

  • The integration partner should  add a '1' at the end of the first name of the Guest profile
  • Check if the OTA_ProfileModifyRS has been sent back without any errors
  • The integration partner has to remove this "1" at the end of the first name of the Guest profile


3

Test an incoming posting → (HTNG_ChargePostingRQ)

Check-in the reservation from the 1-wy test installation test  or  use one that the customer  has chosen

  • The integration partner posts an amount on the folio (i.e. -10 Euro)
  • Open the folio in the PMS and check if the amount was posted correctly
  • In order to void the posting a reversal should be posted


4

Test an incoming Folio request → (HTNG_HotelFolioRQ)

  • provide the ResID of the test reservation 
  • The integration partner sends the HTNG_HotelFolioRQ request
  • Check if the RQ contains the correct ResID and if the HTNG_HotelFolioRS has been sent back without any errors


5

Test an incoming Room Status change request → (HTNG_HotelRoomStatusUpdateNotifRQ)

  • Provide a clear room number and ask from Integration partner to change it to Dirty
  • Check if the HTNG_HotelRoomStatusUpdateNotifRS has been sent back without any errors
  • Ask from Integration partner to change it back to Clean


6

Update the developer portal with the information about the installed customer
