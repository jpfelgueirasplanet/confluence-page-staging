General information

In the instance configuration dialogs, you will see options having different colors.

Option colored in black

These options are valid / usable for both PMS connections, OnPremise SPE/MPE and protel Air

Option colored in blue

Options colored in blue are only valid / usable for connections to OnPremise SPE/MPE. For connections to protel Air, they do not have any relevance and shouldn't be touched.

Option colored on green

Options colored in green are only valid / usable for connections to protel Air. For connections to OnPremise SPE/MPE, they do not have any relevance and shouldn't be touched.

Special Tab

Timeout

By default, the timeout is configured on 100 secs, it defines the amount of time, the IDS is waiting for a response besides the vendor. Please clarify with the vendor, whether this value can be reduced or not. If the vendor doesn't reply on our messages, then the IDS will wait 100 secs on each request. 30 requests will then have an export time of 3000 secs, 50 mins. This also means that new changes that were made during this time, are not export directly. In worst case, this might cause compensation requests besides the customer.

If you set it to "999" the IDS will not wait for the reply from the vendor. But this also means, that failed exports are not recognized and the affected data is never exported again, until it is changed again during a sync.

Automated generated matchings

It is possible to configure the instance to create its matchings automatically. 

Currencies

The instance will read all the configured currencies from the database and will create matchings by using the ISO code as the matching value. These matchings are required if the customer is working with multi currency (e. g. reservation import in US$, but reservation creation on a rate using EUR).

Nationalities

Deprecated option for OnPremise

The instance will read all the configured nationalities from the database (table natcode) and will create matchings by using the ISO code as the matching value.  By default this option is not needed, because the instance will automatically check the received nationality against an internal table and the nationality table in protel.

Transaction accounts

The instance will read all the configured transaction accounts from the database and will create matchings by using the transaction account number as the matching value. The instance will only use the transaction accounts, that are visible for the configured property.

Pickup codes

The instance will read all the configured pickup codes from the database and will create matchings by using internal reference together with the short name of the pickup code as the matching value. The instance will only use the pickup codes, that are visible for the configured property.

Allotments

The instance will read all allotments from the database and will create matchings by using internal reference as the matching value.

Room types

The instance will read all the configured room types and will create matchings by using the short name of the room type (kat.kat) as the matching value. 

The following requirements must be fulfilled:

  • The room type must be visible for the configured property
  • If the room type is added to a room type group, then this room type group must also be visible for the property
  • The option "Show in Internet" in the room type must be activated

Rate codes

The instance will read all the configured rate codes and will create matchings by using the short name of the rate code (ptypgrp.gruppe) as the matching value. 

The following requirements must be fulfilled:

  • The rate code must be visible for the configured property
  • The option "Show in Internet" in the rate code must be activated
  • If the tickbox "Exclude rates that are marked as 'special rates'" is activated, then the rate code is not allowed to be configured as a special rate

Packages

This option is not relevant for I/O and should always be deactivated

MPE Hotel

The instance will create a matching for the configured property. The matching value can be configured in the selection box. It is possible to use

  • MPEHotel ID - The MPE hotel ID as it is configured in the PMS
  • Property Scope ID - In elder interface also called "PHR property ID"
  • Cloud ID - the protel Cloud ID. This must then be configured in the upper right corner of the Common tab.

Activation of PCI Proxy is not required

Export Obsolete Data

If a Room Type is removed from a rate code (using the IDS matchings sheet), there is no proper OTA way how to communicate the change other than inform the partners and trigger a full overlay manually to all connected interfaces. To avoid this, the following checkbox can be enabled on Export/Misc.: "Invalidation of obsolete data" 

If this button is enabled, once a Room Type is removed from the matchings for IDS, the following will happen:

  • OTA_HotelAvailNotifRQ, delta message sent out for the dropped Room Type showing the same as closed to stay /for the period of the IDS export/

    Remove RT
    <AvailStatusMessage>
    <StatusApplicationControl Start="2021-08-13" End="2022-08-12" InvTypeCode="KING" RatePlanCode="CORP"/>
    <RestrictionStatus Restriction="Master" Status="Close"/>
    </AvailStatusMessage>

  • OTA_HotelInvCountNotifRQ, delta message sent out for the dropped Room Type showing the same with ) availability and marking all left rooms of this Room Type as "definite sold" /for the period of the IDS export/

    Remove RT
    <Inventory>

    <InvCount CountType="2" Count="0"/>

    </Inventory>

Final testing of the data export

The IDS development team ask you to perform the following steps to setup a working data export.

  1. Set the Export handling option to "Synchronize with PMS only"
  2. Configure the instance to export data for 14 days
  3. Set the instance to "Running"
  4. Initiate a data synchronization by selecting "Start data export" in the context menu of the instance
  5. After the synchronization is done, enter the Configuration and switch to the tab "Data export"
  6. Verify on this tab whether the data that is shown is correct or not. Her you can also verify whether there is data existing for all matched ans requested room type / rate combination
  7. If there are discrepancies, please correct the configuration, matchings and/or PMS settings
  8. If everything works as expected, set the Export handling option to "Synchronize with PMS and export changes"
  9. Start another export via the context menu of the instance
  10. Monitor the data export. 
  11. If the "Last action return message" for the export action is NOT "Operation performed successfully", then you should verify the configuration and check the configuration of the connected vendor. Check for possibly sent out notification emails
  12. If the "Last action return message" for the export action is "Operation performed successfully", then you can step into the configuration again and change the export date range to the requested one.
  13. Start another export via the context menu of the instance
  14. Monitor the data export.  
  15. If the export action takes too long, please contact the vendor and ensure that he is responding "nearly just in time".
  16. If also this succeeds, then you can inform the customer that the data export is up and running
