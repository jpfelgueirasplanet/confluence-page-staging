One way Installation


1

PWS Configuration

  • enable LiveMode and activate ToggleTrigger for required message types for 1-way connection

Message type

LiveMode

ToggleTrigger

IO_SystemDataRS

OTA_HotelInvCountNotifRQ

Inform Hotel-Spider, that they can pull the system data from the PMS.  Hotel-Spider creates products (combination of room types and rate codes) from all the information pulled in the system data.  If not all combinations are valid, we need to inform Hotel-Spider, which combinations will be used by the hotel.

If the customer has an existing connection to Hotel-Spider, please check the following before continuing:

  1. Hotel-Spider stops export of ARI to external Channels
  2. Old IDS connection to Hotel-Spider deactivated

Once this has been done, Hotel-Spider can switch the connection on their end and pull the system data.

2

Trigger export of

  • availabilities sent with OTA_HotelInvCountNotifRQ (via Trigger URL)
  • rates sent with OTA_HotelRatePlanNotifRQ (via IDS → start full export)
  • rate restrictions with OTA_HotelAvailNotifRQ (via IDS → start full export)

Verify with Hotel-Spider and the customer that the availability, rates and restrictions are showing correctly on the Hotel-Spider portal.

Once availability, rates and restrictions have been confirmed by the customer as being correct, Hotel-Spider can switch on the export to external channels again.

IMPORTANT

  1. Maximum export (ARI) in HotelSpider is 2 years - the configured export period needs to be checked with HotelSpider Team.  The end date for the export data is fixed.  The end date is extended by HotelSpider and the customer is informed.  The customer must then inform protel Support to push the data for the newly opened period.
  2. Restrictions:  We need to check with the Hotel-Spider team how the MinLos and ForwardMinStay is set up in Hotel Spider - they can be configured separately in HS or one can be cloned from the other.  (If the setting is such that ForwardMinStay (Stay through) is to be cloned from MinLOS - then the Hotel must send all restrictions as MinLOS (on Arrival) - in HotelSpider  the ForwardMinStay will be filled and the MinLos is not available for the customer (and vice versa).  This also works with MaxLOS and ForwardMaxStay.
