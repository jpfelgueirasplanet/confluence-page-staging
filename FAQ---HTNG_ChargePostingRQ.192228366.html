Post Tip charge

The hotel wants to allow the Integration Partner to post as well Tip charge

pAir necessary setup

  1. Create Cash Register and set a default one for IO: Settings/Financial Configuration/Cash Registers



  2. Create a method of payment to use for the Tip: Settings/Financial Configuration/Payment Methods


    - set the MOP to Pay-in/Pay-out

  3. Create TAA code for the Tip charge: Settings/Financial Configuration/Items


    - The item should be marked as PAID OUT
    - Add the configured “Pay-in/Pay-out” payment method

HTNG_ChargePostingSamples and evaluating the result

Post the tip always with SubTypeID="Revenue" and positive @Amount 

  1. Post Tip using reservation ID:

    Tip to reservation ID
    <ns3:RevenueDetail SubTypeID="Revenue" Description="tip external 2" PMSRevenueCode="998" Amount="15.00" CurrencyCode="EUR">
    <ns3:Account Type="14" ID_Context="protelIO" ID="5372"/>
    </ns3:RevenueDetail>

  2. Post Tip using folio ID:

    Tip to folio ID
    <ns3:RevenueDetail SubTypeID="Revenue" Description="tip external 2" PMSRevenueCode="998" Amount="15.00" CurrencyCode="EUR">
    <ns3:Account Type="16" ID_Context="protelIO" ID="7865"/>
    </ns3:RevenueDetail>

  3. Post Tip using profile ID:

    Tip to profile ID
    <ns3:RevenueDetail SubTypeID="Revenue" Description="tip external 2" PMSRevenueCode="998" Amount="15.00" CurrencyCode="EUR">
    <ns3:Account Type="21" ID_Context="protelIO" ID="6775"/>
    </ns3:RevenueDetail>

As a result, the cash register balance will reduce with the amount of the posted Tip: PMS/Cashiering/Cash Register Closure (login to the resp cash register setup!)

Usage of Folio Windows in onPrem

During inbound HTNG_ChargePostingRQ it is possible that reservation folio windows would not be enough - or not used in an optimized way. For this purpose - the following two scenarios would give further information on how integrations partners should develop the approach per use case

  1. Use Case 1 - Posting a charge from vendor app

    If the guest wishes to post a room charge using a reservation ID then the following will happen
    1. If there is a folio open then the charge will be posted to the guest folio as defined in the message
    2. If there is no folio open then the PMS will respond with no Folio available, hence posting of room charge is not possible, leaving the guest with the ability to pay for the charge on the app

  2. Use Case 2 - Posting a charge plus the complete payment

    1. In the case of a guest posting a charge and the payment, the Profile ID should be used to create a passerby account, post the charge + payment, close the passerby, and generate the invoice. 
    2. The charge and payment, in this case, will have no relation to the reservation and will only be linked to the profile. 

e.g.

Call passerby invoice
<ns3:RevenueDetail SubTypeID="Revenue" Description="Breakfast" PMSRevenueCode="998" Amount="15.00" CurrencyCode="EUR">
<ns3:Account Type="21" ID_Context="protelIO" ID="6775"/>
</ns3:RevenueDetail>
<ns3:RevenueDetail SubTypeID="Payment" Description="tip external 2" PMSRevenueCode="3" Amount="-15.00" CurrencyCode="EUR">
<ns3:Account Type="21" ID_Context="protelIO" ID="6775"/>
</ns3:RevenueDetail>

PREM-2573

UnknownError in HTNG_ChargePostingRS after PostPayment 

If the default invoice contains a grid with ReplacementCode, then some errors send back in the HTNG_ChargePostingRS however the payment is done.
In this case, all other processes (PDF creation, fiscalization, payment, etc.) go well but only some errors send back.

There is a Prem development ticket open to solve this for the future, but as a workaround removing the Grids containing the ReplacementCode would solve the issue.

</Errors>

IOTEC-2013

No TAA List in MOP setting cause "No reservation found"

While configuring the MOP for fiscalization, TAA List should always be set in a correct way, otherwise, the fiscalization, KSV, and IO request won't function correctly.

OnPrem


Configured hed file for the invoice template is not getting considered correctly

While configuring the MOP for fiscalization, we can set the default FiscalCode related to the MOP. In the FiscalCode also there is a default Form configuration. Then in order to have the correct form while Fiscalizing with MOP, you need to check what is the Default FiscalCode and In the Default FiscalCode which form (HED, DOCX, ...) is attached.
Please note if you are using the IO and your form is DOC file type, then the server which runs Comserver/Taskserver connnected to pWS should have MS Office installed.
