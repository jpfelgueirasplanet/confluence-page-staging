OneClick- installation

For oaky installations, please always use this way of I/O -  installation. 

Please note:
OneClick- installation is only replacing the installation in BOB. All necessary pre-setups are still requested as usual. 

Please log in to the protel store: https://store.protel.io/#/ and search for Oaky in the search bar.


Make sure that you selected the hotel that is meant to get the product. Please compare the CloudID just to make sure. 

  • Click on Get started to start the installation process

  • Installation Wizard starts with step 1 - click on Next
    • Installation Wizard steps are configurable in each Service on Developer Portal
      (order of steps could be changed)


  • Accept the protel Terms of Service and click Next

  • Agree on this data being sent to the integration partner to start the registration process

  • Click on Finish installation

One way Installation


1

PWS Configuration

  • enable LiveMode and activate ToggleTrigger for required message types for 1-way connection

Message type

LiveMode

ToggleTrigger

OTA_HotelResNotifRQ

OTA_HotelInvCountNotifRQ

2

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


3

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ)
    all future reservations starting from the installation date

BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


4

Trigger export of

  • availabilities sent with OTA_HotelInvCountNotifRQ for the next 1 year via Trigger URL

on prem | Triggering messages: API calls (PWS)

5

Data Validation Report - DQE

  • after checking if the bulk successfully finished, provide the integration partner the report
  • The Integration partner will process the DQE and send you a report afterwards
  • The report is to be found in the tab Add-Ons of the front office

https://confluence.protel.net/display/PIODOC/on+prem+%7C+DQE+CKit+Variations

2

Test an incoming room upgrade /trace creation
(IO_ResModifyNotifRQ)

Note

the room type change and adding trace to reservations are done on the same step

  • Create a test reservation (the reservation should have an email attached)  and  provide the reservation number to your integration partner
  • Ask Oaky to do a room upgrade
  • please check if the IO_ResModifyNotifRS has been sent back without any errors
  • Please check the change in the reservation room type, fixed charges and trace
  • Cancel the reservation afterwards
