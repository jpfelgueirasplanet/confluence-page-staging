Which protel Version?

The live version with the required functionality has been released in protel 2018.20.158

What does this functionality do?

The WrittenConfInst functionality is a section within the OTA_HotelResNotifRQ message that allows the user to advise the third party CRM system to send out a confirmation letter

To see the use of this functionality, please see Using the WrittenConfInst Functionality below

Setup of the Letter Types

Setup of the Navigator Response Table and name of the CRM

Protel has a table that enables the recording of the request for a letter to be sent. This table is called navigator_response

As this functionality is available for all CRM's it is possible to name the CRM you are using via the xsetup

crm, name, 'Name of the system' default is CRM

Using the WrittenConfInst functionality

As the xsetup 

CRM, navigator, 1

has been added there is a new option in the invoice list of the Navigator

CRM Vorlage Versenden (Sending CRM Template)

When a user wishes to send a confirmation letter they select this option and they are presented with the following screen

From Vorlagentyp (Letter Type) you can select one of the three options you have entered

Then you can select the person you wish to send it to

A list of profiles linked to the reservation will be provided.

If the profile is not part of the reservation, you can select 'Other'

This will active andere (Others) field where you can manually type in the email address

The last field is for any remarks that you wish to be included in the letter

The user can then select yes, or No. 

If No, is selected then we will record a negative request in the Navigator_Response table (Send = 0). This enable the ability to check if a letter was requested if it was not sent out.

If the user selects a profile with no email address configured then a warning message will be shown

The user can then select No, and go to the profile and add the email address

Now when you look at the table you can see that a request has been made to send a confirmation letter send=1 to the profile

Managing the Data in the Navigator_Response table

There is going to a large amount of data held within this table.

There is an xsetting which can be configured in conjunction with the client which enables the clear down of the table after a certain number of days

crm, navigator_response_table_hold_for_x_days, X (Default is 365 days)

For good housekeeping I would suggest holding the data for no more than 30 days.
