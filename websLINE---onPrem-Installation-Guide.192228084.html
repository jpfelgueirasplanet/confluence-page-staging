Installation Steps

 

General steps

Examples

1

websLINE uses the same connection details for each new installation. EP; basic User/Pass are set as default values in the manifest

Please reach out to websLINE via slack or with an email to systems@websline.com to provide them the HotelID (protel Cloud ID) and token of the new installation 


2

The following System Data exports of the Customer PMS must be provided to websLINE before sending any API messages to them:

  • Room Type Codes (SD/Reservations/Room Types)
  • Rate Plan Codes (SD/Rates/Rate Codes)
  • Transaction Account Codes (SD/Bookkeeping/Revenue and Payments/Transaction Accounts)
  • Articles and corresponding TAA (SD/Bookkeeping/Revenue and Payments/Articles)
  • Market Codes (SD/Reservations/Reservation Codes/Market Codes)
  • Source of Business Codes (SD/Reservations/Reservation Codes/Source codes) 

To export those reports, go to SD/ respective menu, right click on the header of the report, choose "HTML" and save the file in .mhtml format. Once done - delete the locally stored copies of data.

 The import/mapping of the PMS SD codes on websLINE end is semi-automated. It would take the partner approximately 30 minutes to complete the task and have the codes usable with their booking engine.


3

 Confirm with the Customer about the interface:

  • Which Room Types should be mapped for the booking engine (to be used on inbound reservations and rate updates)?
  • Which Rate Codes should be mapped for the booking engine (to be used for inbound reservations and rate updates)?
  • How should the deposit payment be posted - as revenue or as payment and to which TAA code/Payment Method?
  • websLINE posts deposit payments. All payments done will get posted to the cashier no. - which cash register ID should be used for the payment postings?
    Please make sure that the cashier no. really exists
  • Which articles will be applied on the booking engine (to be used with inbound reservations - fixed charges)?
  • What children age ranges should be configured in IDS (e.g. "Child 1" corresponds to what child age range)?

 Confirm with the Partner about the needed DCS matchings (RoomType / RateCode / Market Codes / Source of Business)

 If the Customer connection was initiated from websLINE's end, the list has already been aligned between them - thus protel support can obtain the codes to map from the partner. Else - if it is a protel Customer - obtain per one of the needed codes from the Customer in advance to enable the installation testing.

Configuring an IDS Instance for I/O: How to configure a protel I/O IDS

  • ESB server connection
  • Add a new instance
  • Configure the Comserver URL
  • Add the URL for Export
  • Add the URL for Import
  • Add the SQL database
  • Configure the Email Notification
     If you use reservation import with "Split reservation and assign room", make sure to add an email notification if room assignment fails ("Reservation split"-template) - see print screen
  • Add the Token and Module Name in the Special Tab
  • Export  Set export span for 1,5 years (18 months) in future, every 5 minutes
    • Rate Values - OTA_HotelRatePlanNotifRQ
      • Configure the Children - To be confirmed with partner/Customer
    • Restriction Values - OTA_HotelAvailNotifRQ
      •  Only enable the restriction export - no further configuration required
    • Deactivate the export for OTA_HotelInvCountNotifRQ from IDS (the message is sent from pWS)
  • Import
    • Mode MUST be set to PULL and PUSH
    • Import - Reservation import - Text processing - Choose text assignment "Memo" on room level and target definition "Join notes with Memo field" (see print screen)
    • In menu Import/Reservation Import/:
      • Set the reservation import time to 1 minute
      • In sub-tab Color/Room assignment, confirm with customer which default setting to use, e.g.: "Split reservation and assign room(s)"
  • Create DCS mapping 
    • mpehotel
    • RoomType
    • RateCode
    • Market Codes
    • Source of Business

***Please double check with Websline how the RateCode will be sent inbound on the 2 way Reservation test. Websline might use the Protel SD ratecode short name, or use their own values (i.e. a specific number) for some special ratecodes. Fill the matching value accordingly***

 Note: If automated matchings are used in the IDS configuration, then the IDS will create the matching for the respective category itself. But this will disable the option to do any adjustments on the auto-matched list. Thus, it is recommended to do the matchings manually

 For the purpose of the installation test, it is enough to do the mapping for a few room/rate/market/etc combinations. Then instruct the Customer how to manage the mappings on their own

  

     

 

 

One way Installation


1

PWS Configuration

  • enable LiveMode for required message types for 1-way connection

LiveMode

Message type

LiveMode

OTA_HotelResNotifRS

OTA_HotelAvailNotifRQ

OTA_HotelRatePlanNotifRQ

2

a) Using manual trigger link, send OTA_HotelInvCountNotifRQ for 534 days in future

b) Verify in Kibana that the outgoing message is answered with OTA_HotelInvCountNotifRQ message from the Integration system

on prem | Triggering messages: API calls (PWS):

http://127.0.0.1:8000/ota/hotelinvcountnotif/{mpe}/send/534

3

a) Using manual trigger link, send IO_StatsNotifRQ for 45 days in past and 1 year in future

b) Verify in Kibana that the outgoing message is answered with IO_StatsNotifRS message from the Integration system

on prem | Triggering messages: API calls (PWS):

Future: http://127.0.0.1:8000/io/statsnotif/send/{mpe}/3/365

Past: http://127.0.0.1:8000/io/statsnotif/send/{mpe}/4/45

4

a) Using IDS/Start full data export for 1,5 years (18 months) in future for Rates and Restrictions

 Could be adjustable per Customer

b) Verify in Kibana that the outgoing messages are answered with OTA_HotelRatePlanNotifRS and OTA_HotelAvailNotifRS respectivly from the Integration system

Right-click on the IDS instance line / Start full data export:

5

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


6

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ)
    for the last 1 year and everything in future

BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


7

Data Validation Report - DQE

DQE is evaluated in terms of Gross/Brutto values

  • after checking if the bulk successfully finished, provide the integration partner the DQE report
  • The Integration partner will process the DQE and send you a report afterwards
  • The report is to be found in the tab Add-Ons of the front office
  • The partner will need approx 30 minutes to cross-check the data with the booking engine and reply to the protel support team if data synced successfully - or - if further research is needed.

 

Two way Installation

Examples

1

Test an incoming Reservation 

(OTA_HotelResNotifRQ) through IDS

a) Ask the partner to create a test reservation in their booking engine for at least 1 year in future

b) Verify in Kibana that the OTA_HotelResNotifRS from IDS is present with <Success> and no warning/error.

c) Verify in Kibana that the OTA_HotelResNotifRQ from pws is present and responded with OTA_HotelResNotifRS with <Success> and no warning/error

d) Verify in the PMS UI that the reservation details are captured properly

e) Ask the partner to cancel the reservation from step (a) and repeat the same (b-d) steps afterwards

 websLINE has deactivated modifications on reservations from the booking engine

 If the reservation is a multi-room one - it is split in the Booking engine itself and sent to protel in separate OTA_HotelResNotifRQs with unique ResIDs

 No CC data import is supported
