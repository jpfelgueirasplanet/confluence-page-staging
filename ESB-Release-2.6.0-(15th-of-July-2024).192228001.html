IO-7435

Grafana | Research counts mismatch

Match history data between Kibana and Grafana query


IO-8153

ESB-IN | The alias io.protel.pms not recognized

The io.protel.pms alias is identified as any of the protel PMS IO services: io.protel.air or io.protel.onpremise and can be used as generic replacement of target destination.


IO-8203

ESB-IN | Async-sync large messages are missing ACK handling

If CommunicationType is async-sync and message size exceeds > 100 KB then the ACK handling is currently wrong. The ACK from the sync IFC should be skipped and the RS should be accepted as such.


IO-8262

ESB-IN | CommunicationType is misinterpret if noRS: "true"

If a message type for a service is marked to receive no RS messages, there is only one service installed on a customer and the workflow is async - then the ACK handling is incorrect.


IO-8282

PULL Proxy Service | Wrong transformation for OxiOTAHotelInvCountNotifRQFilter

For the new PULL Proxy service the transformation is applied differently. The end result of the transformation should match completely to the one applied on the old OXI Proxy 


IO-7688

ESB Reporting | Create Dead EP report per vendor

New dashboard created only for DeadEP monitoring in Grafana: https://grafana.protel.io/d/Zctgz6QIz/esb-visualization-of-dead-ep-markers?orgId=1&from=now-1h&to=now


IO-8134

Grafana | Distribution partners dashboard

New Grafana Dashboard created as subset of the dev one - to serve IO protel and partner support teams for PROD data flow monitoring: https://grafana.protel.io/d/ju5HzdQIk/distribution-partners-dashboard-prod?orgId=1&refresh=1m&from=now-15m&to=now


IO-8199

ESB-IN | Change Token and HotelID validation

Hotel ID is validated against the token used for all messages that reach the ESB-IN service. The Hotel ID is taken from a specific xpath depending on the message structure/logic.


IO-8204

ESB FPS/TS/OUT/OXI | Reduce logging of retries

Reduce logging of error retries by showing only the start and end retry per error. All other retries will be hidden to reduce logging.


IO-8205

ESB FPS/TS/OUT/OXI | Reduce number of retries

Reduce number of retries by increasing the period between the retries. The final timespan of the retries remains the same - e.g. 24 h for timeouts.


IO-8208

ESB LB-EXT | Migration to ECS

Part of the ESB microservice upgrade project - moving from Opsworks to ECS due to EOL


IO-8210

ESB LB-INT | Migration to ECS

Part of the ESB microservice upgrade project - moving from Opsworks to ECS due to EOL


IO-8229

BTS | Prevent multiple jobs for same parameters

Prevent multiple bulk jobs for the same destination to run at the same time; If the Bulk TS queue for a destination exceeds 10K msgs - then a bulk job would wait and start only after the count is dropped.


IO-8236

OpenSearch | Anomaly detection

Research options for automatic anomaly detection


IO-8247

ESB-OXI | Generic Error Response & Logging

Extend the content of the ESB Generic RS message - to facilitate support/customer research and follow up on issues with data transfer. The content of the error will now include: HotelID, ModuleName, DestinationServiceName, RequestMessageName. Applied on OXI messages.


IO-8248

ESB-TS | Generic Error Response & Logging

Extend the content of the ESB Generic RS message - to facilitate support/customer research and follow up on issues with data transfer. The content of the error will now include: HotelID, ModuleName, DestinationServiceName, RequestMessageName. Applied on Throttling messages.


IO-8252

PULL Proxy Service | Isolate OXI Proxy from PULL Proxy flow

PULL Proxy and OXI Proxy service are meant to work the same and in parallel for GHA until the latter can be permanently switched off. The switching between them is handled with an environment variable value per hotel installation.


IO-8257

BTS | Repurpose the "Delete job" button

On the Bulk screen, there is always a Delete option available. However - this button does not send a cancellation to the PWS/IPA services if used. Now the Delete button will also serve as Cancellation of the bulk.


IO-8260

OXI PROXY | Enable Deactivation

Create the possibility to deactivate a com.oracle.oxi service


IO-8261

XML Binding | WSDL Automatic generation for ESB

All schema changes require WSDL update for the resp env. This process is currently manual. We aim to automate it.


IO-8263

OXI | Handle GHA ACK

GHA ACK messages should be handled on the ESB Map2Any. This requires an update to all existing XSLTs for GHA - the process of releasing the feature will be done manually.


IO-8264

ESB-TS | Optimize error handling for HTTP4XX and 5XX

For throttling flow, if we have an error of the type HTTP4XX or HTTP5XX (excluding HTTP500), then we will no longer keep the mapping record and wait for 24h for a RS back - we will directly drop the mapping record and log a generic ESB RS for closing the workflow.


IO-8268

OXI v2 | ARI messages from io.protel.pms should be dropped

For GHA flow all ARI messages are sent by the io.protel.ids. Yet - it can be that any of the IO services io.protel.air or io.protel.onpremise also sends these messages. To avoid conflicts in data - the ARI msgs from the PMS IO services will be dropped for com.oracle.oxi destination. The filter will be part of the ESB code and not applied in the manifest.


IO-8274

PULL Proxy Service | Large message transformation flow is incomplete

Rework large messages handling with the new PULL Proxy service.


IO-8219

ESB-OUT | Valid message not recognized as part of config

It has happened that we see mismatch in a service BOB configuration and ESB handling. Now there are cache calls added to Redis config to avoid these gaps.


IO-8279

TS | Messages stuck in sending to TS queue

It was observed that messages in TS queue were blocked. AWS SQS API call has been disabled. Follow up monitoring is required to confirm the case will not occur again.
