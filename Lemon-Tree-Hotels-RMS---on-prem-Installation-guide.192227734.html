    One way Installation

    Examples

    1

    General information about the 1-way activation

    Lemon Tree Hotels Installation team will inform the Integrations Support Team / Distribution Partner about the scheduled date for the initial upload of rates. 

    Integrations Support Team / Distribution Partner need to respond back with a certain time for a call together with Lemon Tree which is then meant for doing the test  and the upload together.

    The customer will be informed only as soon as the uploads have been successfully tested together with Lemon Tree Hotels (upload itself and some Cross Checks of dates). The property will be advised to do checks on their own to guarantee the correctness of the data.

    LTH uses a DB link to retrieve rate and room type information, meaning protel has no impact on the accuracy of data. All rate updates which have been uploaded will feed their Channel Manager connection.  


    2

    Install the Daily Rates cKit 

    Installation

    1. Files with the ending ".ckit" shall be copied into the Folder: System/ckit/XML
    2. The file of the editor "dailyrateedit.exe" shall be copied into folder: System/ckit/exe
    3. The configuration is done within System Data => Options => Construction Kit.
    4. For a new installation the CKit Modul "ckit_xsetup.ckit" you need to click once on "Ckit Settings. Then simply close that window. 
    5. Was the CKit Modul installed with "PersistentForecast.ckit" you need to click once on "Persistent Forecast - Register"
    6. Click within Daily Rates on "Initilize" and on "Initialize data with 0.00".

    DailyRates.ckit:

    Option

    Description 

    Module Information

    Shows the actual version.

    Initialization

    Activates the module and creates database tables 

    Settings

    open the settings and shows all xsetup options 

    Access rights

    Define user access rights 

    Initialize Reservation data with 0.00 (This MPEHotel only)

    Sets for this specific property the rate adjustments of existing reservations to "0". Hence existing reservations are not impacted by Daily Rates. 

    Initialize Reservation data with 0.00 (All MPEHotels)

    Sets for all properties the rate adjustments of existing reservations to "0". Hence existing reservations are not impacted by Daily Rates.

    7

    Test incoming Rate update (Daily BAR by Room Type)

    (OTA_HotelRatePlanNotifRQ)

    • the customer has Daily BAR by Room Type decisions
    • the integration partner should ask the customer which rate code should be updated (only one will be sent by LTH !)
    • make sure that the rate which needs to be updated is set to yieldable
    • ask the integration partner to push some rates
    • ask the customer to check the data via FO - Add Ons - Daily Rates - Enter daily rates editor


    • protel database table is ckit_dailyrates_rates
    • will be received inside an inbound OTA_HotelRatePlanNotifRQ message
    • AdditionalGuestAmounts are also supported - those will be added to the occupancy up to 6 adults
    • Child Rates are not supported as DailyRates could not store rate values for children
