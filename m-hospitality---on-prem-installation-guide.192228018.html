Installation Steps

 

General steps

Examples

1

Request authentication details from the Integration partner for the new installation and request them to send you the authentication details on private Slack message or encrypted file by email and inform you by phone about the password.

Please provide them the HotelID as well - internally for Protel this is the CloudID 

  • Authentication method using a specific given Endpoint for each hotel customer.  The token remains the same for all customers

2

General Preparation On-Premise

IO_Pre-Setup-Checklist_Partner_en.pdf

3

Add licence of I/O Module to system data


4

Special XSetup Configuration

m-hospitality requires both net and gross values - check the following Xsetup is set correctly:
pws | htng_tax_included | 2

5

Bob installation: one for OnPremise (if not existing) and one for the Integration  

  • provide the token corresponding to the integration partner on private Slack message or encrypted file by email and inform you by phone about the password.

6

PWS Configuration

  • update to the latest release version 
  • update I/O configuration


7

Manual Correction (Support-Tool Use Case) 


 

One way Installation


1

PWS Configuration

  • enable LiveMode and activate ToggleTrigger for required message types for 1-way connection

Message type

LiveMode

ToggleTrigger

OTA_HotelResNotifRQ

OTA_HotelStayInfoNotifRQ


2

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 
PLEASE NOTE: 

m-hospitality does not support more than one room per reservation ID. So if the hotel uses the ini setting

ZPLAN

zimmerplanbezug=0

anzahlbuch=1

the reservation will be registered in m-hospitality with only 1 room.  Latest when the group is split, the reservations will be listed separately.

3

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ) - to be defined for each customer together with m-hospitality (in general 1 year past plus all future)
  • for revenue attached to profiles (OTA_HotelStayInfoNotifRQ) -  to be defined for each customer together with m-hospitality (in general 1 year past)


BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


4

Data Validation Report - DQE


  • after checking if the bulk successfully finished, provide the integration partner the list of reservations in the bulk (the reservation numbers are enough) and the total number of messages sent.

After any necessary correction (if any bulk messages have failed), send a list of the reservations in the bulk to m-hospitality and the total number of messages sent for the OTA_HotelResNotifRQ bulk upload.  Check any discrepanices reported back by the partner.

select idvalue from pws_btsmessagejob where pwsbtsjobid=XX (XX is the ID of the bts job)

Export this list to HTML and then create an Excel sheet for this or copy into a text file.

The number of messages sent can be seen in the Bulk Upload on the developer portal

5

Update the protel SMP page with the information about the installed customer

To be found here: 

Development SMP page / Development / Services / Prospect Hotels / Add
