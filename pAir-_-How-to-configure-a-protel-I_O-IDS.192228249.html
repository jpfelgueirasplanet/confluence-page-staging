    • This article is intended to guide you through the installation of a protel.I/O IDS instance, which is required for some of the I/O integrations.

    Reaching the pAir IDS server on Production

    Please open your ProtIDSStarter.exe to reach your IDS for pAir instances. 

    If your company does not have its own IDS including URL and credentials yet, please get in touch with Cloud Support and/or Unknown User (simic) to schedule an IDS training for certification.

    Adding a new instance

    • To create a new instance, open the Instance menu. There, select Add new instance.



    • Enter the CustomerID (not CloudID) into the 'MPE Hotel' field.
    • For the License tag, select ProtelIO.
    • In the 'Connected to' selection field, select the Book@Once / protel Air entry.

    General information

    In the instance configuration dialogs, you will see options with different colors.

    Options colored in black

    These options are valid / usable for both PMS connections, OnPremise SPE/MPE and protel Cloud.

    Options colored in blue

    Options colored in blue are only valid / usable for connections to OnPremise SPE/MPE. They are NOT relevant for connections to protel Cloud and should not be touched.

    Options colored on green

    Options colored in green are only valid / usable for connections to protel Cloud. They are NOT relevant for connections to OnPremise SPE/MPE and should not be touched.

    Configuring instances

    Open configuration dialog box
    Double-click on the instance to open the configuration dialog.

    Common tab

    • In the SMP Service ID field, enter the integration's manifest name via "Edit".
    • In the Property Scope ID field,enter the Scope ID from BOB of the property via "Edit".
    • In the IDS Token field,enter the IDS Token from the BOB IDS installation via "Edit".
    • In the protel Cloud ID field, enter the CloudID of the property via "Edit".



    • In the Description, enter R2 | hotel's name and the type of instance (Windsurfer etc.) to see the connected system in the overview.

    Internal connection

    First of all, it must be ensured that the instance is installed in the same cluster as the protel Cloud backline it should talk to.

    The connection to the backline is configured by two options:

    Connected to

    This is where you tell the instance which cluster to talk to.

    Value

    Description

    protel On Premise

    This item is selected if the instance will be connected to an OnPremise hotel. Therefore, it can be ignored here.

    protel Air Ireland

    The instance will be connected to the protelAir backline of the Ireland / European cluster.

    protel Air Sydney

    The instance will be connected to the protelAir backline of the Australian cluster.

    protel Air Africa

    The instance will be connected to the protelAir backline of the African cluster.

    Global PMS URL

    The URL to use depends on the cluster you are connecting the instance to.

    Cluster

    URL

    protel Air Ireland

    http://prod-pair-intern-alb-euwest1.aws.protel.net/ids-backend/ProtIDSProvider

    protel Air Sydney

    http://prod-pair-intern-alb-apsoutheast2.aws.protel.net/ids-backend/ProtIDSProvider

    protel Air Africa

    https://app-za.protel.net/ids-backend/ProtIDSProvider - This URL might change in the future.

    It is strongly recommended to test the configured values by clicking on the Test button.

    It is only allowed to continue with the configuration if the test was successful.

    URL for Export

    By default, these are set by the instance during initial setup, but please verify them.

    Endpoint: 

    https://service.protel.io/services/ProtelApiService.ProtelApiServiceHttpsSoap12Endpoint

    Export handling

    The option "Export handling" has a central role during installation and initial data export. For detailed information, see the section "Final testing of the data export".

    The option has three possible parameters.

    Parameter

    Description

    Synchronize with PMS and export changes

    This is the standard option for running instances in production. A synchronization is done and the changed data is exported.

    Synchronize with PMS only

    Here the instance only synchronizes the internal cache. The synchronized data can be verified in the tab "Export data". Data that has been changed and not yet exported is underlined.

    Synchronize with PMS and mark changes as exported

    This is an option for development. It is only allowed to use this option after clarifying its usage with the IDS Development team.



    Export Obsolete Data

    If a Room Type is removed from a rate code (using the IDS matchings sheet), there is no proper OTA way how to communicate the change other than inform the partners and manually trigger a full overlay to all connected interfaces. To avoid this, the following checkbox can be enabled on Export/Misc.: "Invalidation of obsolete data" 

    If this checkbox is enabled, once a Room Type is removed from the matchings for IDS, the following will happen:

    • OTA_HotelAvailNotifRQ, delta message sent out for the dropped Room Type showing the same as closed to stay /for the period of the IDS export/

      Remove RT
      <AvailStatusMessage>
      <StatusApplicationControl Start="2021-08-13" End="2022-08-12" InvTypeCode="KING" RatePlanCode="CORP"/>
      <RestrictionStatus Restriction="Master" Status="Close"/>
      </AvailStatusMessage>

    • OTA_HotelInvCountNotifRQ, delta message sent out for the dropped Room Type showing the same with ) availability and marking all left rooms of this Room Type as "definite sold" /for the period of the IDS export/

      Remove RT
      <Inventory>
      <StatusApplicationControl Start="2021-08-13" End="2022-08-12" InvTypeCode="KING"/>
      <InvCounts>
      <InvCount CountType="1" Count="18"/>
      <InvCount CountType="2" Count="0"/>
      <InvCount CountType="4" Count="18"/>
      <InvCount CountType="6" Count="0"/>
      </InvCounts>
      </Inventory>

    pAir Discount Groups

    As of IDS Build 6118, IDS provides a new button for pAir. Clicking the button automatically loads all discount groups as configured by the customer in the respective PMS/SD/Discount Groups menu. Regardless of the order of the discount groups in the pAir/SD menu, they would be loaded in ascending order in the IDS. Please note: pAir/IDS can use up to 3 child buckets, exported in AQC 7,8,9. Any other discount group included may not be captured correctly. Any duplicate child buckets in the pAir SD can also lead to miscalculation of rate prices.

    • In IDS, activate Common > Children Age Ranges > Enable child age range usage

    • Click on the button "Request discount groups from pAir" and save

    • The discount groups' configuration will load automatically - leave only the correct child buckets

    Email Notification

    Do NOT forget to configure the email notification! With the notification feature, the user will receive an email if something is not working as expected with the IDS.

    Global Email Notifications

    Always configure a global email! If the email changes, it is much easier to change the global email than every single instance.

    Please also always configure the following:

    • Name
    • Connection information → please check with the hotel which SMTP server should be used.
    • Authentification information → Sender email address.
    • Select the checkbox "Use this account to send global, instance-independent notifications".


    Please configure the notifications!

    To modify the notification, double click on the type. If the notification is required, please select "Yes" to enable the notification and add the email address from the "Recipients" as well as a subject for the email.

    • Please also add the HotelID in order to make it clear which hotel is sending this notification.
    • Missing reservation → Notify after XX minutes after the last reservation was imported.

    • Disc space exceeded → Notify if the disc space limit in lower than XX GB.

    • General notification

    Email Notification per Instance

    Open the instance - select the "EMail" tab

    Choose whether to use the global email address or not.

    If you don't want to use the global email address, please configure "Connection information" and "Authentification Information" in the same way you did for the global email address.

    If the global email address should be used, the "Connection information" and the "Authentification Information" is greyed out.

    Always select the checkbox "Send emails asynchron"!

    As soon as the email address is configured, please configure the notifications. To add a notification, right-click on the "Notifications" table.

    You MUST configure the following Notifications:

    • General error
    • General Notification

    You can configure the following Notifications:

    • Successful reservation import → Each successfully imported reservation will trigger a notification.
    • FAILED reservation import → Each failed imported reservation will trigger a notification.
    • Reservation split → If a block reservation is split during the import, you will get the information for the reservation IDs.
    • Missing matching → For reservation import, for example, if a Rate Code is missing in the matching.
    • Missing internal connection → No Comserver and SQL connection
    • Missing reservation → Notify XX minutes after the last reservation was imported.
    • Disk space exceeded → Notify if the disc space limit in lower than XX GB.
    • Failed profile import → Notify if the profile import failed during a reservation import.
    • Invalid rate matchings → Notify if a rate code is mapped, even if it not longer exists in protel.


    Profile Merge

    To avoid creating duplicate profiles and losing existing data, please use the following settings to merge existing guest profiles.


    Special Tab


    Timeout

    Please define with each third party vendor who uses an IDS how many seconds our IDS should wait for each request if the vendor does not reply to our messages, and enter the seconds here. 



    For example, if configured to 100 seconds: If the vendor does not reply, the IDS will wait 100 seconds for each request. 30 requests will then have an export time of 3000 secs, 50 mins...

    If you set it to "999", the IDS will not wait for the reply from the vendor.


    ESB server connection

    PROD URL: wss://wsesb.protel.io

    QA URL: wss://wsesb-test.protel.io

    Create ProtIDS path (OLD Version)

    Please log into the system data via Simba to reach the configuration dialogue.

    Simba → System data → Support → Konfigurationseditor

    Create a folder with the name ProtIDS in the configuration editor (if not yet available).

    • In the treeview (nodes), open the folder "Root / currentProperty /
      propData / Interfaces".
    • Click the Add button  located below the treeview 

    After clicking on Add , a new dialog box opens. There, you can specify the details.

    • Enter ProtIDS as the Name.

    • Leave the Content type on null.

    • At the lower right part of the window, click on the Save changes button.

    • At the lower left part of the window, click on the Commit button.

    Create subdirectory with the IDS instance ID

    Create a subdirectory in Root/currentProperty/propData/Interfaces/ProtIDS.
    The subdirectory will then be named after the IDS-Instance ID.

    • In the tree-view (nodes) open the following folder "Root / currentProperty /propData / ProtIDS".
    • Below the tree-view, click on the "Add" button  and then determine the details afterwards.
    • Enter the instance ID, next to the Name (e.g., 4851).
    • Leave the Content type on null.
    • At the lower right part of the window, click on the Save changes button.
    • At the lower left part of the window, click on the Commit button

    Create Matchings subdirectory

    Create a Matchings subdirectory below the file with the IDS instance number. 

    In the treeview (nodes), open the folder "Root / currentProperty /propData / ProtIDS / (Instance ID)".

    Click on the "Add" button located below the treeview.

    • Enter Matchings as the Name.
    • Select XMLList as Content type.
    • At the lower right part of the window, click on the Save changes button.
    • At the lower left part of the window, click on the Commit button.

    Final testing of the data export

    The IDS development team asks you to carry out the following steps to set up a working data export:

    1. Set the Export handling option to "Synchronize with PMS only".
    2. Configure the instance to export data for 14 days.
    3. Set the instance to "Running".
    4. Initiate a data synchronization by selecting "Start data export" in the context menu of the instance.
    5. After the synchronization is done, enter the Configuration and switch to the tab "Data export".
    6. Verify on this tab whether the data that is shown is correct or not. Here, you can also check whether data exists for all matched and requested room type / rate combinations.
    7. If there are discrepancies, please correct the configuration, matchings and/or PMS settings.
    8. If everything works as expected, set the Export handling option to "Synchronize with PMS and export changes".
    9. Start another export via the context menu of the instance.
    10. Monitor the data export. 
    11. If the "Last action return message" for the export action is NOT "Operation performed successfully", check the configuration and also the configuration of the connected vendor. Check for any notification emails sent.
    12. If the "Last action return message" for the export action is "Operation performed successfully", then you can step into the configuration again and change the export date range to the requested one.
    13. Start another export via the context menu of the instance.
    14. Monitor the data export.  
    15. If the export action takes too long, please contact the vendor and make sure that they are responding "nearly just in time".
    16. If this is successful as well, you can inform the customer that the data export is up and running!
