XSetup KeepBlockNumbersPast

The XSetup KeepBlockNumbersPast is a specific development for IDeaS. We cannot change the behaviour for this XSetup because otherwise the Integration with IDeaS will fail.


Expected behavior (SOLL-Zustand)
@SellLimit always reflects the initially allocated number of rooms
@NumberOfUnits in RoomTypePickUpStatus="1" - must be equal to RoomTypePickUpStatus="4"@NumberOfUnits + RoomTypePickUpStatus="5"@NumberOfUnits which should not be set to "0"
RoomTypePickUpStatus="2"@NumberOfUnits get set to "0"
RoomTypePickUpStatus="3"@NumberOfUnits get set to "0"


Sample with Xsetup (Allotment is expired)

In total 4 Rooms are assigned to this Allotment, because of this the number of @SellLimit is set to 4.

3 Reservation are created against this Allotment, because of this the number of @NumberOfUnits is set to 3 for the @RoomPickUpStatus="1".

@RoomPickUpStatus= "4", @RoomPickUpStatus="5" and @RoomPickUpStatus="13" are not set to 0.

@RoomPickUpStatus= "2" and @RoomPickUpStatus= "3" are set to 0

<RoomType RoomTypeCode="n/a">
<RoomTypeAllocations RoomTypePickUpStatus="1">
<RoomTypeAllocation Start="2017-09-08" End="2017-09-08" NumberOfUnits="3" SellLimit="4"/>
</RoomTypeAllocations>
<RoomTypeAllocations RoomTypePickUpStatus="4">
<RoomTypeAllocation Start="2017-09-08" End="2017-09-08" NumberOfUnits="2"/>
</RoomTypeAllocations>
<RoomTypeAllocations RoomTypePickUpStatus="5">
<RoomTypeAllocation Start="2017-09-08" End="2017-09-08" NumberOfUnits="1"/>
</RoomTypeAllocations>
<RoomTypeAllocations RoomTypePickUpStatus="2">
<RoomTypeAllocation Start="2017-09-08" End="2017-09-08" NumberOfUnits="0"/>
</RoomTypeAllocations>
<RoomTypeAllocations RoomTypePickUpStatus="3">
<RoomTypeAllocation Start="2017-09-08" End="2017-09-08" NumberOfUnits="0"/>
</RoomTypeAllocations>
<RoomTypeAllocations RoomTypePickUpStatus="13">
<RoomTypeAllocation Start="2017-09-08" End="2017-09-08" NumberOfUnits="3"/>
</RoomTypeAllocations>
