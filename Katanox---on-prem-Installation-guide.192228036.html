One way Installation


1

PWS Configuration

  • Activate  required message types for 1-way connection

Message type

Activated

OTA_HotelInvCountNotifRQ

IO_RatePackageNotifRQ

IO_SystemDataRS

2

Create and start bulk uploads

  • for packages (IO_RatePackageNotifRQ)
  • availabilities sent with OTA_HotelInvCountNotifRQ  via Trigger URL


BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 

for the Bulk export the days could be different per hotel. Katanox should provide us with the value. 

When activating the PWS, a message for full x days In future is sent out to the integration partner.
In the sample bellow it is about 365 days

In case of need of triggering the full export
again, you may use the Trigger url procedure on prem | Triggering messages: API calls (PWS)

WITH PWS BUILD 3.4 AND HIGHER:

http://127.0.0.1:8000/ota/hotelinvcountnotif/{MPE_hotel}/send/{NUMBER OF DAYS}


 e.g.: http://127.0.0.1:8012/ota/hotelinvcountnotif/3/send/365

3

Trigger export of

  • rates sent with OTA_HotelRatePlanNotifRQ (via IDS → start full export)
  • rate restrictions with OTA_HotelAvailNotifRQ (via IDS → start full export)
