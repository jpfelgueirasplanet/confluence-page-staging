PWS 3.4.X - 13.09.2021

IMPORTANT!

Please check prior to an update to a 3.4 version if and which availability message processing your customer is using.

1. If partner is using 3 times/day overlay, then
- activate the live mode and toggle trigger in pws_outboundmsg for OTA_HotelInvCountNotifRQ
- change the inventory calculation period if necessary (default setting is 365 days in future)

2. If partner has SCS job with PWS for OTA_HotelInvCountNotifRQ 
- activate the live mode and toggle trigger in pws_outboundmsg
- change the inventory calculation period if necessary (default setting is 365 days in future)
- remove the SCS job from protel market place
- remove the .scs installation from MIS if not used for other scs jobs for the customer

3. If partner has IDS only for OTA_HotelInvCountNotifRQ - !!! to be scheduled per integration partner since the CountType="2" is differently handled between IDS/PWS, i.e. if integration partner was using CountType="2" from IDS - this resembles CountType="3" from PWS
- activate the live mode and toggle trigger in pws_outboundmsg for OTA_HotelInvCountNotifRQ
- change the inventory calculation period if necessary (default setting is 365 days in future) to match with the IDS export period
- check for other special settings like oversell - if present in the IDS partner installation - then apply the same settings on the pws config/Inventory/Oversell xsetup
- deactivate the OTA_HotelInvCountNotifRQ export from the IDS - if no other messages are used - IDS instance and IDS installation from MIS can also be removed
