pAir | Triggering messages: API Calls

In order for each of the (protected) REST trigger endpoints to work, one has to be logged in to the respectful testing hotel in pAir on the relative environment.

General steps required for a successful triggering:

  1. Login to the pAir environment
    1. DEV:     https://dev-connect.protel.net
    2. QA:       https://qa-connect.protel.net
    3. PROD:  https://connect.protel.net OR https://sydney.protel.net
  2. Choose the hotel cloud id 
  3. Open a new tab in the browser
    (not yet working with Next Gen - pCC Log in needed!)

  4. Build the triggering link for the desired message using the pattern:

    for OTA/HTNG links: [ENVIRONMENT] / " baoBackline/resources/interfaces/htng/trigger " / [PARAMETERS* per message] ? "step="[STEP NUMBER]

    for IO links: [ENVIRONMENT] / " baoBackline/resources/interfaces/io/trigger " / [PARAMETERS per message]

    where 

    STEP - choose what to be the outcome of the trigger :

    " 1 " = The generated message will be in JSON format

    " 2 " = The generated message will be in XML format

    " 3 " = The generated message will be sent to the receiving party

    If there is no STEP parameter in the url trigger pattern, then by default the related message will be sent out to partner.

  5. Edit the triggering link as shown in the table using data that is available in pAir for the Hotel from step 2 above
  6. Execute the url --> If STEP=1 or 2, then the document will load in the browser in the selected format. If the message is to be sent out, then there will be a confirmation of the sending and the message can be found then in Kibana log.


When using the samples provided - please note that the parameters need to be replaced with a number (brackets are not needed!):

  • {id}         → needs to be replaced with e.g. 12345 or 8765
  • <NR>    → needs to be replaced with e.g. 1, 2, 3

Troubleshooting:

  1. Issue: The link does not generate the required document
    Solution 1: Errors in the pattern of the url --> re-check the link
    Solution 2: The pAir session might be over --> re-sign in to the pAir and re-run the trigger 

  2. Issue: Further authentication required
    Solution 1: You are not logged in to the correct environment
    Solution 2: You might not have access to the environment on which you want to trigger the link

  3. Issue: Error message for "Trigger disabled" for a specific message type
    Solution: The message has been disabled for the hotel. This is done from pAir/SD/ConfigEditor.
