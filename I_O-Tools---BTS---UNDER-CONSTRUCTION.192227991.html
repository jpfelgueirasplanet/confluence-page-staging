I/O Tools - BTS - UNDER CONSTRUCTION

Bulk Transfer Service

protel has created many tools to simplify and automate the Integration Process.  The Bulk transfer service is a GUI based utility for scheduling and executing initial synchronisation of data such as reservations, profiles, stay history etc. File based data transfer is not preferred and is not an option for data transfers. Protel is happy to discuss any special requirements for customers that have very large amounts of data to be transferred. These are generally larger hotels, with high occupancy.

Considerations

API based

BTS API

Real­time

Most API implementations are real­time in nature to support nearly instantaneous delivery of data

File­based APIs are not real­time but can be configured to be as frequent as every 30 minutes. The constraint is typically on extraction from the PMS. protel have coded their PMSs to be compatible and very responsive to BTS jobs

Capturing changes or overlay

API implementations mostly capture changes for future, but also support a future synchronization. Event­driven models can have gaps which will cause systems to become out of sync. The logic for triggering events needs to be rock solid.

 The downside is that there is very limited error handling and data validation for this method

History

protel implementations allow delivery of history which is a requirement for many vendors. RMS, CRM, CRS, etc

flat files are commonly used to export historical reservations. However, protel prefers to utilize BTS which has enhanced logging and re-try capabilities

Flexibility

HTNG APIs are an industry standard, reuse with other integrations is often possible.

Data that vendors would like to transfer is sometimes not readily available in an HTNG format. Protel will examine each use case and create a custom API only if needed

Troubleshooting

Protel BTS allows easy reconstruction of messages

Protel has implemented some automated processes to reconstruct messages where needed

Protel is available to engage with each partner to determine the optimal approach given our technical compatibility and vendor/customer requirements. In general protel prefers to avoid a file based approach due to lack of error handling and checking. Also some vendors or data centers restricting access to SFTP pickup locations. 

Bulk Transfer Format

  • Protel currently supports export/import SOAP/XML, JSON, CSV files and is introducing many custom formats to support the export/import from legacy systems and Interfaces. It is now also possible for Super Users to configure “on the fly”, custom subsets of protel’s data messages using protel’s proprietary Map2Any data mapping and configuration tool

  • Endpoints for the Bulk extracts are user definable. These bulk data exports can be targeted to alternative endpoints such as Data Warehouse, CRM etc for further processing

  • BTS messages can be a subset at a time using parameters such as Start Date/End Date, Name etc. date ranged for delivery of past or future messages.

  • To prevent accidental Denial of Service or performance issues, Reservations can be delivered in “batches” a batch will be typically by a configuration parameter driven number of reservations per message. Alternatively, the message flow can be throttled to prevent stressing target systems or even paused and re-started manually

  • A parameter driven pause can also be implemented between message batches
