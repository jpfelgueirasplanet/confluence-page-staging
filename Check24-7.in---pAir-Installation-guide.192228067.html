Installation Steps

 

General steps

Examples

1

Request authentication details from the Integration partner for the new installation and request them to send you the authentication details on a private Slack message or encrypted file by email and inform you by phone about the password.

Please provide them with the HotelID as well - internally for Protel this is the CloudID 

Authentication method using two different endpoints (one for outbound and one for inbound messaging) and specific User and a Password for each hotel customer.

 

2

General Preparation for pAir General (pAir)


3

Special Configuration Entries



Define User-defined field for EarlyCheckIn

(Even though C/I time has not been reached guest is allowed to check-in ( control of pre check-in functionality)

Settings/System Settings/User-defined fields/New

  • Create a UDF
  • Location should be Reservation
  • Identifier EarlyCheckIn
  • Type Checkbox

Define User-defined field for NoCharge

(Even though the guest has a still unbalanced bill he is allowed to check in)

Settings/System Settings/User-defined fields/New

  • Create a UDF
  • Location should be Reservation
  • Identifier NoCharge
  • Type Checkbox

4

A list of articles needed to post revenues during HTNG_ChargePostingRQ needs to be given to the integration partner with Item.Code.

5

A list of payment methods needed to post payment during HTNG_ChargePostingRQ needs to be given to the integration partner with Item.number

Hint:

Please have in mind that the payment methods in pAir have to be linked to -both- PMS AND CASH - so pAir is capable to interlink the payments to the cashier system you have configured in the step above. In case you refuse to do so, all your payments will be routed by pAir to cashier zero (0) automatically.

6

Request the hotel to prepare a test reservation (preferably with a room amount of max €1) in advance that would be used to test the check-out functionality during the installation date.


7

Bob installation: one for pAir (if not existing) and one for the Integration  

  • provide the token corresponding to the integration partner on private Slack message or encrypted file by email and inform them by phone about the password.


One way Installation


1

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


2

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ)
    starting on the installation date and for the next 10years

BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting on the installation date and ending with 31 of December (i.e. 18.06.2020- 31.12.2021).

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


3

Data Validation Report - DQE

  • after checking if the bulk successfully finished, provide the integration partner the report
  • The Integration partner will process the DQE and send you a report afterwards
  • The report is to be found in the tab Office (Büro) of the front office in pAir.

 

Two-way Installation

Examples

1

Test an incoming Reservation Look Up
(OTA_ReadRQ)

  • Create a test reservation without an allocated room number (arrival date needs to be current PMS date) and provide the reservation number to your integration partner
  • Request the integration partner to search for that reservation
  • Verify-in Kibana that the incoming OTA_ReadRQ is answered with the OTA_ResRetrieveRS and no errors occur


2

Test an Incoming room availability request

(OTA_HotelAvailRQ)

  • Provide the Timespan for which the available rooms will be requested (preferably starting on the current date)
  • Ask the partner to request room availability
  • Check-in Kibana that the incoming OTA_HotelAvailRQ is answered by OTA_HotelAvailRS and contains no errors.


3

Test an incoming Check In
(HTNG_HotelCheckInNotifRQ)

  • Request the partner to assign a room to the test reservation created during OTA_ReadRQ and provide the room number
  • Check whether the room has the correct status "clean" or "checked"
  • Ask the integration partner to check-in the test reservation
  • Check in the PMS if the Reservation has allocated a RoomID and checked in successfully
  • Verify in Kibana that incoming HTNG_HotelCheckInNotifRQ is answered by HTNG_HotelCheckInNotifRS and no error occurs


4

Test an incoming invoice detail request
(HTNG_HotelFolioRQ)

  • Ask the integration partner to request the folio of the previous reservation.
  • Verify in Kibana that incoming HTNG_HotelFolioRQ is answered by HTNG_HotelFolioRS and contains no errors.
  • open the invoice in the pms and compare the amount of the invoice with the amount of the integration partner


5

Test an incoming Posting
(HTNG_ChargePostingRQ)

Testing Revenue Posting

  • Request the integration partner to post a positive amount on the folio using the previous test reservation
  • The partner should use the item.code of the articles you provided during the general preparation step
  • Check that the SubTypeID="Revenue" in the HTNG_ChargePostingRQ that the integration partner is sending
  • open the folio in the PMS and check if the amount was posted correctly
  • The partner should void the posting by sending a negative amount

Testing Payment Posting

  • Request the integration partner to post a negative amount on the folio
  • The partner should use the item.number of the payment methods you provided during the general preparation step
  • Check that the SubTypeID="Payment" in the HTNG_ChargePostingRQ that the integration partner is sending
  • Check if the amount was posted correctly
  • The partner should void the posting by sending a positive amount

6

Undo the check-in and cancel the test reservation afterwards


7

Test an incoming check-out
(HTNG_HotelCheckOutNotifRQ)

  • Provide the reservation number of the test reservation prepared by the Hotel during the general preparation step
  • Request the partner to pay the invoice and then check-out the reservation
  • Check in the PMS if the Reservation has been check-out
  • Verify in Kibana that incoming HTNG_HotelCheckOutNotifRQ is answered by HTNG_HotelCheckOutNotifRS and contains no errors.


8

Test an incoming Reservation (WalkIn)
(OTA_HotelResNotifRQ)

  • Ask the partner to create a test reservation and provide the arrival and departure date of the reservation
  • verify in the PMS whether the reservation was created
  • Check in Kibana that the incoming OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and contains no errors
  • cancel the reservation afterwards


9

Update the protel SMP page with the information about the installed customer
