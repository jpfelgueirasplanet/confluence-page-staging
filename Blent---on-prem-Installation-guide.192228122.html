2

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


3

Testing Occupancy

(OTA_HotelInvCountNotifRQ)

  • trigger the OTA_HotelInvCountNotifRQ once on the server of the customer where the PWS service is running
  • Verify in Kibana that the outgoing messages OTA_HotelInvCountNotifRQ are answered with RS messages from the Integration system

Open a browser to post the URLs:
Blent requests to send 365 day in advance 
(NOTE - you have to change some details here!)

pattern:
http://IP-address(PWS):Port(PWS)/ota/hotelinvcountnotif/{MPE_hotel}/send/{START:YYYY-MM-DD}/{END:YYYY-MM-DD}
e.g.: 
with a Hotel's business date: 14/03/2020 then the dates should be
from 14/03/2020 to 13/03/2021
http://127.0.0.1:8046/ota/hotelinvcountnotif/1/send/2020-03-14/2020-03-13

WITH PWS BUILD 3.4 AND HIGHER:

http://IP-address(PWS):Port(PWS)/ota/hotelinvcountnotif/{MPE_hotel}/send/{NUMBER_OF_DAYS}

e.g.: 
to export for the next 365 days from MPE Hotel 1
http://127.0.0.1:8046/ota/hotelinvcountnotif/1/send/365

4

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ) for the last 2 years and ending with the next 1 year

    BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

    Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2018 - 31.12.2021).

    Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

    In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 

    If the Bulk is interrupted then the next option is 2 Bulks: one for the 2 y past and one for the 1y in front.



5

Configuring an IDS Instance for I/O: How to configure a protel I/O IDS 

  • Rate values and Restriction Values are necessary to be ticked in order to be exported
  • Please note that availability Values are not needed, so no tick is necessary!
  • Ask Blent and the customer which rate plan codes and room types are needed to be mapped in Protel and then include them in the Mappings. Please note that the codes are case sensitive and have to be matched exactly as they are mentioned!
  • Blent does not manage AdditionalGuestAmounts so there is no need to configure Children age ranges under the common tab


  • on Special tab about the authentication topic:

    the Token is from the ids installation but the Target is the integrations installation (2 nd screenshot)




6

Sending Rates to Blent

  • After the installation is done, start a full data Export and check in Kibana that outgoing OTA_HotelAvailNotifRQ, OTA_HotelRatePlanNotifRQ, is answered by OTA_HotelAvailNotifRS, OTA_HotelRatePlanNotifRS and no error occurs. 

7

Please inform Blent that you are done with the Bulk Uploads and sending InvCount, Rate availability and Rates!
