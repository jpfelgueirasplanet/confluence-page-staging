2

Child Rate Amount is exported with 0 values

  • There are no overlapping seasons in the Rate Plan codes configured in pAir/SystemData/Rates
  • There are no virtual rates configured over other virtual rates in pAir/SystemData/Rates
  • IDS matching list does not show any matching with error/warning
  • IDS email notifications have not alerted about an error/warning for rate configurations
  • There are no IFC messages from IDS about errors/warning for rate configurations
  • IDS configuration for Rate Amount handling - check IDS/Export/Calculation Mode drop-down selection

Depending on the setting in IDS/Export/CalculationMode drop-down selection, the IDS will behave differently:

  1. "Request all prices from protel Air" - means that the child rates will be taken from pAir. But pAir does not calculate child rates => child rates will be "0"
  2. "Use NRC (=New Rate Calculation) if all rate codes are valid, otherwise request all prices from protel Air" - means that the New Rate Calculation has been enabled to overwrite the protel Air child rate of "0". But - if there is any misconfiguration on the matches rate codes (e.g. overlapping seasons), then the fallback solution has been activated and the NRC is overwritten again by pAir calculation. This means that option 1 is then sending the child rate for ALL rate codes.
  3. "Use NRC for correct configured rates, all other will be requested from protel Air" - similar as point 2 - but the fallback is activated ONLY for the wrongly configured rates codes. The rest will remain using the NRC calculation and can still work properly

To be in all cases notified by the IDS on errors and activated fallback calculations, make sure to set up all needed notifications from IDS

To receive the correct child rate amount calculation from IDS - choose option 3, check and clear all errors in rate plan code configurations, then re-run the rate updates

 The status of the NRC calculation can be as well seen on the IDS main screen/"Connected to" column

  • "protel SPE/MPE" => option 1 is exporting
  • "protelAIR NRC" => NRC calculation is activated and exporting 
  • "protel AIR TDNRC" => NRC calculation is activated but the fallback option is exporting due to errors. Thus child rates = "0" can be seen due to using pAir calculation

3

Can not find an error during reservation Import

  • IDS instance log
  • IDS log
  • IDS configuration for reservation import
  • Kibana log

If none of the checkpoints helps, do a manual posting in the IDS configuration with the inbound sample. The IDS checks line by line the code posted and replies with the error found. Do corrections and re-posting till the reservations is accepted.

To do a manual reservation posting in IDS configuration, do:

  1. Go to IDS tab Maintenance/Post Request
  2. Paste the full load of the reservations API
  3. Click on "Send request to actual instance"
  4. Go to tab Booking History and click on "Refresh" (if needed adjust the start-end period)
  5. Click on the latest result and check the error message on top of the result screen
  6. Act according to the error message. 
  7. Repeat from step 1 till the result is highlighted in green and no errors/warnings from IDS are logged

 If the posting at step 5 is successful, then the reservation import failure is not due to IDS configuration/API details


4

Can not import customized reservation status

  • OTA_HotelResNotifRQ/../HotelReservation/TPA_Extensions schema
  • px namespace should be present: 

    xmlns:px="http://protel.io/htng/extensions"

  • DCS matching values to match with the value in TPA_Extensions/@Name
  • @ID_Context of the reservation status must only be "protelIO"

Minimum set of changes in the API:

<?xml version="1.0" encoding="utf-8"?>
<soapenv:Envelope
xmlns:px="http://protel.io/htng/extensions" ...>
<soapenv:Body>
<OTA_HotelResNotifRQ ...>
<HotelReservations>
<HotelReservation>
<TPA_Extensions>
<px:ReservationExtensions>
<px:ReservationStatus ID_Context="protelIO" Name="Deposit"/>
</px:ReservationExtensions>
</TPA_Extensions>
</HotelReservation>
</HotelReservations>
</OTA_HotelResNotifRQ>
</soapenv:Body>
</soapenv:Envelope>

5

Daily Rates with Virtual Rates and IDS setup where Max Occupancy is lower than the default for the room type

  • Daily rates installation: Daily Rate - Installation
  • PMS set up of the virtual rate Virtual Rates with Daily Rates
  • IDS configuration
  1. Daily rate module installed
  2. Base rate and virtual rate set up.
    1. For the virtual rate do NOT check the "Apply fixed amount if amount=0.00" checkbox in SD/Rates/RateCodes menu
    2. All rate amounts in the SD should be set to "0.00"
  3. Create Daily Rate split table
  4. Add DCS matching for the Daily Rate
  5. Add base and virtual rate amounts in the FO/Daily Rates module
  6. In IDS - make sure the following is set in Export/RateValues/Misc: "Export rates per occupancy up to the maximum number of adults defined in the room type occupancy definitions"
