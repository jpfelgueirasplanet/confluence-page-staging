Requirements for installation

1

Existing pAir/OnPremise installation

2

Existing Order for this interface from the customer. If the order is existing there should also be an existing JIRA ticket for this installation.

3

Existing Breeze/Infrasys IO POS license for the HTNG.exe for the customer in PMP. If you have to newly create this please keep in mind the refresh time of 15 minutes between PMP and Simba!


Installation Steps

 

General steps

Examples

1

I/O Installation

Request authentication details from the Integration partner for the new installation and request them to send you the authentication details on private Slack message or encrypted file by email and inform you by phone about the password.
Bearer Token is needed from the integration partner, but the frontend URL and the endpoint will always be the same.
Infrasys endpoint: https://hotelifc-eu-west1.shijicloud.com/ifc/htng/api/v0/AsynchResponse
Breeze endpoint: https://secure.breezepos.net/POS/?api=interfaces&method=ProtelIO

Please provide them the HotelID as well - internally for Protel this is the CloudID 

2

HTNG Interface Installation

Please be aware that the BOB surface may change from time to time - this step has been documented to remind you not to forget the HTNG 

-Select Stage in Bob PROD = live environment

-Edit the PMS base instance (the screenshot is from a PAIR installation, but for PMS instances there is no difference in regards to the scope ID compared to OnPrem) and copy the Scope ID, close that instance afterward

-Select Manifest: io.protel.ifc_htng

-You are still in = Live environment

-Select the Scope type: Property and fill in the previously copied Scope ID in here, you will have to select your customer afterward. The HotelID: CloudID of the customer, this needs to be provided to the POS vendor as HotelID

-Please note that if the installation was set up with the wrong ID there is no possibility to change this! You have to delete the wrong and create a new installation. Please be aware that this will cause a new token!

-No Environment settings are needed for this installation


 

Configuration in HTNG Interface 


1

Configure Interface <-> ESB connection

Through this connection the interface will communicate with the ESB, receiving HTNG_ReadRQ and HTNG_ChargePostingRQ messages and sending HTNG_ResRetrieveRS and HTNG_ChargePostingRS messages.

ASYNC Communication

This connection is always an ASYNC WebSocket connection which means that the interface will send an ACK.
An acknowledgment (ACK) is a signal passed between communicating processes, computers, or devices to signify acknowledgment, or receipt of the message, as part of a communications protocol after receiving a message from the ESB.

Configuration Steps:

  • Configure the pCon

  • Select “Device Settings”

  • Select “Socket Client”

  • Enter wss://wsesb.protel.io as ip/hostname and 443 as port number

  • Click “Save Changes”

pCon Configuration septs explained (German): pCon for on premise (SPE/MPE)






2

Configure Client Mapping in Interface

This is necessary to map the HotelID which will be sent by POS vendor to the correct hotel.

  • Select “Specials”

  • Select “Client Properties”

  • Add mapping Hotel = always 1 for pAir installations and MPE Hotelnumber for OnPremise, CloudID = CloudID of the customer from protel Cloud Center. Please be careful this is NOT the CustomerID from Simba! Currency ISO code should be left empty



3

Set Bearer Token in Interface

  • Select “Interface Settings”
  • Select “ESB”
  • Please ignore the data warehouse settings as this function is currently not available!
  • Enter the Bearer Token (Token HTNG) from your Interface installation, see Create Interface Installation for the Customer.
  • Click “Save Changes”



4

Configure Mappings

Mappings should be created as usual for POS XML interfaces.

5

Update the protel SMP page 

To be found here:  Development SMP page / Development / Services / Prospect Hotels / Add

Detailed explanation: Add installed property as a prospect hotel in protel market place (on prem + pAir)
