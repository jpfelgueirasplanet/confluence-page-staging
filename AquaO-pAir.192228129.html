Installation Steps

 

General steps

Examples

1

Get the authentication details from the integration partner for the new installation

Request the partner to send you the User/Password details on a private Slack message or encrypted file.

Please provide them the HotelID as well - internally for Protel this is the CloudID

2

General Preparation for pAir

https://confluence.protel.net/display/PIODOC/pAir+%7C+General+preparation

3

2-Way Optin x-setting (Marketing Allowed/Not Allowed) 

Explanation

Optin and Optout in pAir is controlled via the marketing checkbox within the guest profile shown below

In order for a third party to be able to alter this checkbox the following settings have to be in place

Allow setting of the checkbox

Marketing and Data processing can be allowed by 3rd party if: 

"Opt in to marketing" is set to true

Allow removal of the checkbox

Marketing and Data processing can be disallowed by 3rd party

"Opt out to marketing" is set to true

4

Bob installation: one for pAir (if not existing) and one for the integration. Provide the token corresponding to the integration partner on private Slack message or encrypted file.


5

Ask the customer whether specific payments need to be created or if AquaO should use existing ones. Please verify that the payment was configured properly.

6

A list of articles needed to post revenues during HTNG_ChargePostingRQ needs to be given to the integration partner with Item.Code.

7

Special configurations:

  • Set a default cash register no. for AquaO charge postings


This should be clarified in advance with the customer in case that there has to be a new, specific cash register to be set up. 

In order to register the incoming PostPayments with a specific cash register number in pAir, the below configuration is needed to define and map a default cash register to tag on the incoming post payments through the HTNG_ChargePostingRQ.

  1. Define a default cash register in pAir

System data / POS / Payment methods / POS

2. Add an xsetup that is used to define the default cash register for all IO-related postings 

  • Go to System Data - Xsetups (https://app.protel.net/settings/xsetups)
  • Interfaces - IO
  • Enter the Cash Register Number under Default IO Cash Register

    Help page: https://confluence.protel.net/x/sgDVC


Hint:

Please have in mind that the payment methods in pAir have to be linked to -both- PMS AND CASH - so pAir is capable to interlink the payments to the cashier system you have configured in the step above. In case you refuse to do so, all your payments will be routed by pAir to cashier zero (0) automatically.

8

Ask the integration partner to trigger the IO_SystemDataRQ message, to get all codes from the customr's DB.
If the customer's codes were not successfully imported with the SystemData message, please make a screenshot from pAir System Data with the short names of the room types, rate codes managed by AquaO for the customer (ask the integration partner in Slack which rates), and a screenshot of the reservation codes and send them to AquaO.








One way Installation


1

Create a test reservation in the future with a test profile (at least one year after the system date) and provide the reservation number to your integration partner

  • Make a modification of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ and OTA_ProfileCreateRQ is answered by OTA_HotelResNotifRS and OTA_ProfileCreateRS and no error occurs.
  • Provide the reservation and profile number to your integration partner, and ask if the data was transferred correctly


2

Test the outgoing profile messages
(OTA_ProfileCreateRQ/OTA_ProfileModifyRQ) 

  • Modify the profile of the test reservation created during the previous step. Verify in Kibana that the OTA_ProfileModifyRQ is answered by the a OTA_ProfileModifyRS and no errors are present. 
  • Create a test example of a company profile. Verify in Kibana that outgoing OTA_ProfileCreateRQ is answered by RS and no error occurs.
  • Modify the company profile by changing name and/or address details. Verify in Kibana that the OTA_ProfileModifyRQ is answered by the a OTA_ProfileModifyRS and no errors are present. 
  • Cancel your test reservation afterwards.



3

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ) for the past 1 year and next 1 year from the installation date (starting with 1 January and ending with 31 of December (i.e. 01.01.2020 - 31.12.2024)
  • for profiles (OTA_ProfileCreateRQ) for the past 1 year and next 1 year from the installation date (starting with 1 January and ending with 31 of December (i.e. 01.01.2020 - 31.12.2024)





Two way Installation


2

Tell the partner to create a test reservation in the future with a test profile (at least one year after the system date) and check if the reservation was correctly created in the PMS

  • After confirming the reservation, tell the partner to make a modification of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ and OTA_ProfileCreateRQ is answered by OTA_HotelResNotifRS and OTA_ProfileCreateRS and no error occurs.


3

Test an inbound modification of the profile message
(OTA_ProfileModifyRQ) 

  • Ask the partner to modify the profile of the test reservation created during the previous step. Verify in Kibana that the OTA_ProfileModifyRQ is answered by the a OTA_ProfileModifyRS and no errors are present. 
  • Let the partner create a test example of a company profile. Verify in Kibana that OTA_ProfileCreateRQ is answered by RS and no error occurs.
  • The partner should modify the company profile by changing name and/or address details. Verify in Kibana that the OTA_ProfileModifyRQ is answered by the a OTA_ProfileModifyRS and no errors are present. 


4

Test an incoming Profile LookUp 
(OTA_ReadRQ) 

  • ask the integration partner to do a profile lookup 
  • check if the OTA_ProfileReadRS is getting sent back to them with success



Test an incoming invoice detail request with AquaO

(HTNG_HotelFolioRQ) 

  • Request AquaO to send a FolioRQ message
  • Verify in Kibana that incoming HTNG_HotelFolioRQ is answered by HTNG_HotelFolioRS and contains no errors.



Test an incoming Payment posting
 (HTNG_ChargePostingRQ)

  • Request AquaO to post a charge in the folio of the test reservation
  • Verify in Kibana that incoming HTNG_ChargePostingRQ is answered by HTNG_ChargePostingRS and contains no errors
  • Check in PMS if the amount was posted correctly



Void the payment and cancel the reservation


5

Update the protel SMP page with the information about the installed customer
