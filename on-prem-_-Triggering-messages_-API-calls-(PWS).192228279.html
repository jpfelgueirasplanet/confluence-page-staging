Steps to execute the call

  1. Use a server where the PWS is installed - or - a server from the same network where the PWS is listening to
  2. Make sure the PWS service is running and the OTA message is enabled in the PWS config on "Live Mode"
  3. Open any browser
  4. Copy the link for the respective message functionality from column "Trigger Link" from the table below
  5. Make sure the IP and PORT match to the ones where the PWS is running (is listening to)
  6. Edit the link per the explanation in column "Parameters"
  7. Execute the call by pressing "Enter"
    Note: When executing the call for the first time - a login to the PWS will be required. The login follows the pattern `user=pass: apiuserX`, where X is the mpe number of the hotel
  8. If the message is meant to be sent out to the ESB, then it should be found in Kibana log. Otherwise, it will be found in folder /PWS/HTNG_Log (per details in column "Case" from the table below)

Case

OTA_HotelResNotifRQ

Create Message for a certain ReservationID in the HTNG_Log folder

TransactionAction (check Note 3):

Initiate = 3 (Default)

{ResId}= The unique ID of the reservation in the PMS



http://127.0.0.1:8000/resources/3/ota/hotelresnotif/{ResID}/{TransactionAction}/{Mpe}


e.g http://127.0.0.1:8045/resources/3/ota/hotelresnotif/75908/3/1

OTA_HotelInvBlockNotifRQ

Create message for a certain BlockID number. The message is stored in the pws/HTNG_log folder

{kontinnr} = The unique ID of the allotment created  in the PMS

EventTypes
24 = Bulk (not working)
5 = Commit (new)
6 = Modify (modification - default!)

http://localhost:8000/ota/hotelinvblocknotif/{kontinnr}/{eventtype}/(mpehotel}

e.g http://127.0.0.1:8045/ota/hotelinvblocknotif/195/6/1

ALT: http://127.0.0.1:8040/resources/3/ota/hotelinvblocknotif/73598

OTA_HotelInvCountNotifRQ


Create message for a certain period. The message is stored in the pws/HTNG_log folder

START - Starting date of the Report

END - Ending Date of the Report 

NUMBER OF DAYS - Number of days for the Report. Start date is always the CBD

http://127.0.0.1:8000/resources/3/ota/hotelinvcountnotif/{MPE_hotel}/{START:YYYY-MM-DD}/{END:YYYY-MM-DD}

 e.g.: http://127.0.0.1:8046/resources/3/ota/hotelinvcountnotif/1/2019-08-05/2019-09-09

WITH PWS BUILD 3.4 AND HIGHER:

http://127.0.0.1:8000/resources/3/ota/hotelinvcountnotif/{MPE_hotel}/{NUMBER OF DAYS}

 e.g.: http://127.0.0.1:8012/resources/3/ota/hotelinvcountnotif/3/365

Send message. The message is sent to the ESB and generated in the pws/HTNG_log folder

START - Starting date of the Report

END - Ending Date of the Report

NUMBER OF DAYS - Number of days for the Report. Start date is always the CBD

http://127.0.0.1:8000/resources/3/ota/hotelinvcountnotif/{MPE_hotel}/send/{START:YYYY-MM-DD}/{END:YYYY-MM-DD}

e.g.: http://127.0.0.1:8046/resources/3/ota/hotelinvcountnotif/1/send/2019-08-05/2019-09-09

WITH PWS BUILD 3.4 AND HIGHER:

http://127.0.0.1:8000/resources/3/ota/hotelinvcountnotif/{MPE_hotel}/send/{NUMBER OF DAYS}

 e.g.: http://127.0.0.1:8012/resources/3/ota/hotelinvcountnotif/3/send/365
