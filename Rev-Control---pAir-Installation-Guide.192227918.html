Installation Steps

 

General steps

Examples

1

Request authentication details from the Integration partner for the new installation and request them to send you the authentication details  on a private Slack message.

Please provide them the HotelID as well - internally for Protel this is the CloudID 

  • Authentication method using a specific given Endpoint, a User and a Password for each hotel customer:


2

General Preparation for pAir General (pAir)


3

Bob installation: one for pAir (if not existing) and one for the Integration  as well as one for SCS

  • provide the HotelID and token corresponding to the Integration to the integration partner (ex. screenshot token starting with 461d4d) 

 

One way Installation


1

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ) for the last 1 year and ending with the next 100 years
  • for allotments (OTA_HotelInvBlockNotifRQ) for the last year and ending with the next 100 years
  • for inventories (OTA_HotelInvCountNotifRQ) for the next 1 year (pAir | Triggering messages: API Calls)

Please set the Pause per message setting in the bulk upload to 1


Please note - RevControl can make settings in their application for the following situations:

  1. Out of order rooms:  should OOO rooms be included in availability or not. Default = include in availability
  2. Tentative (OOA) reservations: should these be deducted from availability or not.

Please ask RevControl, which settings have been done for the hotel.


2

Data Validation Report (DQE)

Please do the DQE for past dates before sending the OTA_HotelsStatsNotif.  Once the stats are sent, RevControl overwrites the revenue data from the reservations with the revenue data from the Stats message, so the data on the DQE will not match. 

3

Trigger and send OTA_HotelStatsNotifRQ for 365 days back and 365 days forward (all 6 messages)

Trigger and send IO_StatsNotifRQ for 365 days back and 365 days forward


4

a) Create 2 SCS Jobs in the SMP

OTA_HotelStatsNotifRQ | all 6 messages | 2-hourly rhythm | Past: 45 days and Current: 365 days

IO_StatsNotifRQ | both messages | 2 hourly rhythm | Past: 45 days and Current: 365 days

c) Verify in Kibana that the outgoing message (OTA_HotelStatsNotifRQ)  is answered with RS message from the Integration system




5

Create a test reservation in the future (at least one year after the system date)

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 
  • always provide the reservation number to your integration partner, so that the reservation can afterwards be cancelled


 

Two way Installation

Examples

1

Rates are updated by the Integration system

(OTA_HotelRatePlanNotifRQ)

  • RevControl requires a list of Room Type Codes (Short names) and also the minimum and maximum occupancy of each room type
  • RevControl requires the Rate Plan Code (Short) for which the rates will be imported into protel Air
  • For BW Hotels, RevControl needs the Short names of the Rate groups

In general, only one rate plan will be updated - however, on request from the hotel, more than one can be updated.
RevControl currently only support sending the same rates for all number of guest in the room (except a subtraction of a potential breakfast price)

RatePlan level:                                                                      RateGroup level:

                       

2

Test incoming Rate update

(OTA_HotelRatePlanNotifRQ)

  • make a screenshot from pAir System Data with the short names of  all rate codes from Protel (please sort them by short name before), make a screenshot and send it to the Integration partner  
  • Attention if the customer manages rates on rate group level (i.e. Best Western properties). In such case the RateGroup screenshot with all the shortnames has to be sent 

3

Test incoming Restrictions 

(OTA_HotelAvailNotifRQ)

  • Make sure that the rate which needs to be updated is set to yieldable.
  • Ask the Integration Part in the Slack channel of the integration to push a restriction for a specific day on the Test rate code and then check in the Price availability for the given date if the restriction has been set. 

RevControl only sends minimum length of Stay and minimum length of stay on arrival

4

Update the protel SMP page with the information about the installed customer
