Configuring an IDS Instance for I/O for rate and restriction export: 

pAir | How to configure a protel I/O IDS

pAir | Export Handling

pAir | How to configure a protel I/O IDS#CreateMapping
(Room types, Rate Codes, Market Code )

  • Export should be set to every 5 mins for the next 24 Months
  • Rate values and Restriction Values are necessary to be ticked in order to be exported
  • Please note that availability Values are not needed, so no tick is necessary!
  • Ask Ivvy and the customer which rate plan codes and room types and Market Codes are needed to be mapped in Protel and then include them in the Mappings. Please note that the codes are case sensitive and have to be matched exactly as they are mentioned!

Rates:

  • Receiving Child rates is not supported by iVvy (they will send them to us with the blocks)
  • iVvy can store rate values for up to 4 adults


Restrictions:

  • Only Master Close is supported (no MINLOS, CTA etc.) by iVvy



 

One way Installation


1

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


2

Create and start bulk uploads

  • For Blocks (OTA_HotelInvBlockNotifRQ)
    from current date until 31/12/2099


  • for reservations (OTA_HotelResNotifRQ)
    from current date until 31/12/2099


BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


3

Trigger export of

  • availabilities sent with OTA_HotelInvCountNotifRQ (via Trigger URL)

pAir | Triggering Messages or API calls
Trigger OTA_HotelInvCountNotifRQ for the next 2 years

4

  • rates sent with OTA_HotelRatePlanNotifRQ (via IDS → start full export)
  • rate restrictions with OTA_HotelAvailNotifRQ (via IDS → start full export)
