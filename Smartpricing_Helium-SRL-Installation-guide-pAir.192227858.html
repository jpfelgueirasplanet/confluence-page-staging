Installation Steps

 

Needed action

Examples

1

Request authentication details from the Integration partner for the new installation and request them to send you the authentication details  on a private Slack message.

Please provide them the HotelID as well - internally for Protel this is the CloudID 

  • Authentication method using the same endpoint, which is set as default and therefore not visible during installation, but a unique User and a Password for each hotel customer is required and should be requested from the integration partner

2

General Preparation for pAir General (pAir)

Smartpricing provides rate updates on a "per room basis". 
A License for "Daily Rates" is required  

3

Bob installation: one for pAir (if not existing) and one for the Integration  

  • provide the token corresponding to the Integration to the integration partner (ex. screenshot "token" ) 
  • Special Bob configuration => N/A

 

Needed action - Two way Installation

Examples

1

Make a test booking far in the future  

(OTA_HotelResNotifRQ)

  • ask the Integration partner to cancel the reservation after the verification is done.
  • Important: check if the integration needs reservations with Credit Card, and create a test reservation with CC details. Verify if the details have been correctly saved in the PMS.


2

Rates Codes and Room Types are retrieved by the partner 

(IO_SystemDataRQ)

  • make a screenshot from pAir System Data with the short names of  all rate codes from Protel (please sort them by short name before), make a screenshot and send it to the Integration partner  
  • Attention if the customer manages rates on rate group level (i.e. Best Western properties). In such case the RateGroup screenshot with all the shortnames has to be sent 

RatePlan level:                                                                      

                     

RateGroup level:

3

Test incoming Rate update

(OTA_HotelRatePlanNotifRQ)

  • the integration partner should ask the customer to create a Test rate code
  • after the Test rate code has been created, ask the integration partner to push some rates
  • Make sure that the rate which needs to be updated is set to yieldable. Ask in the channel for which day the rate has been pushed, check the date and send a screenshot from the Protel Daily Rates if the rate for the specific date is showing in the list. 

4

Data Validation Report - DQE

  • after checking if the bulk successfully finished, provide the integration partner the report
  • The Integration partner will process the DQE and send you a report afterwards
  • The report is to be found in the tab Office (Büro) of the front office in pAir.

Configuring an IDS Instance for I/O: How to configure a protel I/O IDS 

not required for this installation
