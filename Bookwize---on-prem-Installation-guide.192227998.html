One way Installation


1

PWS Configuration

  • enable LiveMode and activate ToggleTrigger for required message types for 1-way connection

Message type

LiveMode

ToggleTrigger

OTA_HotelResNotifRQ

OTA_ProfileModifyRQ

2

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


3

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ) Bookwize request all the reservations. So the request is for the last 1 year and ending with the next 2 years
    eg if the current year is 2020 then: from 01-Jan-2019 to 31-Dec-2022


BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December  (e.g. 01.01.2015 - 31.12.2022).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


4

Configuring an IDS Instance for I/O: How to configure a protel I/O IDS 

  • Export should be set to every 5 mins for the next 24 Months
  • "Availability Values", "Rate Values", "Restriction Values" are necessary to be ticked in order to be exported
  • For "Availability Values" Bookwize would like OOS Rooms to be excluded (screenshot)


  • Bookwize is using outbound OTA_HotelResNotifRQ for statistics for their BI, Outbound reservations are not accepted if Reservation's Rate is not mapped. So all configured in System Data, Rate Codes should be mapped. Please note that the codes are case sensitive and have to be matched exactly as they are mentioned!
    -Bookwize expect on Rate Plan mapping to have the "Fix Rate" option for the most of them and the "Import only" for the Rates that the calculation are on Bookwize's side (Normally ore 1 or 2 of them)
  • All Children age ranges, under the common tab, are needed to be mapped in Protel


  • on Special tab about the authentication topic:

    the Token is from the ids installation but the Target is the integrations installation (3rd screenshot)

    PLEASE Dont send Full Rate export without Full restriction export.

    Restrictions are connected with Rate/RoomTypes on Bookwize's side,
    In case of a Room type remove from rate plan with a Full only rate export
    then the Room Type is removed also on bookwize from this rate and all the restrictions are connected on that.
    If this Room Type removed by accident and then added again, following a rate only Full export (without restriction)
    then Room Type added to Bookwize's Room Plan again but the restriction on that are removed from the previous status
    So it is necessary to have a full restriction export too.




5

Sending Rates to Bookwize

  • After the installation is done, start a full data Export and check in Kibana that outgoing OTA_HotelRatePlanNotifRQ, OTA_HotelInvCountNotifRQ, OTA_HotelAvailNotifRQ are answered by  the appropriate RS and no error occurs. 

6

Please inform Bookwize that you are done with the Bulk Uploads and sending InvCount, Restrictions and Rates!
PLEASE Dont send Full Rate export without Full restriction export


7

Data Validation Report - DQE
is done by Bookwize

  • after checking if the bulk successfully finished, provide the integration partner the report (brutto)
  • The Integration partner will process the DQE and send you a report afterwards 
  • The report is to be found in the tab Add-Ons of the front office
