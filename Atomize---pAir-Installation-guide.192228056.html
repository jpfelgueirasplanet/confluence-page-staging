General steps

Examples

1

You don't need to request authentication details for the new installation

Atomize uses the same user and password as authentications for the installations. The endpoint is always the same, and it does not need to be inserted during the installation. 

EP: https://protel.integration.atomize.one/prod

General Preparation for pAir General (pAir)



3

  • provide the token corresponding to the Integration to the partner on a private Slack message or encrypted file by email and inform you by phone about the password. Please provide them the HotelID as well - internally for Protel it's the CloudID



One way Installation

Examples

1

Create a test reservation in the future (at least one year after the system date)

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 
  • always provide the reservation number to your integration partner, so that the reservation can afterwards be cancelled




2

Create an SCS installation module in Bob


  • When requesting a Token for an installation from another colleague, please don't forget to mention that an SCS installation is needed as well

Create an SCS Job in the SMP

  • OTA_HotelInvCountNotifRQ for 365 Days sending once a day at 3:00, starting with the present day


Verify in Kibana that the outgoing messages OTA_HotelInvCountNotifRQ is answered with RS message from the Integration system

3

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ) for the last 2 years and ending with 31st Dec 2099
  • for blocks (OTA_HotelInvBlockNotifRQ) for the last 2 years and ending with 31st Dec 2099
  • for packages (IO_RatePackageNotifRQ) starting today ending with 31st Dec 2099



4

BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one.

  • Activate in the IDS the rate message to be exported. Please note that availability Values and Restriction values are not needed, so no tick there is necessary!
  • Child rates are not supported by IDS pAir

7

Sending Rates to Atomize

  • After the installation is done, start a full data Export and check in Kibana that outgoing OTA_HotelRatePlanNotifRQ is answered by OTA_HotelRatePlanNotifRS and no error occurs. 

8

Please inform Atomize that your done with the Bulk Uploads and sending InvCount, Rates!


9

Data Validation Report - DQE
Is done by Atomize

  • after checking if the bulk successfully finished, provide the integration partner the report
  • Atomize is interested in occupancy as well as Gross and Net Revenues
  • The Integration partner will process the DQE and send you a report afterwards
  • The report is to be found in the tab Office (Büro) of the front office in pAir.


10

Rates are updated by the Integration system

(OTA_HotelRatePlanNotifRQ Inbound)

  • Could be done as a preparation for 2-way during the 1-way installation
  • Create a new rate called "Atomize" which should be transmitted for testing by Atomize.
  • make a screenshot from pAir System Data with the short names of all the required rate codes from Protel (please sort them by short name before), and send it to the Integration partner




Two way Installation

Examples

1

Test incoming Rate update

(OTA_HotelRatePlanNotifRQ)

  • you can find Settings for specific Integration systems (RMS) on General (pAir)
  • you need to activate the Daily Rates per XSetUp
  • Atomize always wants to transmit the previously created rate "Atomize" as default.
  • after the Test rate code has been created, ask the integration partner to push some rates
  • Make sure that the rate which needs to be updated is set to yieldable.
  • Ask in the channel for which day the rate has been pushed, check the date and send a screenshot from the Protel Daily Rates if the rate for the specific date is showing in the list. 
  • Child Rates are not supported as DailyRates could not store rate values for children


Atomize always wants to transmit the previously created rate "Atomize" as default. Not only for test, also as Liverate
