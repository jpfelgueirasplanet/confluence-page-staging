Installation Steps

 

General steps

Examples

1

Request authentication details from the Integration partner for the new installation and request them to send you the authentication details on private Slack message or encrypted file by email and inform you by phone about the password.

Please provide them the HotelID as well - internally for Protel this is the CloudID 

  • Authentication method using the same Endpoint as default and the same Token given by the third party system for all hotels

2

General Preparation On Premise

IO_Pre-Setup-Checklist

3

Add licence of I/O Module to system data


4

Special XSetup Configuration

Set tax handling to net and gross

In the pws set tax handling to net and gross values.

pws | htng_tax_inlcuded | 2


Set default cashier

Check with the customer which cashier should be used for HotelResBot and if necessary set up a new cashier in protel system data.

Assign the cashier to the pWS with following Xsetup

pws | io_default_kasse | 0...n (local)

It can also be set in the pWS GUI on the Invoice Tab

5

Send list of Payment methods to Hotelresbot

Check with the customer which payment methods will be used by HotelResBot and send a list of these payment methods from the system data to the vendor so they can set up the payment codes.

6

Bob installation: one for OnPremise (if not existing), one for IDS and one for the Integration  

  • provide the token corresponding to the integration partner on private Slack message or encrypted file by email and inform you by phone about the password.
  • BOB:  Target and ResTarget should be set to io.protel.ids and protel IDS respectively

7

Configuring an IDS Instance for I/O: How to configure a protel I/O IDS 

The IDS is used for Reservation Import only

  1. Instace configuration - Export is not required - remove Export data flag
  2. Import configuration
  3. Activate Enable automatic generation for Room types and Rate code in Common - Channel Matching in the IDS. 



Deactivate Export data

Automatic Matchings in IDS

One way Installation


1

PWS Configuration

  • enable LiveMode and activate ToggleTrigger for required message types for 1-way connection

Message type

LiveMode

ToggleTrigger

OTA_HotelInvBlockNotifRQ

2

Create and start bulk uploads

  • for allotments (OTA_HotelInvBlockNotifRQ)
    for all future allotments


BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


3

Data Validation Report - DQE


  • No DQE required

Examples

2

Test an incoming Reservation 

(OTA_HotelResNotifRQ)

  • Important: check if the integration needs reservations with Credit Card, and create a test reservation with CC details. Verify if the details have been correctly saved in the PMS.


3

Test an incoming payment

(HTNG_ChargePostingRQ)

  • ask the integration partner to send a payment to the above reservation and check if the posting is done correctly
  • in the end, ask the integration partner to cancel above reservation


4
