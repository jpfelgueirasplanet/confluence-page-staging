How to handle support

 If there are any support issue you have to differentiate between

  •  Interface issues - like mapping errors, wrong guest data within the interface, communication issues between interface and pAir

These issues can be handled by the normal interface support users.

  • ESB issues - like communication issues between 3rd party vendor and ESB

These issues should be checked by the IO support users as they have the necessary access to Kibana. Please always provide the Correlation-ID of the issued message.

Logfiles

 The HTNG Interface has three different log files:

  • HTNG = default logfile as known from other interfaces which contain all communication between interface, pAir, and ESB
  • htng_in = which contains all detailed XML incoming messages from the ESB (HTNG_ReadRQ, HTNG_ChargePostingRQ, and the alive messages)
  • htng_out = all detailed XML messages send from the interface to ESB (HTNG_ResRetrieveRS and HTNG_ChargePostingRS)

The detail on this string: [7] 2020.02.18 08:30:34.489

Sting

Explanation

<OL>1003</OL>

outlet

<UN>37</UN>

employee ID / waiter ID

<SF>1</SF>

serving period

<TR>3332436219</TR>

???

<IN>WEL06884</IN>

invoice number vendor Infrasys - Information is placed as additional information in guest bill detail

<CT>324112942419148800</CT>

???

<PR>17257</PR>

???

<PG>16</PG>

product group transmitted by vendor system

<TX>19</TX>

tax value transmitted by vendor system

<WP>9,00</WP>

individual article price

<PT>S/S 2 Stunden</PT>

clear text article due to POS system

<SQ>1</SQ>

 sequence number of every posting, in case a posting is consisting of three individual strings you have sequence 1 for string 1, sequence 2 for sting 2, etc.

Important information as partners and Infrasys inquired extended functionality from HTNG_ResRetrieveRS

HTNG_ResRetrieveRS is providing information from PMS to POS systems.
Currently, the data volume provided in the string is restricted and not so powerful as many vendors and or partners like it to have.

Due to information by interface development Friedemann Vorwerk - 19.02.2020:

The interface POS HTNG connection will be rewritten in the year 2020.
It is planned to raise the availability and complexity of the information being provided to third party POS systems.
That means in the opposite, that no changes will be implemented in the current interface and on the OTA_ResRetrieve_RS string.
This applies to all currently released IO HTNG installations

(status 02/2020: Breeze und Infrasys)
