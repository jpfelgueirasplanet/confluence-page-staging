on prem | Configuration needed for PDF Invoice creation for inbound charges sent by PWS

This article explains which configuration needs to be in place for on prem properties to get the invoice forms created for inbound payments sent by PWS and additionally be able to send correct data out to any integration partner via HTNG_HotelCheckOutNotifRS in order to enable PDF invoice sending via email to the guest on their side.

It is split into two major cases - properties which are using fiscalization of invoices (+ special part for German properties with KSV) and properties which are not using fiscalization at all.

Configuration needed


PRE-REQUISITES

The invoice form will be generated by the COMServer (not PWS) and therefore it is important to check the comserver.ini settings in regards to:

  • correct program path (path also used for the client stations - not the local one of the server, e.g. c:\protel) in order to look up the form templates in the correct location and in order to save the created invoices in the right path
  • all necessary taxkey ini entries existing in order to fill the replacement codes properly on the invoice


Please also make sure

  • that you install LibreOfficePortable (not OpenOffice) on the same server the COMServer is installed at
    (check further down the respective link)

  • COMServer application
    that it gets installed with the windows user the COMServer is running on

    or

  • COMServer service
    that the service will be logged in with the windows user you installed the LibreOffice application with

Logging/Debugging Possibility

Testing Steps

  1. Find a reservation that departs on the current business date of the hotel
  2. Activate NDebug (Level 1) at the Comserver (start it as an application if it is only running as a service)
  3. Post a payment that balances the invoice through the 3rd party system (HTNG_ChargePostingRQ)
  4. Send a CheckOut through the 3rd party system (HTNG_HotelCheckOutNotifRQ)
  5. Check the outbound HTNG_HotelCheckOutNotifRS in Kibana

HTNG_HotelCheckOutNotifRQ

Good to know

The part sent in "px:DocumentData" in the Response (RS) message holds the base64-encoded PDF file and needs to be decoded by the Integration partner.

With the following tool (or any similar one) you can check the data from the RS on your own:

https://base64.guru/converter/decode/file
