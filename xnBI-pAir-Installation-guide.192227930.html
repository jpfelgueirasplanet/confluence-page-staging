One way Installation


1

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


2

Send Statistics

--> Trigger the OTA_HotelStatsNotifRQ once per type
--> each of the mentioned URL's - one by one
--> 45 days past / 365 days future



Log in to the pCC (System Data) for the specific HotelID and open an additional browser tab to post the URLs (not yet working with Next Gen - pCC Log in needed!):

Future
http://connect.protel.net/baoBackline/resources/interfaces/htng/trigger/stats/true?daysForth=365&step=3
http://connect.protel.net/baoBackline/resources/interfaces/htng/trigger/stats/false?daysForth=365&step=3
http://connect.protel.net/baoBackline/resources/interfaces/htng/trigger/stats/false?daysForth=365&segment=true&step=3

Past
http://connect.protel.net/baoBackline/resources/interfaces/htng/trigger/stats/true?daysBack=45&step=3
http://connect.protel.net/baoBackline/resources/interfaces/htng/trigger/stats/false?daysBack=45&step=3
http://connect.protel.net/baoBackline/resources/interfaces/htng/trigger/stats/false?daysBack=45&segment=true&step=3

pattern:
http://connect.protel.net/baoBackline/resources/interfaces/htng/trigger/stats/{true/false}?{daysBack/daysForth}=XXX&step=3

See pAir | Triggering messages: API Calls for further explanation


3

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ)
    for the last 5 years and ending 31.12.2099
  • for blocks OTA_HotelInvBlockNotitfRQ from the current date until 31.12.2099
  • HTNG_HotelFolioNotifRQ for the last 5 years and ending 31.12.2099



BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


4

Data Validation Report - DQE

  • after checking if the bulk successfully finished, provide the integration partner the report
  • The Integration partner will process the DQE and send you a report afterward
  • The report is to be found in the tab Reporting of the front office in pAir.

Partner will only validate the number of messages sent and received via the Bulk export and  so no

DQE is required. 

5

Update the developer portal with the information about the installed customer
