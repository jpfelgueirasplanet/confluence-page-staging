Installation Steps

 

General steps

Examples

1

Request authentication details from the Integration partner for the new installation and request them to send you the authentication details on a private Slack message or encrypted file by email and inform you by phone about the password.

Please provide them the HotelID as well - internally for Protel this is the CloudID 

  • EP is always the same and is set as default value in the manifest: 
    https://api.smarthosts.co/protel                
  • Authentication method is basic User/Pass provided per hotel - they have to be requested by the vendor

2

General Preparation for pAir General (pAir)



3

  • Special configuration 
    Opt-out by 3rd Party


SmartHost uses only Usubscribe from email main address 
Add OptOutBy3rdParty  at System Data
System Data →xSetup → Client→ Profile

4

  • Provide list with all Items (TAAs) related to fix charges

TAAs can be found in

System Data → Financial Configuration→ Items
ItemID = ID, Item Code = ITEM NUMBER (TAA)

5

  • Children Buckets (Discount Groups)

Children ages are configured in 
System Data → Rate Configuration→ Discount Groups

6

  • Bookable packages

Provide a list with all bookable packages related to the Rate Codes that Smarthost will use to create reservations
Make sure that the provided package item has the Bookable check box

System Data → Rate Configuration→ Rates→ Items
ID and Name  of the bookable package


7

  • Payment Methods

Provide a list of the payment methods the hotel decides to allow Smarthost to use

System Data → Financial Configuration→ Payment Methods

PMSRevenueCode =Item Number (TAA)

Deposit Items Setup

In order to be able to post Deposit payments, the following setup needs to be added in system data
https://help.protelair.com/en_US/billing-en/deposit-items


8

Bob installation: one for pAir (if not existing) and one for the Integration  

  • provide the token corresponding to the integration partner on private Slack message or encrypted file by email and inform them by phone about the password.

7

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ)
    for the last 10     years and ending in 31.12.2099

  • for profiles (OTA_ProfileCreateRQ) 
    for the last 10 years and ending in 31.12.2099


  • for Guest Stays ( OTA_HotelStayInfoNotifRQ)
    only upon customer's or integration partner's request


    BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

    Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

    In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


8

Data Validation Report - DQE

  • after checking if the bulk successfully finished, provide the integration partner the report
  • The Integration partner will process the DQE and send you a report afterwards
  • The report is to be found in the tab Office (Büro) of the front office in pAir.

DQE reports must be extracted for both Gross and Net Amounts
