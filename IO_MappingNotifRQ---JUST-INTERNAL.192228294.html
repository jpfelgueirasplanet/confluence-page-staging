Message Description

protel I/O uses a request and response message pair to notify the ESB about mapping that are made in protel. Some vendors require specific mappings to process messages that are transferred from the PMS to the vendor. In order to use the existing mapping UI in protel, a mapping service in the ESB has been created that performs the mapping. The following diagram shows the flow.

Red line = Configuration push from the PMS to the Mapping Service
Blue line = Operational flow (e.g. reservation message outbound)



The ACK indicates that the transaction has been completed successfully. To inform the User if the Mappings are stored successfully in the ESB, the ESB should reply with a IO_MappingNotifRS | Success message. If anything fail, the ESB should reply with an Error msg.
