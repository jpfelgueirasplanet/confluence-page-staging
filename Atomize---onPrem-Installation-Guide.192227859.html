2

Create a test reservation in the future (at least one year after the system date)

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 
  • always provide the reservation number to your integration partner, so that the reservation can afterwards be cancelled


3

Testing Occupancy

(OTA_HotelInvCountNotifRQ)

  • Trigger the OTA_HotelInvCountNotifRQ for 1 year once on the server of the customer where the PWS service is running 
  • Verify in Kibana that the outgoing messages OTA_HotelInvCountNotifRQ is answered with RS message from the Integration system
  • for Occupancy the PWS will send the OTA_HotelInvCountNotifRQ automatically 3 times a day at 5,14 and 22 o'clock

Open a browser to post the URLs:
(NOTE - you have to change some details here!)

pattern:
http://IP-address(PWS):Port(PWS)/ota/hotelinvcountnotif/{MPE_hotel}/send/{START:YYYY-MM-DD}/{END:YYYY-MM-DD}

e.g.: http://127.0.0.1:8046/ota/hotelinvcountnotif/1/send/2020-01-13/2020-12-31

4

Sending Rates to Atomize

  • After the installation is done, start a full data Export and check in Kibana that outgoing OTA_HotelRatePlanNotifRQ is answered by OTA_HotelRatePlanNotifRS and no error occurs. 

5

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ) for the last 2 years and ending with 31st Dec 2099
  • for packages (IO_RatePackageNotifRQ) starting today ending with 31st Dec 2099
  • for blocks (OTA_HotelInvBlockNotifRQ) → PWS release >= 2.13.x needed
    • Start date: 1st of running month 2 years ago (if today is 6th March 2019 - start date would be 01/03/2017)
    • End date: 31/12/2099

BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 



6

Please inform Atomize that you are done with the Bulk Uploads and sending InvCount and Rates!


7

Data Validation Report - DQE
Is done by Atomize

  • after checking if the bulk successfully finished, provide the integration partner the report
  • The report is to be found in the tab Add-Ons of the front office
  • IO Hotel Status sent both Gross and Net Revenue. They are interested in both revenues and occupancy. 
  • The Integration partner will process the DQE and send you a report afterwards

Test incoming Rate update

(OTA_HotelRatePlanNotifRQ)

  • Atomize always wants to transmit the previously created rate "Atomize" as default.
  • make sure that the rate which needs to be updated is set to yieldable
  • ask the integration partner to push some rates
  • ask the customer to check the data via FO - Add Ons - Daily Rates - Enter daily rates editor
  • protel database table is ckit_dailyrates_rates
  • will be received inside an inbound OTA_HotelRatePlanNotifRQ message
  • AdditionalGuestAmounts are also supported - those will be added to the occupancy up to 6 adults
  • Child Rates are not supported as DailyRates could not store rate values for children

Atomize always wants to transmit the previously created rate "Atomize" as default. Not only for test, also as Liverate
