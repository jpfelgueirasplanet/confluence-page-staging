on prem | Resend IO Messages CKit

  • CKit
  • Versions History
  • Installation Guide
  • User guide


This Ckit provides a user the following possibility

  • to check, if an IO message is waiting in the queue to be processed (Awaiting Pickup)
  • to check, if an IO message has already been successfully processed (Message Picked up)
  • and to resend IO messages if needed (Resend IO messages)

Installation Guide

Step

Description

Image

1

Download and copy the CKit file to {ProtelPath}\System\Ckit\Xml


2

Restart protel System Data and go to [protel SD > Misc. -> Construction Kit-> IO Messaging

3

Click on [Initialize] to initialize the CKit. 

4

Click on [User Group access rights] to configure the User groups. 

By default all user group don't have access (permitted No), with a double-click on one item to enable permission to a group of users or disable permission. Example, double-click on “Front Office Agents” user group.

Note: Only users in an enabled permission group user can access to edit reason function in Add-Ons


5

Click on [IO Reservation Manipulate] to check the Version.

User guide

Step

Description

Image

1

Go to FO -> Add-Ons -> IO Messaging -> Awaiting Pickup



1a

System shows input window

You have the possibility to select between

  • AllotmentID = kontinge.kontinnr
  • ProfileID = kunden.kdnr
  • ReservationID = buch.leistacc/buchold.buchnr/stobuch.buchnr

1b

If ID exists, system show list and the current State of the Message

  • Waiting to be picked up
  • Message is waiting for the recalcutation of the TaskServer

1c

If no Message Type is selected and the ID input do not exist, system show, a list of all message that are waiting to be picked are displayed in the result.


1d

If a specific Message Type is selected and the ID input do not exist, system show, a list of all message that are waiting to be picked are displayed in the result.

2

Go to FO -> Add-Ons -> IO Messaging -> Message Picked up

2a

System shows input window

You have the possibility to select between

  • AllotmentID = kontinge.kontinnr
  • CorrelationID = unique ID for the Message (provided by Kibana, or the integration partner)
  • ProfileID = kunden.kdnr
  • ReservationID = buch.leistacc/buchold.buchnr/stobuch.buchnr

2b

If ID exists, system show list

2c

If the ID input do not exist, system show message

3

Go to FO -> Add-Ons -> IO Messaging -> Resent IO messages

3a

System shows input window

You have the possibility to select between

  • AllotmentID = kontinge.kontinnr
  • ProfileID = kunden.kdnr
  • ReservationID = buch.leistacc/buchold.buchnr/stobuch.buchnr


You have the possibility to select one ID or multiple IDs, multiple IDs must be separated with coma (,)

3b

Users choose message type and input ID then click OK

3c

If the ID input do not exist, system show message

3d

If user do not have permission, system show message:
