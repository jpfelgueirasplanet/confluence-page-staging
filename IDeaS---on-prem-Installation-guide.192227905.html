Check for reservations AND allotments without market code with the provided Ckits (IDeaS only accepts reservations and allotments with market code - if you don't correct it, there will be discrepencies in the DQE)

Reservations:

--> let the customer check the ones which are shown in CKit 'ReservationsNoMarket' and fill the market code manually OR ask the customer which value should be set in the database in general BEFORE starting the Bulk Upload

--> it is also needed to update the varbuch (RBD / ZPÜ) table accordingly
--> the following query will update the RBD entries with the market code from buch table (reservation screen) for all future & inhouse reservations

update varbuch set varbuch.market = buch.market
from varbuch
inner join buch on buch.buchnr = varbuch.buchnr
where varbuch.market < 1 and buch.market > 0
and buch.buchstatus < 2 and buch.reschar <> 6
and buch.mpehotel = XY


--> you also need to check if there are past reservations without market code - this could be done with CKit 'dayhistNoMarket' - the Ckit will only display a list of reservations
--> you need to fill them with a default market code (provided by the customer) via SQL query:

To find out the internal ref of the market code:

select *from market

To update the reservations in tables 'dayhist' and 'buchold' (please change xy with the correct values and enter the start date of the planned Bulk upload:

update dayhist set market=xy where market= -1 and mpehotel=xy and datum >= '2018-01-01'
update buchold set market=xy where market= -1 and mpehotel=xy and datumbis >= '2018-01-01'

Allotments:

--> The customer should go through the allotments and add the needed market codes
--> you can provide a list of the allotments the market code is missing in with the following query:

select kontname, datumvon, datumbis from kontinge where market = -1 and mpehotel= xy

General information about the 2-way activation

IDeaS Installation team will inform the Integrations Support Team / Distribution Partner about the scheduled date for the initial upload of decisions.

Integrations Support Team / Distribution Partner need to respond back with a certain time for a call together with IDeaS which is then meant for doing the test of the upload together. IDeaS will send an invitation for the call.


The customer will be informed only as soon as the uploads have been successfully tested together with IDeaS (upload itself and some Cross Checks of dates). The property will be advised by IDeaS to do checks on their own to guarantee the correctness of the data.

Before handing the data over to the customer a fresh set of StatsNotif messages need to be sent from protel side to trigger a new decision upload on IDeaS side.
