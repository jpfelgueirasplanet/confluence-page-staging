Case 3 - Checking Reported Discrepancies (on prem)

PWS release version >= 2.16.x

This version includes a stored procedure which is used to calculate the numbers for the Stats messages.
It also allows you to check the included reservations/allotments which might help in case you discover any discrepancies when comparing the numbers to any reports in protel.

In order to see the calculated content of this stored procedure, you need to run it first of all.
You can achieve that by

  • either triggering the messages manually
  • or execute the stored procedure via SQL Management Studio.

Executing the stored procedure via SQL Management Studio

  1. Find it and select 'Execute Stored Procedure'

      
  2. Tick All "Pass Null Value"



Display the calculated numbers

SQL Management Studio

  1. Enter the Start and End date + MPE Hotel number



  2. Execute the query in order to display the numbers

SQL Editor - protel FO

Sample query:

EXEC proteluser.pws_getreservationstatistics
@start = '2021-04-11',
@end = '2021-04-15',
@mpe = 1
