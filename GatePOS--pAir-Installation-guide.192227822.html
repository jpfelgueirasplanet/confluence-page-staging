Installation Steps

 

General steps

Examples

1

Request authentication details from the Integration partner for the new installation and request them to send you the authentication details.

Please provide them the HotelID as well - internally for Protel this is the CloudID 

Data Security

Please have in mind that both vendor and IO interface token information is sensitive information that might be transported by Slack,
team viewer session in case of a 1:1 session, or with two-way authentication methods by other media. 

  • Authentication method using the same Endpoint as default and a specific Token given by the third party system for each hotel customer

2

General Preparation for pAir 
https://confluence.protel.net/display/PIODOC/pAir+%7C+General+preparation



3

Bob installation: one for pAir (if not existing) and one for the Integration  

Data Security

Please have in mind that both vendor and IO interface token information is sensitive information that might be transported by Slack,
team viewer session in case of a 1:1 session, or with two-way authentication methods by other media. 

4

  • Provide list with all Items (TAAs) related to GatePOS charges


TAAs can be found in

System Data → Financial Configuration→ Items → ItemID = ID, Item Code = ITEM NUMBER (TAA)

5

  • Payment Methods

Provide a list of the payment methods the hotel decides to allow GatePOS to use

6

  • Add a cash Register

This cashier is going to be used from GatePOS in order to add payments in the billing of the guest

1.Define a default cash register in pAir
System Data→ Financial Configuration→ Cash Registers



2.Add an xsetup that is used to define the default cash register for all IO-related postings in the configuration editor
X-Setups→ Interfaces→ IO→ Default IO Cash Registers

7

Create a profile for the Passer - By Customers

For all Passer-By postings, a profile needs to be created and used.
It is necessary to provide Vendor with this profile ID.
If the Hotel has already a profile for this purposes then this profile ID provided to the Vendor 
else it is necessary to Create a new one.

 

One way Installation


1

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


 

Two way Installation

Examples

1

  • Use a Test reservation ( in non Checked-In status) to add a fix charge
IO_ResModifyRQ Sample
<p:IO_ResModifyNotifRQ xmlns:p="http://protel.io/soap" CorrelationID="0ac40fad-e66e-4eb5-a722-b7d1ac6c056f" TimeStamp="2024-02-01 08:21:24.913" Version="1.00">
<p:Reservations HotelCode="3599">
<p:Reservation ArrivalDate="null" DepartureDate="null" ResStatus="Reserved">
<p:UniqueID ID="" ID_Context="protelIO" Type="Reservation"/>
<p:FixedCharges>
<p:FixedCharge Action="CREATE" From="2024-02-01" Quantity="1" Recurrence="Fixed" To="2024-02-01">
<p:InvoiceText>Adult - SA</p:InvoiceText>
<p:Price CurrencyCode="ZAR" SingleAmountAfterTax="67"/>
<p:Item Code="4003"/>
</p:FixedCharge>
</p:FixedCharges>
</p:Reservation>
