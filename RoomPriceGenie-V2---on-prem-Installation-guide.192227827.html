    Installation Steps

    General steps

    Examples

    Request authentication details from the Integration partner for the new installation and request them to send you the authentication details  on a private Slack message or encrypted file by email and inform you by phone about the password.

    Please provide them the HotelID as well - internally for Protel this is the CloudID 

    Authentication method using the same Endpoint as default and same Token (separate toekns for protel Air and protel onPrem) given by the third party system for each hotel customer




    General Preparation On Premise

    IO_Pre-Setup-Checklist

    Add licence of I/O Module to system data


    Bob installation: one for OnPremise (if not existing), one for the Integration, one for SCS and one for IDS

    • provide the token corresponding to the integration partner on private Slack message or encrypted file by email and inform you by phone about the password.

    • Create an SCS installation module in Bob

      • When requesting a Token for an installation from another colleague, please don't forget to mention that an SCS installation is needed as well

    Global Installation for onPrem or pAir  plus the Integration Installation and the SCS  installation 

    OneClick- installation

    For Room Price Genie installations, please always use this way of I/O -  installation. 

    Please note:
    OneClick- installation is only replacing the installation in BOB. All necessary pre-setups are still requested as usual. 

    Please log in to the protel store: https://store.protel.io/#/ and search for RoomPriceGenie in the search bar.


    Make sure that you selected the hotel that is meant to get the product. Please compare the CloudID just to make sure. 


    Click "Get".


    Go through the wizard by clicking "next"

    Accept the Terms of Service

    Finish the installation:


    PWS Configuration

    • update to the latest release version 
    • update I/O configuration

    PWS | IOStatsEodDaysPast | 730

    PWS | IOStatsEodDaysForecast | 365

    One way Installation


    1

    a) Create four SCS Jobs in the SMP

    Details

    * OTA_HotelInvCountNotifRQ for 730 Days - one job for each of the following times (Daily frequency)

    --> 09:00
    --> 12:00
    --> 18:00
    --> 22:00


    b) Verify in Kibana that the outgoing message (OTA_HotelInvCountNotifRQ) is answered with RS message from the Integration system




    2

    Create and start bulk uploads

    • for reservations (OTA_HotelResNotifRQ) for the last 2 years and ending with the next 2 years


    Please set the Pause per message to 1 second (to prevent the overloading of Roompricegenie's servers)


    BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

    Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

    Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

    In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 

    3

    PWS Configuration

    • enable LiveMode and activate ToggleTrigger for required message types for 1-way connection


    Message type

    ToggleTrigger

    LiveMode

    OTA_HotelResNotifRQ

    OTA_HotelInvCountNotifRQ 


    OTA_HotelInvBlockNotifRQ

    OTA_HotelStatsNotifRQ

    4

    Configuring an IDS Instance for I/O: How to configure a protel I/O IDS

    • ESB server connection
    • Add a new instance
    • Configure the Comserver URL
    • Add the URL for Export
    • Add the URL for Import
    • Add the SQL database
    • Configure the Email Notification
    • Add the Token and Module Name in the Special Tab
    • Export  Set export span for 1,5 years (18 months) in future, every 5 minutes
      • Rate Values - OTA_HotelRatePlanNotifRQ
        • Configure the Children 
    • Create DCS mapping
      • mpehotel
      • RoomType
      • RateCode
      • dailyrate

     Note: If automated matchings are used in the IDS configuration, then the IDS will create the matching for the respective category itself. But this will disable the option to do any adjustments on the auto-matched list. Thus, it is recommended to do the matchings manually

     For the purpose of the installation test, it is enough to do the mapping for a few room/rate combinations. Then instruct the Customer how to manage the mappings on their own

     Important: Only Rate updates (OTA_HotelRatePlanNotifRQ) export should be activated in IDS. If any other export is activated (e.g. OTA_HotelInvCountNotifRQ or/and OTA_HotelAvailNotifRQ) - make sure to deactivate it!

     

    - Set the Daily rates for the 2 way test:

    link: protel Daily Rates - SPE/MPE Documentation - Confluence

    check with the customer in advance about the rate codes and room types that will be mapped with the vendor.

    RPG supports only one base rate






    5

    • Using IDS/Start full data export for 1,5 years (18 months) in future for Rates 

     Could be adjustable per Customer

    •  Verify in Kibana that the outgoing messages is answered with OTA_HotelRatePlanNotifRS and OTA_HotelAvailNotifRS respectively from the Integration partner 


    6

    Data Validation Report - DQE


    • after checking if the bulk successfully finished, provide the integration partner the report
    • The Integration partner will process the DQE and send you a report afterwards
    • The report is to be found in the tab Add-Ons of the front office

    • Rooms

    Request Message

    <?xml version='1.0' encoding='utf-8'?>
    <env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">
    <env:Header>
    <htnga:CorrelationID xmlns:htnga="http://htng.org/PWSWG/2007/02/AsyncHeaders">CorrelationID123</htnga:CorrelationID>
    <wsa:Action xmlns:wsa="http://www.w3.org/2005/08/addressing">IO_SystemDataRQ</wsa:Action>
    </env:Header>
    <env:Body>
    <p:IO_SystemDataRQ xmlns:p="http://protel.io/soap" TimeStamp="2020-08-20T07:15:51.4023488Z" Version="1.000" CorrelationID="CorrelationID123" EchoToken="CorrelationID123">
    <p:Request xmlns:p="http://protel.io/soap">
    <p:UniqueID Type="10" ID="2575" ID_Context="protelIO"/>
    <p:Rooms/>
    <p:Rates/>
    </p:Request>
    </p:IO_SystemDataRQ>
    </env:Body>
    </env:Envelope>

    4

    Test incoming Rate update

    OTA_HotelRatePlanNotifRQ Inbound

    • the integration partner should ask the customer to create a Test rate code
    • after the Test rate code has been created, ask the integration partner to push some rates
    • Make sure that the rate which needs to be updated is set to yieldable. Ask in the channel for which day the rate has been pushed, check the date and send a screenshot from the Protel Daily Rates if the rate for the specific date is showing in the list. 

    • as soon as this has been tested successfully repeat the tests for incoming rate updates for a rate or rate group chosen by the customer


    5

    Test incoming Restrictions 

    OTA_HotelAvailNotifRQ Inbound

    --> only minimum stay restrictions are supported (e.g. m3 - see screenshot)

    • ask the Integration partner to clarify with the customer to have the Test rate code invalidated (through expired validity) after the verification of the Restriction message is done
    • Make sure that the rate which needs to be updated is set to yieldable. Ask in the channel for which day the rate has been pushed, check the date and send a screenshot from the Protel Daily Rates if the rate for the specific date is showing in the list. 
    • Ask the Integration Part in the Slack channel of the integration to push a restriction for a specific day on the Test rate code and then check in the Price availability for the given date if the restriction has been set.

    Update the protel SMP page with the information about the installed customer
