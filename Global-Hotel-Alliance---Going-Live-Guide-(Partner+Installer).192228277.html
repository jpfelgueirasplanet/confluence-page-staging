Before starting to work with the Global Hotel Alliance interface for protel.I/O, please make sure you have fulfilled all of the requirements from the pre-setup documents.

To be able to go live with a property, you have to follow a certain order when exporting the data to GHA DISCOVERY. It is very important to keep this in the correct order, otherwise a correct transmission of data cannot be guaranteed.

  • Inventory > via IDS
  • Package > via PWS
  • Rate > via IDS
  • Restriction > via IDS
  • Inbound Reservation > ask GHA DISCOVERY to push a inbound Reservation
  • Profiles > via pWS
    • Profile LookUp/Download
    • Fetch the next free GHA DISCOVERY Number
    • Live Mode for Profile upload
  • Stay > via pWS
  • Loyalty Info > via PWS
  • Loyalty Inquiry > via PWS
  • Loyalty Charge > via PWS

 Please note: Profiles and Stay must not be synced during the going live of the property. GHA DISCOVERY will receive the profile update as soon as a profile is modified.

 

General steps

Examples

1

IDS Configuration

  • Enter 1440 minutes into the Export changes every…. Minutes selection box. This ensures that no data is exported during the going-live process.
  • change the date range to 1 year. Normally the data for the following year is pushed to GHA DISCOVERY. If a customer would like to push more days, changes need to be made in the Export tab of the IDS.
    • Please note: On GHA DISCOVERY’s side, there is a minimum of 365 days and a maximum of 396 days. This means that no more than 396 days can be synchronized into the future.

2

To start with the inventory (RTAV) Export, please enable the Export Availability Values check box in the Export tab.

Double check that all other messages are deactivated. GHA just expected Inventory messages.

After running an export for the availability, please unselect the Export Availability values checkbox.

To make sure everything has been exported correctly, compare the availability shown in GHA DISCOVERY with the availability indicated in the protel PMS.

3

The second export is the packages export (IO_RatePackageNotifRQ – PACKAGE). This message is sent via the protel Web Services (pWS). For the first full upload, please create a bulk upload in the protel Service Market Place for the message type “IO_RatePackageNotifRQ”.



After the bulk upload has been successfully completed and all packages are uploaded to GHA DISCOVERY, the property should check the packages in GHA DISCOVERY. To successfully export the packages, the live mode needs to be activated. To do so, go to [pWS > Settings > protel IO Configuration]. 

In the line with “IO_RatePackageNotifRQ”, select the Live Mode checkbox. Also make sure the Live Mode checkbox, located in the upper part of the window, is activated. Otherwise, no messages will be exported.


4

Next, we need to export the rates. The corresponding message (OTA_HotelRatePlanNotifRQ – RATE) is sent via the IDS.

In the Export tab, click on the Rate Values tab and select the Export Rate Values checkbox.

Please note: Since GHA DISCOVERY cannot handle spaces in rate codes, make sure the mapping values for rate codes have been named properly.


As soon as the export is complete, go back to the IDS settings and unselect the Export rate values checkbox.

5

The last export to be executed is the restrictions export (OTA_HotelAvailNotifRQ– RAVR). This message is also sent via the IDS.

There will always be a combination of room type and rate code that is sent to GHA DISCOVERY on a daily basis.

In the Export tab located in the IDS, go to the Restriction Values tab. Select the Export Restriction Values checkbox.


As soon as the export is complete, go back to the IDS settings and unselect the Export restriction values checkbox.


6

End of sync

GHA DISCOVERY now checks whether everything has been properly transmitted. If everything works, the status will change to LIVE. You can now, for example, change the rhythm of the export in the protel IDS to three minutes.

Afterwards, go back to the Availability values, Rate values, and Restriction values tabs and select the Export values checkbox.


Once all the restrictions have been synced, please contact the GHA DISCOVERY support to verify that all messages have been processed. This is the end of the sync process. The property now has to continue with the GHA DISCOVERY OCM configuration so that the property can be set live on discoveryloyalty.com.

7

After the above settings have been completed, you can start the reservation import (OTA_HotelResNotifRQ – RESERVATION). To do so, select the Import reservations checkbox located in the Import tab of the IDS settings.

The PULL and PUSH mode will be required for importing the reservation.


If the GHA DISCOVERY support is pushing a test reservation, please check in the IDS if everything is working as expected with this inbound Reservation.

8

Profile LookUp/Download

Go to protel [protel FO > Profiles > Private]. Type in the desired profile name and start the search function.

You can search for first or last name, city, zip code, email address, or telephone number.

In the search results, some profiles written in blue letters are marked with a little globe in the first column. These are the results of the GHA DISCOVERY profile look up. Each line displayed is one configured address in GHA DISCOVERY.

By double clicking on a line, the respective profile is downloaded and opened. As soon as a profile has already been downloaded, it is displayed in black letters with a black arrow in the first column.

  • Please check back with the GHA DISCOVERY support after the first profile download if the download worked correctly and if they received a protel profile ID for the specific GHA DISCOVERY profile.

 The red arrow marks a profile from protel.

To start a search with only a part of the full name, use “%” before or after the name in the search field. You can search for last name, first name, city, country or loyalty card.



9

Fetching the next free GHA DISCOVER number

When working with GHA DISCOVERY, a loyalty card is assigned for each imported profile. To assign the respective number to the loyalty card, the next free number needs to be fetched from GHA DISCOVERY.

Please note: only one valid/active GHA DISCOVERY membership card can be added. A notification message appears when attempting to add another predefined GHA DISCOVERY membership card.

Irrespective of this, other types of valid/active membership cards (apart from GHA DISCOVERY) can still be added or changed.


To start fetching a new number for a profile, please proceed as follows:

Go into an existing profile and create a new member card by clicking your right mouse button in the Member cards line. In the following window, choose DISCOVERY from the Loyalty program drop down menu. Afterwards, the New number button appears.


Click on New number and the next free number is downloaded from GHA DISCOVERY and appears in the Member ID field.


To finally add the number to the profile, click on OK to save. To make sure everything works as expected, please check back with the GHA DISCOVERY support.






10

Profile Upload

The last step to successfully import the profiles, is to switch to live mode in the pWS. This is done in the protel.I/O Configurations located in the pWS settings. Select the following checkboxes in the Live Mode column: OTA_ProfileModifyRQ, OTA_ProfileCreateRS, OTA_ProfileCreateRQ, OTA_ProfileModifyRS.


Remember to select the Live Mode checkbox located at the top of the window.

Afterwards, please check back once again with the GHA DISCOVERY support.


11

IO_MappingNotifRQ

Enable the Live Mode for the IO_MappingNotifRQ, automatically a message will be push to the ESB, and the ESB will store all needed information for the Stay Message


12

Stay Upload

The export of stay revenue information is executed automatically every day after the end-of-day procedure. The export includes information per guest and reservation, as well as check-outs and no-shows for the respective day that is ending. By means of this message, GHA DISCOVERY is calculating the length of the stay, by using the information on used market codes during a stay. Information concerning the guest’s invoice is also included. It is performed for all profiles with a GHA DISCOVERY loyalty card.

To make sure everything gets properly exported, the live mode needs to be activated. This is done in the pWS. Go to [pWS > Settings > protel IO Configuration]. In the respective message’s (OTA_HotelStayInfoNotifRQ) line, select the Live Mode checkbox.


13

DISCOVERY Dollars

Members earn "DISCOVERY Dollars" with every stay in an eligible hotel. They can be redeemed for goodies, like experiences, or even the whole stay.

Enable the Live Mode for 

  • IO_LoyaltyAccountInfoRQ/RS
  • IO_LoyaltyAccountInquiryRQ/RS
  • IO_LoyaltyAccountChargeRQ/RS


Test if you can use the Discovery Dollars for paying an invoice.
