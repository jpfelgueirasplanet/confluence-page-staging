Post Tip charge, pAir

pAir necessary setup

  1. Create Cash Register and set a default one for IO: Settings/Financial Configuration/Cash Registers

  2. Create method of payment to use for the Tip: Settings/Financial Configuration/Payment Methods

    - set the MOP to Pay-in/Pay-out
  3. Create TAA code for the Tip charge: Settings/Financial Configuration/Items

    - The item should be marked as PAID OUT
    - Add the configured “Pay-in/Pay-out” payment method

HTNG_ChargePostingSamples and evaluating the result

Post the tip always with SubTypeID="Revenue" and positive @Amount 

  1. Post Tip using reservation ID:

    Tip to reservation ID
    <ns3:RevenueDetail SubTypeID="Revenue" Description="tip external 2" PMSRevenueCode="998" Amount="15.00" CurrencyCode="EUR">
    <ns3:Account Type="14" ID_Context="protelIO" ID="5372"/>
    </ns3:RevenueDetail>

  2. Post Tip using folio ID:

    Tip to folio ID
    <ns3:RevenueDetail SubTypeID="Revenue" Description="tip external 2" PMSRevenueCode="998" Amount="15.00" CurrencyCode="EUR">
    <ns3:Account Type="16" ID_Context="protelIO" ID="7865"/>
    </ns3:RevenueDetail>

  3. Post Tip using profile ID:

    Tip to profile ID
    <ns3:RevenueDetail SubTypeID="Revenue" Description="tip external 2" PMSRevenueCode="998" Amount="15.00" CurrencyCode="EUR">
    <ns3:Account Type="21" ID_Context="protelIO" ID="6775"/>
    </ns3:RevenueDetail>

As a result, the cash register balance will reduce with the amount of the posted Tip: PMS/Cashiering/Cash Register Closure (login to the resp cash register setup!)


Tip to profile ID
<ns3:RevenueDetail SubTypeID="Revenue" Description="tip external 2" PMSRevenueCode="998" Amount="15.00" CurrencyCode="EUR">
<ns3:Account Type="21" ID_Context="protelIO" ID="6775"/>
</ns3:RevenueDetail>

Usage of Folio Windows in onPrem:

PREM-2573

During inbound HTNG_ChargePostingRQ it is possible that reservation folio windows would not be enough - or not used in an optimized way. For the purpose - the following two scenarios would give further information how integrations partners should develop the approach per use case:


  1. Use Case 1 - Posting a charge from vendor app

    If the guest wishes to post a room charge using a reservation ID then the following will happen
    1. If there is a folio open then the charge will be posted to the guest folio as defined in the message
    2. If there is no folio open then the PMS will respond with no Folio available, hence posting of room charge not possible, leaving the guest with the ability to pay for the charge on the app

  2. Use Case 2 - Posting a charge plus the complete payment

    1. In the case of a guest posting a charge and the payment, the Profile ID should be used to create a passerby account, posting the charge + payment, closing the passerby and generating the invoice. 
    2. The charge and payment in this case, will have no relation to the reservation and will only be linked to the profile. 

e.g.

Call passerby invoice
<ns3:RevenueDetail SubTypeID="Revenue" Description="Breakfast" PMSRevenueCode="998" Amount="15.00" CurrencyCode="EUR">
<ns3:Account Type="21" ID_Context="protelIO" ID="6775"/>
</ns3:RevenueDetail>
<ns3:RevenueDetail SubTypeID="Payment" Description="tip external 2" PMSRevenueCode="3" Amount="-15.00" CurrencyCode="EUR">
<ns3:Account Type="21" ID_Context="protelIO" ID="6775"/>
</ns3:RevenueDetail>
