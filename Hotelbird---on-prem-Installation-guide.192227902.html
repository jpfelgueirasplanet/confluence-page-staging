13

Errors regarding the HTNG_ChargePostingRS

  • The third-party would not be able to process the payment because it took more than 30s for the HTNG_ChargePostingRS to be sent out. In Kibana The HTNG_ChargePostingRS will include a success

 To mitigate the problem the following configuration settings are suggested as may  reduce the comserver load

  1. Make sure PWS is connected to its own taskserver
  2. The new taskserver should have nothing else running against it, no ckits, no IDS
  3. IDS should be pointed to its own taskserver and not the comserver
  4. Taskserver should be set to reload system data cache every 60 minutes
    TaskServer_Setup
  5. Set this on taskserver.ini as that is where the pws is pointing
    [system]
    debugevtimerexecutiontimer=1
    thresholdevtimer=900
  6.  Make sure passerby account is configured and available
  7. Add the xsteup on global level
    ComServer | Whitelist | GetComputerName 
  8. If there is a timeout or error, check what the taskserver displays on the screen.

14

Hotelbird is not saving a profile when it is not linked to a reservation. So they will send an error in the OTA_ProfileModifyRS for the profile which are not linked to a reservation

Error Code="635">Guest profile not found

This error can be ignored

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


3

Test outbound profile creation/modification
(OTA_ProfileCreateRQ/OTA_ProfileModifyRQ )

  • Modify the profile of the test reservation created during the previous OTA_HotelResNotifRQ
  • Create test examples of the individual, company profiles. Verify in Kibana that outgoing OTA_ProfileCreateRQ is answered by RS and no error occurs.
  • Modify the individual company profiles by changing name and address details. Verify in Kibana that the OTA_ProfileModifyRQ is answered by the a OTA_ProfileModifyRS and no errors are present. 
  • Cancel your test reservation afterwards.


4

Create and start bulk upload

  • for reservations (OTA_HotelResNotifRQ) for the next 3 years starting on the installation date

BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one.

Checkin Workflow

Please inform Hotelbird to do a private to business check-in

1. Test an incoming private profile lookup → (OTA_ReadRQ)

  • Create a test reservation  (arrival date needs to be current PMS date) the profile should hold an email address and provide the reservation number to Hotelbird
  • Verify in Kibana that the incoming OTA_ReadRQ is answered by OTA_ProfileReadRS and contains no errors.

2.  Test an incoming Profile Modification → (OTA_ProfileModifyRQ)

  • Verify in Kibana that the OTA_ProfileModifyRQ is answered by RS and no errors occurs
  • Check in the PMS if the profile has been modified

3. Test an Incoming  adding a template routing instruction → (IO_ResModifyNotifRQ)  

  • Verify in Kibana that the incoming IO_ResModifyNotifRQ is answered by IO_ResModifyNotifRS and contains no errors.
  • Check in the PMS if the routing instruction has been added to the reservation

4. Test an Incoming invoice detail request → (HTNG_HotelFolioRQ)  

  • Verify in Kibana that incoming HTNG_HotelFolioRQ is answered by HTNG_HotelFolioRS and contains no errors.

5. Test an incoming company profile lookup → (OTA_ReadRQ)

Note

  • If the company profile does not exist in the pms, Hotelbird will create a new profile.
  • If the profile exists in the pms, Hotelbird will not create the profile and use the existing one.
  • However, if the profile exists but includes an invoice recipient attached, Hotelbird will create a new profile
  • Verify in Kibana that the incoming OTA_ReadRQ is answered by OTA_ProfileReadRS and contains no errors.

6. Test an incoming Company profile Creation → (OTA_ProfileCreateRQ)  

  • Verify in Kibana that incoming OTA_ProfileCreateRQ is answered by OTA_ProfileCreateRS and contains no errors.

7.  Test an incoming Change invoice profile → (IO_ResModifyNotifRQ)  

  • Change the invoice recipient of Folio B of the previous reservation
  • Please note:  It is not possible to modify the first folio as it is reserved for the profile of the arriving guest!
  • Verify in Kibana that incoming IO_ResModifyNotifRQ is answered by IO_ResModifyNotifRS and no error occurs
  • Check in the PMS if the profile of Folio B is changed to the company profile created in the OTA_ProfileCreateRQ step

8. Test an incoming reservation lookup → (OTA_ReadRQ)

  • Verify in Kibana that the incoming OTA_ReadRQ is answered by OTA_ResRetrieveRS and contains no errors.
  • Confirm room number specifically

9. Test an incoming Check-In →  (HTNG_HotelCheckInNotifRQ)

  • Check whether the room of the test reservation has the correct status (as configured with XSetUp)
  • Verify in Kibana that incoming HTNG_HotelCheckInNotifRQ is answered by HTNG_HotelCheckInNotifRS and no error occurs
  • Check in the PMS if the Reservation is checked-in successfully

Undo check-in and cancel the test reservation afterwards

Please inform Hotelbird to do a business to private check-in

1. Test an incoming Profile Lookup → (OTA_ReadRQ)

  • Create a test company reservation  (arrival date needs to be current PMS date) the profile should hold an email address and provide the reservation number to Hotelbird
  • Verify in Kibana that the incoming OTA_ReadRQ is answered by OTA_ProfileReadRS and contains no errors.

2.  Test an incoming changing attached guest profile  → (IO_ResModifyNotifRQ)

  • Verify in Kibana that incoming IO_ResModifyNotifRQ is answered by IO_ResModifyNotifRS and no error occurs
  • Check in the PMS if the company profile changed to the private reservation

3. Test an incoming Check-In →  (HTNG_HotelCheckInNotifRQ) 

  • Check whether the room of the test reservation has the correct status (as configured with XSetUp)
  • Verify in Kibana that incoming HTNG_HotelCheckInNotifRQ is answered by HTNG_HotelCheckInNotifRS and no error occurs
  • Check in the PMS if the Reservation is checked-in successfully

Undo check-in and cancel the test reservation afterwards

Manual Room assignment by Hotelbird

  • Test an incoming Availability check → (OTA_HotelAvailRQ) 

please check if the OTA_HotelAvailRQ was responded successfully with an OTA_HotelAvailRS

  • Test an incoming Room assignment via IO_ResModifyNotifRQ and HTNG_HotelRoomMoveNotifRQ/RS

please check if the reservation got assigned to the room which was allocated through the OTA_HotelAvailRS and check if the HTNG_RoomMoveNotifRQ/RS flow was success

CheckOut Workflow (Departure must be on current day)

1. Test an incoming invoice detail request  → (HTNG_HotelFolioRQ) 

  • Provide the reservation number of the test reservation prepared during the general preparation step
  • Ask the integration partner to view the invoice
  • Verify in Kibana that incoming HTNG_HotelFolioRQ is answered by HTNG_HotelFolioRS and contains no errors.
  • Open the invoice in the PMS and compare the amount of the invoice with the amount of the integration partner

2. Test an incoming Payment posting → (HTNG_ChargePostingRQ)

  • Request the partner to pay the invoice of the previous reservation
    The partner should use the PMS payment codes you provided during the general preparation step
  • Check if the amount was posted correctly in the PMS
  • Verify in Kibana that incoming HTNG_ChargePostingRQ is answered by HTNG_ChargePostingRS and no error occurs

3. Test an incoming Check-Out (HTNG_HotelCheckOutNotifRQ)

  • Check if the invoices have no open balance
  • Request the partner to check-out the reservation
  • Check in the PMS if the Reservation has been check-out
    Verify in Kibana that incoming HTNG_HotelCheckOutNotifRQ is answered by HTNG_HotelCheckOutNotifRS and contains no errors.
  • Verify the Document data from the HTNG_HotelCheckOutNotifRS and ask Hotelbird whether they have received an email with the invoice (check evtl. the SPAM folder).

Void the payment/revenue once finished the test.
