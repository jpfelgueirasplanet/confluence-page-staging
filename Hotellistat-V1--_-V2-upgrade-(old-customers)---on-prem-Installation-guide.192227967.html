Installation Steps

 

General steps

Examples

1

As part of the installation, the customer will have to first set up all the rate codes, room type codes, occupancy and validity of the rates in the Hotellistat interface. 
For this to be completed, Hotellistat needs to organize the setup call with the customer before the I/O Installation on Protel side.
From the Protel installer side, it should be asked on Slack or email before the installation day if the setup call has been done.


2

  • Create a new installation with the V2 Manifest Name (com.hotellistat.version2rms)
  • Delete or inactivate the old Hotellistat installation for the specific HotelID (com.hotellistat.version2rms)

3

Hotellistat uses the same user and password as authentications for the installations, these are already set as default, so you don't need to request authentication details from the Integration partner for the new installation

Bob installation: one for OnPremise (if not existing), one for the Integration and one for the SCS Job 

  • provide the token corresponding to the integration partner on private Slack message or encrypted file by email and inform you by phone about the password.

One way Installation


1

PWS Configuration

  • enable LiveMode for required message types for 1-way connection

Message type

LiveMode

OTA_HotelInvCountNotifRQ 

OTA_HotelStatsNotifRQ 

OTA_HotelInvBlockNotifRQ

OTA_HotelResNotifRQ

IO_StatsNotifRQ

2

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


3

Create a test allotment/block in the future (at least one year after the system date) & provide the allotment name to your integration partner

  • insert a room type with a number of rooms to be blocked, and ask the partner if he received the allotment with this room type
  • Verify in Kibana that outgoing OTA_HotelInvBlockNotifRQ is answered by OTA_HotelInvBlockNotifRS and no error occurs
  • cancel the test allotment


4

Create SCS Jobs in the SMP


Details

  • one for OTA_HotelInvCountNotifRQ for 365 Days every 1 hour, starting with the next day from the installation date, at 00:01

  • one for OTA_HotelStatsNotifRQ
    ReportCode: HTNG_PastDailyPropertyStatistics_v1 for 730 days, yearly, starting with the present day.
    Please trigger this message immediately after it was created and remember that this should be triggered only once from the SCS tool.

    As additional info:
    The usual set of statistic reports with the message OTA_HotelStatsNotifRQ will be triggered automatically after an EOD is done by the customer in the PMS.     Hotellistat in interested in the following reports (and filters out the other ones) CurrentDailyPropertyStatistics and PastDailyPropertyStatistics)

OTA_HotelInvCountNotifRQ 


OTA_HotelStatsNotifRQ 

5

Verify in Kibana that outgoing OTA_HotelInvCountNotifRQ and OTA_HotelStatsNotifRQ is answered by OTA_HotelInvCountNotifRS and OTA_HotelStatsNotifRS from Hotellistat. 

Check with the integration partner if the two messages were received by their end as well. Provide the CID for these two messages if necessary.


6

Please delete the Job for OTA_HotelStatsNotifRQ from the SCS Job after the integration partner confirmed receiving the message OTA_HotelStatsNotifRQ.


7

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ) the last 2 years and the next 2 years from the installation date
  • for allotments/blocks (OTA_HotelInvBlockNotifRQ) the last 2 years and the next 2 years from the installation date




Two way Installation

Examples

1

PWS Configuration

  • enable LiveMode for required message types for 2-way connection

Message Type

LiveMode

OTA_HotelAvailNotifRS

OTA_HotelRatePlanNotifRS

2

Test incoming Rate update

(OTA_HotelRatePlanNotifRQ)

  • the integration partner should ask the customer to create a Test rate code
  • after the Test rate code has been created, ask the integration partner to push some rates
  • Ask in the channel for which day the rate has been pushed, check the date and send a screenshot from the Protel Daily Rates if the rate for the specific date is showing in the list. 


3

Test incoming Restrictions 

(OTA_HotelAvailNotifRQ)

  • Ask the Integration Part in the Slack channel of the integration to push a restriction for a specific day on the Test rate code and then check in the Price availability for the given date if the restriction has been set. 
  • Ask the Integration partner to clarify with the customer to have the Test rate code invalidated (through expired validity) after the verification of the Restriction message is done


4

Update the protel SMP page with the information about the installed customer

Development SMP page / Development / Services / Prospect Hotels / Add
