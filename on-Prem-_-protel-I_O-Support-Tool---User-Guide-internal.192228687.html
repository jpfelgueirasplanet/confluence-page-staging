General Information

In the protel I/O Support Tool here is a hidden admin area, this area is exclusive for employees of protel Hotelsoftware GmbH. In this area, failed messages can be resent, MarketCode can be added to reservations, ProfileNumber reservations that no longer exist can be added again, XML can be imported and the internal progress queue of the PWS can be checked. 


User Guide

How to use the Admin Area in the Support Tool?

At the top of the support tool there is a button called "ADMIN", click on this button. 


A login window opens, use the protel master password (version 23.000)


All additional features are enabled automatically.

Resend Failed Messages

In the tab, failed messages can be easily retriggered. 

Message cannot be sent in live mode due to various reasons. These messages are saved in the Protel database with “MessageGenerationFailed” or “SendingFailed”.

Select the correct time period, message types and errors , with a click on the "refresh" Button all messages will be displayed. The Messages will be displayed in a table with Eventvalue, CorrelationID, Knownmessage, Created, Updated, LastError, Retries and MessagePayload.


To resent a message just select an entry and click on "Resend" at the bottom.

Of course it is possible to trigger multiple message at once.

As soon as you click on "Resend" the table will be updated automatically.

Resend Failed Messages from a Bulk

Similiar to the tab "Resend Failed Messages",  failed messages from a bulk upload can be easily retriggered. 

Select the correct time period, message types and errors, with a click on the "refresh" Button all messages will be displayed. 





The Messages will be displayed in a table with pwsbtsjobid, idvalue, CorrelationID, Created, Updated, LastError, Retries.

To resent a message just select an entry and click on "Resend" at the bottom.

Of course it is possible to trigger multiple message at once.

As soon as you click on "Resend" the table will be updated automatically.

This message will be resend in the  Live Mode.

Progress

In the tab "Progress" the Result from a Web Call is displayed, that shows which message the PWS is currently processing or whether the PWS is processing a message at all.

If the PWS is processing any messages, it is looking like this.

Please use the "Refresh" Button to modify the progress.

In the column the "queue" can be checked, if the "queue" is "BuildQueue" the pws tries to build the specific message. If the "queue" is in "SendQueue" the pws tries to sent the message out.

In the column "startUTC" the start time can be control, if it is abnormal long, something could be wrong with the message.

XMLRead/Import

In this tab "XML Read/Import"  messages can be read or imported, without any connection to the ESB. 

To use this function, simply paste your message into the text field

As soon as you click on "read" the XML will be checked. If an XML is wronly formated an Error will popup. If all is correct, nothing will popup.

If you click on "Import", Message will be imported, and the PWS LogFile will open. 

In the "Incoming Messages" tab you can check the Results as well.

Please keep in mind, if you use this function, the message will be really push to the system.

MarketCodes

Manual correction of reservation is necessary for all IDeaS Integration.  This UI was developed so that no direct database corrections have to be made. But in the meantime, a easier-to-use ckits have been developed. To find this ckit, please check the IDeaS Installation Guide.

This UI is so no longer needed.

ProfileNumber

Manual correction of profiles was necessary for PWS version < 2.13.x, otherwise they were not sent correctly during the bulk upload. This UI was developed so that no direct database corrections have to be made. But it is now fixed in the PWS directly, so no manually database correction are needed anymore.

This UI is so no longer needed.
