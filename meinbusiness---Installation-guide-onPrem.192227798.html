Installation Steps

 

Needed action

Examples

1

Request authentication details from the Integration partner for the new installation and request them to send you the authentication details  on a private Slack message.

Please provide them the HotelID as well - internally for Protel this is the CloudID 

  • Integration Partner differenciates Hotel solely via Hotel ID 
  • Authentication method for all installations is the default Endpoint and default TOKEN for each hotel customer:
    • at the manifest are 3 dedicated default stages ( prod, prod.test & dev.test ) 
      • "prod" is the overall default for installations 



2

General Preparation for onPremise

  • no special x-Setups needed to this integration

IO_Pre-Setup-Checklist

3

Bob installation: one for onPrem (if not existing) and one for the Integration  

  • provide the token corresponding to the integration partner on private Slack message or encrypted file by email and inform them by phone about the password.

 

Needed action - One way Installation


1

Create a test reservation in the future (at least one week after the system date)  to trigger messages

  • confirm with Partner if messages have been recieved 
  • cancell testreservation





Check if message OTA_HotelInvCountNotifRQ is answered by OTA_HotelInvCountNotifRS and no error occurs.



Check if message HTNG_HotelFolioNotifRQ is answered by HTNG_HotelFolioNotifRS and no error occurs.

2

Integration Partner will inform us if Bulk Upload are required and specify date range  


If required by Hotel / Integration Partner: 

Create and start bulk uploads

  • OTA_HotelInvCountNotifRQ
  • HTNG_HotelFolioNotifRQ


BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 

Date Range for Bulk Upload if requested by Integration Partner

e.G: 

Month to Date on Installation Date .

e.G.

  • Install Date - 15.06,YYYY
  • => Date Range 01.06.YYYY - 15.06.YYYY

To controll if BTS Jobs is a success, compare Messages total send with Messages recieved by Partner. 

3

Data Validation via Manager Report 

  • after checking if the bulk successfully finished, generate the Manager Report Report to compare  revenue (MTD/ LYTD) with the integration partner 
    • revenue accomodation
    • revenue F&B
    • revenue extra / other
  • The Integration partner will review his system and provide revenues to compare 

Attention, if hotel uses the onPrem Bankett Module:

  • invoices generated out of the bankett module don't trigger HNTG_HotelFolioNotifRQ and therefore are not send to integration partner 
  • meinbusiness is aware and will subtract Bankett revenues out from their calculation

via the Manager Report it easy to compare "Month to Date" and if applicable "Last Year to Date" 

4

The one-way installation is complete once the bulk uploads have taken place and meinbusiness has confirmed the data is fine.

5

Update the protel SMP page with the information about the installed customer
