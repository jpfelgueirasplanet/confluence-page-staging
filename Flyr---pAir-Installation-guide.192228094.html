One way Installation


1

Create a test reservation in the future (at least one year after the system date)

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 
  • always provide the reservation number to your integration partner, so that the reservation can afterwards be cancelled


2

Create and start a bulk upload

  • for reservations (OTA_HotelResNotifRQ) for the last years and ending with the next 31st Dec 2099 (2 years)


BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


3

Data Validation Report - DQE

  • after checking if the bulk successfully finished, provide the integration partner the report
  • The Integration partner will process the DQE and send you a report afterwards
  • The report is to be found in the tab Office (Büro) of the front office in pAir.

Which rooms/configs to consider for the DQE:

  • Do not consider → PM/Pseudo, OOO, same day departure, cancellation 


 

Two way Installation

Examples

1

Rates are updated by the Integration system

(OTA_HotelRatePlanNotifRQ)

  • make a screenshot from pAir System Data with the short names of  all rate codes from Protel (please sort them by short name before), make a screenshot and send it to the Integration partner  
  • Pace defines the rate together with the customer and Pace is telling us the rate.
  • Pace can only update one single Rateplan


                       

2

Test incoming Rate update

(OTA_HotelRatePlanNotifRQ)

  • Pace defines the test rate code with the customer
  • after the Test rate code has been created, ask the integration partner to push some rates
  • Make sure that the rate which needs to be updated is set to yieldable. Ask in the channel for which day the rate has been pushed, check the date and send a screenshot from the Protel Daily Rates if the rate for the specific date is showing in the list. 

3

  • Create subdirectory with the IDS instance ID

4

  • Activate in the IDS the rate message to be exported. Please note that availability Values and Restriction values are not needed, so no tick there is necessary!
  • Ask Pace and the customer which rate plan codes and room types are needed to be mapped in Protel and then include them in the Mappings. Please note that the codes are case sensitive and have to be matched exactly as they are mentioned!

Sending Rates to Pace
