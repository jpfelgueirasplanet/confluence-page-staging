Challenges during installations


Possible reasons for failures during the decision upload:

  • IDeaS sending AmountBeforeTax instead of AmountAfterTax
    If OTA_HotelRatePlanNotifRQ to PMS fails, check first if IDeaS is sending the correct type of amount in the message. There is a filter in the ESB that transforms the message. The filter is based only on the AmountAfterTax values. If this amount is not provided, then the message, that reaches the PMS will be with empty rate values - and thus - will fail.

  • IDeaS sending RatePlanNotifRQ messages with empty CurrencyCode 
    Sample Error in RS:
    Error:No (yieldable) Rateplan found with Rate plan code 'RACK' and with Currency code ''



  • IDeaS sending wrong SOAP Action (needs to be equal to the Message Type name)

Full Data Export in case of discrepancies

IDeaS sometimes asks for running a "Full Data Export" or "Full Data Sync" in case they have encountered a lot of discrepancies for a specific customer.

This Sync always contains

  • Bulk upload for reservations
  • Bulk upload for blocks (allotments)
  • Trigger a set of all 6 StatsNotif messages

for a specific time period.

The time period usually covers 2 years of past data + at least 1 year in the future. The important thing is that the StatsNotif messages sent cover the whole time period of sent reservations / blocks.

A Sync can also be done just for a specific month. In such cases the following is valid for sending the StatsNotif messages:

  1. In case the sync is only done for a future period, the regular daily (past 45 and future 365/560 days - depending on the customer's set up) summary statistics (set of 6) need to be sent
  2. In case the sync is for past days only (before the daily past 45 day period), then we would require the summary statistics message (set of 6) which starts from the beginning of the sent period until future 365/560 days - depending on the customer's set up
