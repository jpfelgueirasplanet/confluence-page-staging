Installation Steps

 

General steps

Examples

1

InterAlp uses the same EP, Basic User-Pass for all of their customer installations. Those are set as default in the manifest and do not need to be changed. InterAlp distinguishes the different customers by the protel HotelID.

Please provide them the HotelID as well - internally for Protel this is the CloudID 

 InterAlp does not use the protel bearer token for sending RS messages. The unique reference for the workflow is the CID + CloudID pair

One way Installation


1

PWS Configuration

  • enable LiveMode and activate ToggleTrigger for required message types for 1-way connection

Message type

LiveMode

ToggleTrigger

OTA_HotelResNotifRQ

OTA_HotelInvCountNotifRQ

IO_HotelStatsNotifRQ

2

a) Using manual trigger link, send OTA_HotelInvCountNotifRQ for 2 years (730 days) in future

b) Verify in Kibana that the outgoing message is answered with OTA_HotelInvCountNotifRS message from the Integration system

on prem | Triggering messages: API calls (PWS):

http://127.0.0.1:8000/ota/hotelinvcountnotif/{mpe}/send/730

3

a) Using manual trigger link, send IO_StatsNotifRQ for 5 years in past and all data in future

b) Verify in Kibana that the outgoing message is answered with IO_StatsNotifRS message from the Integration system

on prem | Triggering messages: API calls (PWS):

Future: http://127.0.0.1:8000/io/statsnotif/send/{mpe}/3/365

Past: http://127.0.0.1:8000/io/statsnotif/send/{mpe}/4/1825

4

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


5

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ)
    for the past 5 years and everything in future

BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


6

Data Validation Report - DQE

The DQE is to be validated based on Gross/Brutto values


  • after checking if the bulk successfully finished, provide the integration partner the report
  • The Integration partner will process the DQE and send you a report afterwards
  • The report is to be found in the tab Add-Ons of the front office

 Once the Integration Partner receives the protel DQE report, they require about 1 week of time to be able to properly evaluate the quality of the connection. The Integration Partner Team will get in touch with the protel support once they are finished.

7

Update the protel SMP page with the information about the installed customer

To be found here: 

Development SMP page / Development / Services / Prospect Hotels / Add
