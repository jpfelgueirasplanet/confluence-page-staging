How to trigger Statistic messages manually (on prem)

  • Connect to Server where PWS is running

  • Open Google Chrome or Internet Explorer (or any other Browser)

  • Post the following URL's (one by one - always wait with the next one until the actual one is finished):

    Sample

    http://127.0.0.1:8000/ota/hotelstatsnotif/send/4/1/45
    http://127.0.0.1:8000/ota/hotelstatsnotif/send/4/2/365
    http://127.0.0.1:8000/ota/hotelstatsnotif/send/4/3/365
    http://127.0.0.1:8000/ota/hotelstatsnotif/send/4/4/45
    http://127.0.0.1:8000/ota/hotelstatsnotif/send/4/5/365
    http://127.0.0.1:8000/ota/hotelstatsnotif/send/4/6/45


Number of Days

  • 45 for the past
  • 365 for the future

    per default!


This could be different (configurable per XSetUps):

Structure URL API Call

http:// IP-Address: Port (of PWS) /ota/hotelstatsnotif/send/mpehotel/ReportCode/NumberOfDays


Example

  • Port: 8000
  • MPE: 3
  • Reportcode: 2
  • Number of days: 45

--> http://127.0.0.1:8000/ota/hotelstatsnotif/send/3/2/45


  • a pop up will appear & you have to log in to PWS with:

User: apiuser<MPE>

PW: apiuser<MPE>

 

Log In PWS

Use the apiuser for the property you want to send the Statistics for!

 

Example

→ mpehotel=4 

  • User: apiuser4
  • PW: apiuser4


Internet Explorer:



Google Chrome:

 

  • the browser will load until PWS is done with the request (could take some minutes or seconds)

  • Internet Explorer (depending on settings for the browser):
    → a little yellow box will pop up at the bottom of the window (--> you don’t have to do anything with it)

    OR

    Google Chrome:
    → you will get some kind of "success" message in the browser (–> this is the path where the message is stored additionally)





  • Post the next URL...and the next...and the next….

Close the browser (this is important to delete the Cache for the next time)!
