One way Installation


1

  • create a test reservation  with the approval of the hotel on the current date 
    • please check that the guest Profile has a language chosen 
    • assign a room and check-in
  • after check-in, inform the Integration Partner and ask them to execute a OTA_ReadRQ
  • Check  if Integration Partner has provided the correct Res ID

2

  • verify in Kibana that the incoming OTA_ReadRQ message is answered with a corresponding OTA_ResRetrieveRS message and no error occurred

3

  • adjust the test reservation and insure that a connected Guest profile with a different language is chosen 
  • ask the integration Partner to send  a OTA_ReadRQ for a Profile Read
  • check if the language change is recognized


2-step async approach

  1. Request Reservation (no guest language available)
  2. Request Profile (guest language available)

If no Language is chosen, WND defaults to the property default language.

4

  • verify in Kibana that the incoming OTA_ReadRQ message is answered with a corresponding OTA_ProfileReadRS message and no error occured

5

  • Check the  in-house total number 


  • WND counts all Rooms with status in-house regardless of departures or stayovers

6

  • Undo the check-in 
  • cancel the  test reservation
  • set room type back to the original status


  • if the status is unknown, change it to "touched" and inform Hotel to check the room status

7

Update the protel SMP page with the information about the installed customer
