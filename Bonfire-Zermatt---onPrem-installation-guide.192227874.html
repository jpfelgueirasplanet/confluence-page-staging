Special XSetup Configuration

Add the following XSetups:

Interface_rescommon | agequalifyingcode_child1

Interface_rescommon | agequalifyingcode_child2

Interface_rescommon | agequalifyingcode_child3

Interface_rescommon | agequalifyingcode_child4


The value should be 7, 9 or 10 depending on how the hotel has defined the child age groups.

Children in Zermatt:

The following resort fee amounts are valid in Zermatt:

Adults: CHF 3.00 per person and night

Children 9-16 years: CHF 1.50 jper person and night

Children below 9 years : free of charge

Bonfire needs to get the Age Qualifying Code 7 or 9 to identify the children:

Children with Code 9 = Children between 9 and 16 years

Children with Code 7 = Children under 9 years

Depending on how the hotel works, the Child 1 - 4 fields should be defined as either 7, 9 or 10 (for adults) with the exisitng Xsetups.

The integration partner does not send us any RS message - we only get an ACK.  This is also configured in the manifest.


3

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ)
    from January 1, 2017 years and ending with the next 10     years

BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


4

Data Validation Report - DQE


  • after checking if the bulk successfully finished, provide the integration partner the report
  • The Integration partner will process the DQE and send you a report afterwards
  • The report is to be found in the tab Add-Ons of the front office

For this integration, we are interested in the number of adults and children and not the number of rooms.  So AFTER running the DQE Report, pull up the following data via SQL Query either in the protel SQL Editor or direct via the SQL Server Management Studio:

select convert(varchar, date, 104), adult_occ, ch2_occ, ch1_occ, ch4_occ, ch3_occ from ptemp_hotelstatus where mpehotel=XY order by date asc

XY should be substituted by the MPE Hotel number and the children counts must be amended according to the child age groups used in the hotel. (eg. if Child 3 and 4 are not used, then these columns ch4_occ and ch3_occ can be removed from the above query)
