General information

The IO POS interface is a combination of IO and classic interface and can be used by various vendors.

There is a classic interface communication between the interface and pAir / OnPremise for getting guest data and send postings.
The communication between interface and 3rd party vendor is routed through the ESB (Enterprise Service Bus).
Important to know: if a customer has installed a PWS and a POS Interface on the same PC, the websocket connection will close each other.  So please make sure the PWS and the POS Interface are installed on different servers.

Every message has its own CorrelationID.
This unique ID is necessary for any support issues as it makes it possible to track this message within the ESB communication flow.

HTNG_ReadRQ

sent from POS to ESB to request guest data, the ESB will forward the message to the classic interface which will send the needed information back as HTNG_ResRetrieveRS

  

HTNG_ResRetrieveRS

sent from Classic Interface to ESB to respond to HTNG_ReadRQ, the ESB will forward the message to POS 

HTNG_ChargePostingRQ

sent from POS to ESB to charge any posting, the ESB will forward the message to the classic interface which will post the data to pAir and respond with an HTNG_ChargePostingRS

HTNG_ChargePostingRS

sent from Classic Interface to ESB to respond to HTNG_ChargePostingRQ, the ESB will forward the message to POS
