Installation Steps

 

General steps

Examples

1

Request authentication details from the Integration partner for the new installation and request them to send you the authentication details on private Slack message or encrypted file by email and inform you by phone about the password.

Please provide them the HotelID as well - internally for Protel this is the CloudID 

Qualitelis identifies for which hotel the data is being sent solely on the basis of the HotelID sent.

  • Authentication method using the same Endpoint, User and password for all hotels.  All the values are stored as defaults and do not need to be changed per hotel.

Bob installation: one for OnPremise (if not existing) and one for the Integration  

  • provide the token corresponding to the integration partner on private Slack message or encrypted file by email and inform them by phone about the password.
  • The 3 supplementary variables - ENV_QUALITELIS_CREATE, ENV_QUALITELIS_MODIFY and ENV_QUALITELIS_DELETE have default values and these values do not need to be changed.

One way Installation


1

PWS Configuration

  • enable LiveMode and activate ToggleTrigger for required message types for 1-way connection

Message type

LiveMode

ToggleTrigger

OTA_HotelResNotifRQ

2

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Qualitelis does not send us an RS.


3

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ)
    for all future reservation    s


BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


4

Data Validation Report - DQE

No DQE required for this integration.

*** Please note the constraints on Qualitelis side *** :

    1. Any reservations sent without an e-mail Adress or telephone number will be IGNORED by Qualitelis
    2. Qualitelis does not allow two reservations on the same day for the same guest. The second message is ignored by Qualitelis.
    3. Qualitelis does not allow two guest profiles with the same e-mail address. The second message is ignored by Qualitelis.

These constraints make it impossible to conduct a DQE.


5

Update the protel SMP page with the information about the installed customer
