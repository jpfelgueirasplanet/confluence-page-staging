Installation Steps

 

General steps

Examples

1

Request authentication details from the Integration partner for the new installation and request them to send you the authentication details on a private Slack message or encrypted file by email and inform you by phone about the password.

Please provide them the HotelID as well - internally for Protel this is the CloudID 

Authentication method using the same Endpoint as default and a specific User and Password given by the third party system for each hotel customer

EP: https://app.rateboard.io/RateboardEA/ws/htng/2014b/async

2

General Preparation for pAir General (pAir)


3

Bob installation: one for pAir (if not existing) and one for the Integration  

  • provide the token corresponding to the integration partner on private Slack message or encrypted file by email and inform them by phone about the password.

 

One way Installation


1

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


2

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ) for the last 2 years and ending with 31st Dec 2099

BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


3

Data Validation Report - DQE

  • after checking if the bulk successfully finished, provide the integration partner the report
  • The Integration partner will process the DQE and send you a report afterwards
  • The report is to be found in the tab Office (Büro) of the front office in pAir.
  • Please have in mind, that you should leave some time between the bulk finish and generating the CSV (statistics-export) from the Rateboard system, as when not enough time has elapsed  you have differences in Rateboard data in your CSV-file and may be looking for an error in Pair, while the root cause of your difference is that Rateboard has not computed all data, provided by bulk upload yet.

  • DQE has to be done due to T. Psenner Rateboard until: 31.12.2020 ONLY


4

1-way installation is completed with this point

  • Could be done as a preparation for 2-way during the 1-way installation
  • The customer should clarify with Rateboard which rate codes need to be updated by the integration partner when activating 2-way
  • Make a screenshot from pAir System Data with the short names of all the required rate codes from Protel (please sort them by short name before), and send it to the Integration partner

  • If the customer has a best western interface or an IDS interface, it might be necessary to deactivate the IDS from exporting changes until the rate is updated and checked.




Two way Installation

Examples

1

Test incoming Rate update

(OTA_HotelRatePlanNotifRQ)

  • you can find Settings for specific Integration systems (RMS) on General (pAir)
  • you need to activate the Daily Rates per XSetUp
  • the integration partner should ask the customer to create a Test rate code
  • after the Test rate code has been created, ask the integration partner to push some rates
  • Make sure that the rate which needs to be updated is set to yieldable.
  • Ask in the channel for which day the rate has been pushed, check the date and send a screenshot from the Protel Daily Rates if the rate for the specific date is showing in the list. 
  • Child Rates are not supported as DailyRates could not store rate values for children

Please note! If the customer has previously manually processed the daily rates with another mode (e.g. percentage surcharge), there will be an error message at rate update 

"ASSERTION_FAIL mixing modes not possible: 1!=OverrideWithAbsoluteValue"

These manual entries needs to be deleted before an upload can be done.




2

Test incoming Restrictions 

(OTA_HotelAvailNotifRQ)

  • Make sure that the rate which needs to be updated is set to yieldable.
  • Ask the Integration Part in the Slack channel of the integration to push a restriction for a specific day on the Test rate code and then check in the Price availability for the given date if the restriction has been set. 


3

Ask the Integration partner to clarify with the customer to have the Test rate code invalidated (through expired validity) after the verification of the Restriction message is done


4

Update the protel SMP page with the information about the installed customer
