What is the Self-Testing Portal?

The protel I/O Self-Testing Portal is a tool, that is build for external Integration Partner, to run the certification for outbound messages by themselves. Additionally, inbound test cases will be documented in this tool. The Integration Partner also has the possibility to check our message logging. 

How to open the Self-Testing Portal?

Open the integrationHUB → search for your product (our sample product calls "Self-Testing").


Click on the 3 dots on the right side, click on "Edit Product"


Click on the highlighted icon:


The Self-Testing Portal will open. Select the PMS you would like to certify - OnPremise or pAir:


The test cases will be loaded, pending on the selected PMS:



Page structure of the Self-Testing Portal

On the top right side, you have the possibility to check the Installation Token that is configured for your test hotel. Please use this token for your inbound request. The PMS token is used by us to push the request to you.


Additionally, you have the possibility to search for CorrelationIDs. You can check inbound or outbound CorrelationIDs. In addition, you can select the time range for which the CorrelationID should be searched, by default 15 minutes are selected. 


In the center of the page, the configured inbound and outbound messages are displayed. These messages are configured in your manifest according to your Integration Agreement.

The structure for inbound and outbound is almost identical. The minor differences will be explained in the next sections.


XSD Validation

In the inbound section, you always have to check first the XSD Validation. This means, you have to check if your XSD for the message is in general valid, according to our XSD definition. To use the XSD Validation, just run an inbound request and open the specific message type.

Run an inbound request and click on XSD Validation.

You can check in the tool, if your inbound request was successful or failed. If you click on the status "Failed" or "Success" you can see our logging for this specific CorrelationID.


Test Cases

As soon as you open a special message type for inbound or outbound, the test cases we expect will be displayed:


To get further information for a specific test case, just click on the test case:

On the top, the name and the description of the test case is displayed:

Under the test case description, you have different possibilities for outbound or inbound test cases:

  • Outbound: you have the possibility to "Trigger XML", this means, that the test message will be pushed to your endpoint. This enables you to check the content by yourself.
  • Inbound:  you need to enter the used CorrelationID manually. The inbound test cases needs to be done together with the Integration Specialist. Currently there is no possibility for you to check the inbound requests in our PMS.


As soon as you have either used "Trigger XML" or added your CorrelationID, the structure of the page is again the same.
Under the CorrelationID the attributes / values we would like to check are displayed:

Please add for each attribute the correct value in the free text box. If an attribute is not supported by you, please select the Status "Not Applicable", the Integration Specialist will check if "Not Applicable" is acceptable for a connection with protel I/O.

Additionally the Integration Specialist will check your values later, and will add the status

  • "Pass"
  • "Fail"

As soon as you have completed a test case, please select the corresponding status:

You can select between

  • "Success" → from your point of view everything is successfully
  • "Pending" → you are still working on it
  • "N/A" → in general not supported by your system
  • "Question Submitted" → you have open questions

If you select "Question Submitted", please use the free text field box under the status for your questions:


Please click on "Save" before leaving the screen. The test case will be updated on the overview page.


The Integration Specialist will check your test results and will add their status: 

The Integration Specialist can use the following status: 

  • "Approved" → everything is as expected
  • "Pending" → outstanding to be checked by the Integration Specialist
  • "Not Approved" → something is wrong and needs to be checked again by the Integration Partner
  • "Question Checked" → Question from the Integration Partner is answered


If both parties have added the status "Success" and "Approved", the test case will be marked as "Success" in the overview:




As soon as all Messages Types are certified successfully, you are ready to start the pilot phase.
