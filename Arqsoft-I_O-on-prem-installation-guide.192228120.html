One way Installation


1

PWS Configuration

  • tick Activated for required message types for 1-way connection

Message type

LiveMode

ToggleTrigger

OTA_HotelResNotifRQ

OTA_ProfileCreateRQ

OTA_ProfileModifyRQ

2

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


3

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ)
    for the last  month and ending with the next one month

  • for profiles (OTA_ProfileCreateRQ) 
    for the last one month and ending with the next one month


BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 

NO DQE REQUIERED !!! Only compare the number of messages send and received !!!

Payment Posting on ProfileID

Arqsoft is using the HTNG_ChargePostingRQ when the payment will be done directly in their interface. They are posting the revenue and the payment on a profile so they are using type= 21; a passerby invoice is then created. Type =21 is used due to the fact on a fiscalized db we have limited number of folio.

Payment Posting
<HTNG_ChargePostingRQ xmlns="http://htng.org/2014B" EchoToken="EchoToken_String" Version="0" CorrelationID="CID_string">
<POS>
<Source xmlns="http://www.opentravel.org/OTA/2003/05">
<RequestorID ID="28545424" Type="21" ID_Context="protelIO"/>
<BookingChannel Type="4" Primary="true"/>
</Source>
</POS>
<Posting ID="7">
<Transaction TicketID="13">
<RevenueDetails>
<RevenueDetail Description="American Express" PMSRevenueCode="4" CurrencyCode="EUR" Amount="-100.00" SubTypeID="Payment">
<Account ID="23889" Type="14" ID_Context="protelIO"/>
</RevenueDetail>
</RevenueDetails>
</Transaction>
</Posting>
</HTNG_ChargePostingRQ>
