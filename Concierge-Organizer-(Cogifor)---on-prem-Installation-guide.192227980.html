Special Configuration

Trace Department(s)

  • Check with the hotel on which Trace Department CO should send their traces to.  It is also possible for CO to send the information to different trace departments.
  • The trace department or departments are to be set up in system data - under Reservation - Departments - Departments

  

  • Once the trace department is set up, inform CO that these can be pulled via the IO_SystemDataRQ (please note IO_SystemDataRS needs to be in lIve Mode for this to work) or
  • provide CO with a list of the trace departments they should use

One way Installation


1

PWS Configuration

  • enable LiveMode and activate ToggleTrigger for required message types for 1-way connection

Message type

LiveMode

ToggleTrigger

OTA_HotelResNotifRQ

OTA_ProfileCreateRQ

OTA_ProfileModifyRQ

2

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


3

Create and start bulk uploads

  • for reservations (OTA_HotelResNotifRQ)
    for the next 10     years (please verify with customer and vendor)


BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


4

Data Validation Report - DQE


  • The DQE is NOT mandatory, only in case of obvious discrepancies a check should be done together with the CO Support team.





 

Two way Installation

Examples

1

PWS Configuration

  • enable LiveMode for required message types for 2-way connection

Message Type

LiveMode

OTA_ResRetrieveRS

IO_ResModifyNotifRS

HTNG_HotelFolioRS

OTA_ProfileReadRS

HTNG_ChargePostingRS

IO_SystemDataRS

2

Test an incoming trace

(IO_ResModifyNotifRQ)

  • Create a test reservation
  • ask the Integration partner to send a new trace to the reservation
  • Check that the trace is set correctly in the reservation.
  • Ask the integration partner to modify the trace
  • Check that the trace is modified correctly in protel


3

Test incoming revenue posting

(HTNG_ChargePostingRQ)

  • using the same reservation above, ask the integration partner to send a revenue posting to the invoice
  • ask the integration partner to cancel the posting and send a new posting
  • Check that the two postings plus the correction posting is sent correctly to the invoice in protel


4

Update the developer portal with the information about the installed customer

To be found here: 

Developer portal / Development / Services / Prospect Hotels / Add
