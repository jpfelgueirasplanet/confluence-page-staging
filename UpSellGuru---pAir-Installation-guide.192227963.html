General steps

You don't need to request authentication details for the new installation (they are always the same).

Please provide them the HotelID as well - internally for Protel this is the CloudID 

Authentication method using the same Endpoint as default and always the same User and Password

General Preparation for pAir General (pAir)



3

  • provide the token corresponding to the integration partner on private Slack message or encrypted file by email and inform them by phone about the password.

One way Installation


1

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


2

Create and start bulk uploads

  • for all future reservations (OTA_HotelResNotifRQ) , starting with the installation day till 2099


BTS Jobs will be processed one by one as only one bulk per property is allowed at the same time. 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


3

Data Validation Report - DQE

  • DQE is not necessary for this integration


 

Two way Installation

Examples

1

Test an incoming modification for a test reservation

(IO_ResModifyNotifRQ)

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • After the testing is done, cancel your test reservation. Verify in Kibana that incoming IO_ResModifyNotifRQ is answered by IO_ResModifyNotifRS and no error occurs

UseCases to test: 

  • Room type change with Upsell (fixed charges)
  • Create a fixed charge using an article (with UpsellInd=false)


2

UseCase1: Room type change with Upsell = true

  • A list of all to be used room types need to be given to Upsell Guru if not already provided 
  • let the partner modify the reservation for the room type
  • ask the customer to check the change in the reservation room type and the Fixed charges 
 Room type change with Upsell = true
<p:IO_ResModifyNotifRQ xmlns:p="http://protel.io/soap" CorrelationID="test05" TimeStamp="2020-03-10T10:33:51+00:00" Version="1.00">
<p:Reservations HotelCode="2572">
<p:Reservation>
<p:UniqueID ID="11493" Type="14" ID_Context="protelIO"/>
<p:Slices>
<p:Slice From="2020-02-04" To="2020-02-05">
<p:Room RoomType="GVS" RoomTypePreset="LVR"/>
</p:Slice>
</p:Slices>
<p:FixedCharges>
<p:FixedCharge Action="CREATE" UpsellInd="true" Recurrence="OnArrival" Quantity="1">
<p:InvoiceText>Room upgrade from LVR to GVS</p:InvoiceText>
<p:Item ItemID="66"/>
<p:Price CurrencyCode="AUD" SingleAmountAfterTax="20"/>
</p:FixedCharge>
</p:FixedCharges>
</p:Reservation>
</p:Reservations>
</p:IO_ResModifyNotifRQ>

3

UseCase2: Create a fixed charge using an article (with UpsellInd=false)

  • A list of all to be used fixed charges need to be given to Upsell Guru with Item.ID and Item.Code if not already done
    IMPORTANT: In the message sent from Upsell Guru these codes need to be sent as ItemID = Item.ID and Code = Item.Code!

  • let the partner modify the reservation for the fixed charges. The message should look like 

Create a fixed charge using an article (with UpsellInd=false)
<p:IO_ResModifyNotifRQ xmlns:p="http://protel.io/soap" CorrelationID="TestResmodify1" TimeStamp="2020-01-30T10:58:51Z" Version="1.00">
<p:Reservations HotelCode="2967">
<p:Reservation ArrivalDate="2020-02-04">
<p:UniqueID ID="26261" Type="14" ID_Context="protelIO"/>
<p:FixedCharges>
<p:FixedCharge Action="CREATE" Code="Upsell" UpsellInd="false" Recurrence="Fixed" Quantity="1">
<p:InvoiceText>Bottle of Wine - Saturday 20 Apr</p:InvoiceText>
<p:Price SingleAmountAfterTax="19.5" CurrencyCode="EUR"/>
<p:Item Code="202000" ItemID="01"/>
</p:FixedCharge>
</p:FixedCharges>
</p:Reservation>
</p:Reservations>
</p:IO_ResModifyNotifRQ>
