 To mitigate the problems:

  1. Make sure PWS is connected to its own task server
  2. The new task server should have nothing else running against it, no ckits, no IDS
  3. IDS should be pointed to it own task server and not the comserver
  4. Task server should be set to reload system data cache every 60 minutes
    https://wiki.protel.net/index.php?title=TaskServer_Setup
  5. set this on taskserver.ini as that is where the PWS is pointing
    [system]
    debugevtimerexecutiontimer=1
    thresholdevtimer=900
  6.  Make sure passerby account is configured and available
  7. If there is a timeout or error, check what the task server displays on the screen.

Issues to log in into Code2Order dashboard:
