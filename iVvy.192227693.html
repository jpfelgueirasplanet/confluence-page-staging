    OTA_HotelResNotifRQ

    Reservations
    • iVvy ignores reservations without BlockID
    • Pick Ups done in protel sent for a prospect block will only be shown in the number of pick ups (no reservation details)
    • for reservations with a different RateCode than saved in the block, iVvy will send back the following error:
      "Relevant accommodation group for this reservation not found. Possible mismatches: Room type, rate plan, arrival date, departure date."

    PMS

    PWS

    OTA_HotelResNotifRQ

    Reservations
    • Not supported details:
      • source codes
      • Credit Card Details

    • market codes / reservation notes could only be changed on Block level
      → this will not trigger a Reservation message from iVvy
      → only in case of changing something else on Res level it will get changed for the reservation as well

    • Cancelled reservations can't be re-instated
    • no Multi Unit Reservations - each reservation will be sent in a separate message
    • only Child 2 supported
    • Rate Values will be set on reservation level from IVvy using the linked rate code from block
    • pick ups for blocks after the Cut-Off has passed in iVvy are possible (configurable if they will be delivered to PMS or not)

    PWS

    OTA_HotelInvBlockNotifRQ

    Allotments
    • as soon as a block is Definite (blocking) it will not be reverted to Prospect (non-blocking)
    • contact profile could be private or company
    • blocks will always contain specific room types, no global allotments supported by iVvy
    • one note type configurable to be synced from iVvy
    • Cancellation reason for blocks related to Ivvy is supported

    PMS

    PWS

    OTA_ProfileModifyRQ

    Profile Modification
    • iVvy sends modifications only for profiles attached to blocks created in iVvy
      (Creation of the profiles is done along with the inbound reservations/blocks)
    • Profile Types supported:
      • Private
      • Company
    • Profile Details supported:
      • First/Lastname
        (for company profiles: first name is saved in kunden.name2 - as company profiles do not have this field)
      • Address Line 1-3
      • ZIP
      • City
      • Country (check ISO codes, they support 2 or 3 digit codes → configuration in iVvy)
      • Phone (no Mobile, Fax)
      • eMail (in case the contact hasn't an email address, iVvy will attach a fake one as this entry is mandatory in iVvy)
      • Profile deletion is not supported from Ivvy
