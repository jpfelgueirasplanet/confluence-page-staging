One way Installation

Examples

1

PWS Configuration

  • enable LiveMode and activate ToggleTrigger for required message types for 1-way connection & additionally the Global Live Mode.

Message type

ToggleTrigger

LiveMode

OTA_HotelResNotifRQ

OTA_HotelInvCountNotifRQ


2

Create a test reservation in the future (at least one year after the system date) & provide the reservation number to your integration partner

  • make a modification and finally a cancellation of your test reservation. Verify in Kibana that outgoing OTA_HotelResNotifRQ is answered by OTA_HotelResNotifRS and no error occurs. 


3

Create and start bulk uploads

  • for all future reservations (OTA_HotelResNotifRQ) , starting with the installation day till 2099 

Bulks should be starting with 1 January and ending with 31 of December (i.e. 01.01.2017 - 31.12.2021).

Check for failed messages after the bulk upload: Manual Correction (Support-Tool Use Cases) 

In case you have issues with a stuck bulk upload, investigate why messages were not sent. As an option, delete the old job and create a new one. 


4

Testing Occupancy

(OTA_HotelInvCountNotifRQ)

  • trigger the OTA_HotelInvCountNotifRQ starting once on the server of the customer where the PWS service is running (for a period of 6 months)
  • Verify in Kibana that the outgoing messages OTA_HotelInvCountNotifRQ is answered with RS message from the Integration system
  • for Occupancy the PWS will send the OTA_HotelInvCountNotifRQ automatically 3 times a day at 5,14 and 22 o'clock

Important: At MPE database you must be logged in to PWS with the user of the certain hotel you want to generate InvCount message for.
(usually apiuser1, apiuser1 => for MPE Hotel 1, apiuser2, apiuser2 => for MPE Hotel 2....)!

Open a browser to post the URLs:

(NOTE - you have to change some details here!)

pattern:
http://IP-address(PWS):Port(PWS)/ota/hotelinvcountnotif/{MPE_hotel}/send/{START:YYYY-MM-DD}/{END:YYYY-MM-DD}

e.g.: http://127.0.0.1:8046/ota/hotelinvcountnotif/1/send/2020-01-13/2020-12-31


WITH PWS BUILD 3.4 AND HIGHER:

http://127.0.0.1:8000/ota/hotelinvcountnotif/{MPE_hotel}/{NUMBER OF DAYS}

 e.g.: http://127.0.0.1:8012/ota/hotelinvcountnotif/3/365


 For more details on the URL triggers please visit this page: on prem -Triggering messages - API calls (PWS)

 

Two way Installation


1

PWS Configuration

  • enable LiveMode for required message types for 2-way connection

Message Type

LiveMode

IO_ResModifyNotifRS 

2

Test an incoming modification for a test reservation                     (IO_ResModifyNotifRQ)

UseCases to test: 

  • Room type change incl. fixed charges for upsell charges
  • Create a fixed charge using a default TAA (with attribute UpsellInd=true)
  • Create a fixed charge using an article (with attribute UpsellInd=false)


3

UseCase1: Room type change with Upsell using fixed charges 

  • Request the partner to change the room type with upsell of the reservation created during OTA_HotelResNotifRQ
  • The partner should use the room types you provided during the general preparation step
  • Verify in Kibana that incoming IO_ResModifyNotifRQ is answered by IO_ResModifyNotifRS and no error occurs
  • Open the Navigator in the PMS and check the room type change and the Fixed charges were posted successfully. 


Room type change with Upsell using fixed charges 
<?xml version='1.0' encoding='utf-8'?>
<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">
<env:Body>
<p:IO_ResModifyNotifRQ xmlns:p="http://protel.io/soap" CorrelationID="AKO_Room_Upgrade5" TimeStamp="2020-01-30T10:58:51Z" Version="1.00">
<p:Reservations HotelCode="3329">
<p:Reservation ArrivalDate="2019-09-01">
<p:UniqueID Type="14" ID="2389726" ID_Context="protelIO"/>
<p:Slices>
<p:Slice From="2019-09-01" To="2019-09-03">>
<p:Room RoomType="FCA"/>
</p:Slice>
</p:Slices>
<FixedCharges>
<p:FixedCharge Action="CREATE" Code="Upsell" UpsellInd="true" Quantity="1">
<p:InvoiceText>Upsell from 1BR to FCA</p:InvoiceText>
<p:Price SingleAmountAfterTax="25.00" CurrencyCode="EUR"/>
</p:FixedCharge>
</FixedCharges>
</p:Reservation>
</p:Reservations>
</p:IO_ResModifyNotifRQ>
</env:Body>
</env:Envelope>

4

UseCase2: Create a fixed charge using a default TAA (with UpsellInd=true)

  • Request the partner to create a fixed charge with UpsellInd=true
  • The partner should use the ukto.kto you provided during the general preparation step
  • Verify in Kibana that incoming IO_ResModifyNotifRQ is answered by IO_ResModifyNotifRS and no error occurs
  • Open the Navigator in the PMS and check whether the Fixed charges were created. 



Create a fixed charge using a default TAA (with UpsellInd=true)
<?xml version='1.0' encoding='utf-8'?>
<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">
<env:Body>
<p:IO_ResModifyNotifRQ xmlns:p="http://protel.io/soap" CorrelationID="AKO_Upsell23" TimeStamp="2020-03-10T10:33:51+00:00" Version="1.00">
<p:Reservations HotelCode="3329">
<p:Reservation ArrivalDate="2019-08-19">
<p:UniqueID ID="2389724" Type="14" ID_Context="protelIO" />
<p:FixedCharges>
<p:FixedCharge Action="CREATE" Code="Upsell" UpsellInd="true" Quantity="1">
<p:InvoiceText>Upsell</p:InvoiceText>
<p:Price SingleAmountAfterTax="15" CurrencyCode="EUR" />
</p:FixedCharge>
</p:FixedCharges>
</p:Reservation>
</p:Reservations>
</p:IO_ResModifyNotifRQ>
</env:Body>
</env:Envelope>



5

UseCase3: Create a fixed charge using an article (with UpsellInd=false)

  • Request the partner to create a fixed charge with UpsellInd=false
  • The partner should send ItemID = articles.short  and Code = ukto.kto
  • Verify in Kibana that incoming IO_ResModifyNotifRQ is answered by IO_ResModifyNotifRS and no error occurs
  • Open the Navigator in the PMS and check whether the Fixed charges were created. 


Create a fixed charge using an article (with UpsellInd=false)
<?xml version='1.0' encoding='utf-8'?>
<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">
<env:Header>
<p:Action xmlns:p="http://protel.io/soap">IO_ResModifyNotifRQ</p:Action>
<htng:RelatesToCorrelationID xmlns:htng="http://htng.org/PWSWG/2007/02/AsyncHeaders">Upsell19</htng:RelatesToCorrelationID>
</env:Header>
<env:Body>
<p:IO_ResModifyNotifRQ xmlns:p="http://protel.io/soap" CorrelationID="Upsell19" TimeStamp="2020-03-10T10:33:51+00:00" Version="1.00">
<p:Reservations HotelCode="3329">
<p:Reservation DepartureDate= "2019-08-21" ArrivalDate="2019-08-19">
<p:UniqueID ID="2389724" Type="14" ID_Context="protelIO" />
<p:FixedCharges>
<p:FixedCharge Action="CREATE" Code="Upsell" UpsellInd="false" Recurrence="Daily" Quantity="1">
<p:InvoiceText>Bottle of Wine - Sunfay 20 Apr</p:InvoiceText>
<p:Price SingleAmountAfterTax="15" CurrencyCode="EUR" />
<p:Item Code="100104" ItemID="201" />
</p:FixedCharge>
</p:FixedCharges>
</p:Reservation>
</p:Reservations>
</p:IO_ResModifyNotifRQ>
</env:Body>
</env:Envelope>
