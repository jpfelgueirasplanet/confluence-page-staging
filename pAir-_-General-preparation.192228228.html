Daily Rates

  • Settings for specific Integration systems (RMS) 
    • In case you have a system installed that is handling rates i.e. Hotelpartner or Rateboard check that the DailyRates have been activated in the XSetup
    • All rates must have at least a value of 0,00

For a rate push to work, no manual entries of the customer should be set in the Daily Rates module for the specific rate codes.

Since the support cannot delete entries in the pAir Daily Rates covering a longer period of time, it is advisable to create a development ticket in advance so that the developers can delete the entries on the day of installation, if necessary.  It must be done in advance so that it can be scheduled by the product management.

For distribution partners:
Please check with Integrations Support Team to create such a ticket if required.

If the integration manages Rates in the PMS

In order to have a Rate managed by a third party system through the Incoming OTA_HotelRatePlanNotifRQ message, the customer must not influence the calculation of the desired yieldable rate in the DailyRates by adding prices or formulas. The customer should simply set the yieldable rate to a minimum value of 0.
The following error occurs otherwise: "ShortText="mixing modes not possible: 2!=OverrideWithAbsoluteValue DailyRate(179)"
